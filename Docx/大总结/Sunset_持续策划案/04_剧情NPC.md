# Sunset 剧情推进、春 1 日切片与 NPC 系统

## 1. 文档目标

本卷用于统一 `Sunset` 当前剧情推进与 NPC 系统的正式口径，重点明确：

1. `spring-day1` 为什么是当前叙事主入口。
2. 剧情如何真正驱动农田、制作、状态条与新手教学。
3. NPC 如何从功能验证对象推进为角色化、场景化、可持续成长的系统。
4. 聚落复兴当前是如何被剧情、角色与经营系统共同承载的。

## 2. 当前叙事定位

### 2.1 当前项目叙事前提

`Sunset` 当前主叙事前提已经明确为：

玩家扮演一名失忆工匠，流落到残破聚落，在采集、制作、建造与日常劳动中逐步恢复聚落功能，并通过工作台、记忆闪回、角色关系与阶段性事件，一点点找回自己的过去。

这意味着 `Sunset` 的叙事不是外挂在经营玩法外面，而是天然与以下内容绑定：

1. 劳动与生存。
2. 聚落基础设施恢复。
3. 记忆与知识的重新接通。
4. 与聚落住民的关系建立。

### 2.2 当前剧情线的正式任务

当前剧情线的首要任务不是铺开长篇主线，而是先证明以下三件事能够成立：

1. 剧情能作为玩家进入经营系统的入口。
2. 教学、事件、对话、提示与世界交互能共存于同一条体验链。
3. NPC 和聚落不是空场景说明牌，而是真正参与节奏与引导的角色。

## 3. 为什么 `spring-day1` 是当前叙事主入口

### 3.1 当前切片策略

当前项目没有先做一条大而全的长主线，而是先把“春 1 日”做成正式验证切片。

这一步的意义非常明确：

1. 先验证玩家第一次进入世界时，剧情如何带出系统。
2. 先验证对话、CG、事件触发、教学提示与 NPC 引导能否协同。
3. 先验证农田、工作台、状态条与精力警告能否被剧情自然接起，而不是硬塞教程面板。

### 3.2 当前 Day1 的正式承载目标

`spring-day1` 当前不是单段剧情，而是一条完整体验链，至少负责：

1. 建立世界前提与主角处境。
2. 让玩家认识村庄、住处与第一批角色。
3. 让血条、精力条、工作台、农田、砍树等系统首次进入玩家视野。
4. 在晚餐、冲突、夜间提醒与自由时段中，把“今天先活下来”与“明天还要继续经营”这件事落稳。

### 3.3 当前 Day1 的正式范围

根据 `requirements.md`、`OUT_design.md` 与场景承载约束，当前 Day1 不应被理解成一张大而全场景，而应拆成多段承载区：

1. 开场 / 坠落 / 初遇区。
2. 入村过渡区。
3. 农舍院落主场景。
4. 小屋室内 / 治疗 / 床铺区。
5. 工作台与闪回点。
6. 农田 / 砍树教学区。
7. 晚餐 / 冲突区。
8. 归途 / 夜间提醒 / 自由时段承载区。

当前 `SceneBuild_01` 的正式身份，已经被收束为：

`14:20 进入村庄` 后，到 `15:10 耕种 / 砍树教学` 前后的住处安置与教学主场景。

这条约束非常关键，因为它防止 Day1 被误做成一个杂糅所有剧情段落的大场景。

## 4. `spring-day1` 当前正式流程

### 4.1 当前阶段顺序

当前 `Story_Manager` 已经明确以固定顺序组织 Day1：

1. 开场 CG
2. 坠落与相遇 `14:00`
3. 初入村庄 `14:20`
4. 疗伤与血条 `14:30`
5. 工作台与闪回 `14:45`
6. 耕种教学 `15:10`
7. 晚餐与冲突 `16:00`
8. 归途与夜间提醒 `17:00`
9. 自由时段
10. 睡觉结束

这不是一份松散建议，而是当前剧情切片的正式骨架。

### 4.2 各阶段的叙事作用

#### 开场 CG

当前职责：

1. 用黑屏、星空、流星坠落与昏迷，先建立“来到陌生世界”的起点。
2. 在玩家还未接管操作前，完成氛围定调与身份悬念埋设。

#### 坠落与相遇 `14:00`

当前职责：

1. 让玩家第一次被角色接住并进入关系网络。
2. 引入骷髅兵远景威胁，只做视觉压力，不直接进入战斗。
3. 通过乱码对话与记忆线索，建立主角“认知断裂”的核心设定。

#### 初入村庄 `14:20`

当前职责：

1. 把玩家正式带进聚落空间。
2. 让引导不只是路点导航，而是进入“有人带我回到一个还能活下去的地方”的叙事情境。

#### 疗伤与血条 `14:30`

当前职责：

1. 第一次让血条作为世界里的真实状态出现，而不是冷启动 UI。
2. 用治疗场景把“角色脆弱性”与“聚落提供最低庇护”一起交代出来。

#### 工作台与闪回 `14:45`

当前职责：

1. 让工作台第一次不只是功能按钮，而是记忆触发器。
2. 通过白闪与闪回，把主角“拥有某种跨时代制作知识”的设定接进玩法。
3. 闪回结束后解锁基础制作配方，让叙事直接兑现为生产能力。

#### 耕种教学 `15:10`

当前职责：

1. 正式把劳动主线接上世界。
2. 通过锄头、种子、教学提示与地块交互，让农田成为 Day1 最重要的玩法入口。
3. 让精力条第一次出现，并把“劳动成本”明确交给玩家感知。

#### 晚餐与冲突 `16:00`

当前职责：

1. 用晚餐恢复部分精力，建立“生活节奏”和“吃饭就是恢复”的日常规则。
2. 用冲突或立场差异，把角色关系从纯引导升级为真正存在张力的人际结构。

#### 归途与夜间提醒 `17:00`

当前职责：

1. 把脚本化剧情逐步交回自由探索。
2. 在夜间提醒里告诉玩家：这一天没有结束，聚落生活接下来会进入更长期的经营节奏。

#### 自由时段与睡觉结束

当前职责：

1. 让玩家在已理解基础规则后拥有短暂自由决策空间。
2. 通过床铺交互完成 Day1 收束，并把时间推进到春 2 日。

### 4.3 当前 Day1 的系统绑定关系

当前 Day1 至少已经把以下玩法入口正式缝进剧情：

1. `Dialogue_System` 负责信息推进。
2. `CG_System` 负责开场、闪回、过场切换与白闪。
3. `Tutorial_Prompt_System` 负责移动、教学、警告与提示。
4. `NPC_Guide_System` 负责角色带路与空间引导。
5. `Story_Manager` 负责阶段推进、时间控制与变量管理。
6. 农田、工作台、精力、血条、背包与交互系统作为被剧情接入的已有玩法层。

这意味着 Day1 的价值不只是“把新系统搭起来”，而是证明剧情系统已经能接入玩法骨架。

### 4.4 当前 Day1 的正式阶段表

当前 `spring-day1` 不是松散事件串，而是有明确阶段目标、系统挂点与玩家焦点的正式推进链：

| 阶段 | 时间 | 当前核心动作 | 当前系统挂点 | 当前玩家焦点 |
|---|---|---|---|---|
| `CrashAndMeet` | `14:00` | 与首个关键 NPC 完成第一次接触 | 对话、首段引导、世界前提建立 | 靠近 NPC 并按 `E` 开始首段对话 |
| `EnterVillage` | `14:20` | 进入聚落、听完后续说明 | 角色引导、空间切换、聚落导入 | 听完村长说明，正式进入聚落节奏 |
| `HealingAndHP` | `14:30` | 疗伤与血条首次出现 | 剧情演出、状态条显示 | 看完疗伤对白与血量卡片 |
| `WorkbenchFlashback` | `14:45` | 靠近工作台并触发回忆 | 工作台交互、白闪、记忆解码、基础配方解锁 | 靠近 `Anvil_0`，按 `E` 打开工作台并完成回忆 |
| `FarmingTutorial` | `15:10` | 锄地、播种、浇水、砍树、回工作台制作 | 农田、精力、背包、木材收集、工作台制作 | 按顺序完成开垦、种植、浇水、收木材与一次真实制作 |
| `DinnerConflict` | `16:00` | 晚餐事件与关系张力建立 | 事件对白、角色冲突、恢复节奏 | 看完晚餐事件，承接情绪收束 |
| `ReturnAndReminder` | `17:00` | 夜间提醒与归途过渡 | 提示、时间控制、归家逻辑 | 听完提醒，接住 Day1 最后一段过渡 |
| `FreeTime` | - | 交还玩家短时自由控制权 | 自由探索、床铺交互、世界提示 | 在已理解规则后自由活动，再决定回屋休息 |
| `DayEnd` | - | 睡觉收束 Day1 | 睡觉、次日推进 | 完成春 1 日收口并进入下一天 |

## 5. Day1 的关键叙事机制

### 5.1 乱码与认知恢复

当前需求已经明确：

1. 初期对话可以带有乱码或认知不完整表现。
2. 记忆闪回触发后，`languageDecoded` 被设置为不可逆真值。

这说明 `Sunset` 的记忆线不是纯文本设定，而是直接作用于剧情理解能力的系统变量。

### 5.2 状态条首次展示

当前 Day1 已明确把状态条展示做成剧情事件，而不是开局常驻：

1. 血条在 `14:30` 疗伤阶段首次显示。
2. 精力条在 `15:10` 耕种教学前后首次接入。
3. 低精力会触发警告提示，并伴随移动惩罚。

这个设计的意义是：状态 UI 首次出现本身也成为叙事的一部分，玩家不是在读说明书，而是在体验“身体开始真实受规则约束”。

### 5.3 工作台闪回与制作解锁

当前工作台闪回是 Day1 最关键的“叙事兑现玩法”节点之一：

1. 玩家在工作台旁按交互键触发白闪与回忆。
2. 闪回结束后解锁基础制作配方。
3. 这一步把“主角会做东西”从设定直接变成可操作能力。

这条链路决定了 `Sunset` 的叙事不是空讲背景，而是通过交互对象直接发放玩法权限。

## 6. NPC 系统的当前正式结构

### 6.1 当前四条主线

NPC 系统当前已经明确拆成四条主线：

1. 主线 A：导航协作边界。
2. 主线 B：场景化真实落点与角色日常。
3. 主线 C：轻交互、关系成长与双气泡体系。
4. 主线 D：受击 / 工具命中 / 反应兼容。

这说明 NPC 已经不再被当成“对话对象”看待，而是被拆成一个真正可扩展的角色系统。

### 6.2 当前三名核心 NPC 基线

当前 NPC 文档已经明确至少三名核心角色：

1. `001` 村长
2. `002` 村长女儿
3. `003` 村长二儿子 / 研究型

这三名角色当前并不是挂了编号的空壳，而是已经被写入角色职责与说话方向。

#### `001` 村长

当前定位：

1. 承担秩序、判断与聚落运行感。
2. 更适合出现在村庄主通路、广场与“能观察全局”的位置。
3. 说话简洁、稳重、偏事务判断。

#### `002` 村长女儿

当前定位：

1. 承担生活感、情绪层与人情温度。
2. 更适合庭院边缘、采光舒适、生活气更强的位置。
3. 说话更柔和、更生活化，更适合承接轻松与情绪反馈。

#### `003` 村长二儿子 / 研究型

当前定位：

1. 承担观察、记录、现象解释与变化感。
2. 更适合研究台、边角观察位、路口与材料变化明显的位置。
3. 说话有记录感、推测感，天然适合与“主角的制作 / 记忆 / 现象理解”形成呼应。

## 7. NPC 的场景化、轻交互与关系成长

### 7.1 当前场景化原则

当前 NPC 线已经明确一个重要原则：

`不是只摆一个 Home Anchor。`

这意味着角色场景化当前必须至少考虑：

1. 家 / 常驻区。
2. 常见停留点。
3. 路过点。
4. 更可能遇见别人的公共点。
5. 适合独处或自言自语的边缘点。

NPC 是否“像个活人”，首先取决于他为什么在这里，而不是有没有一条巡逻线。

### 7.2 当前双轨交互结构

NPC 交互当前已经被正式拆成两条轨：

1. 正式对话轨。
2. 日常气泡轨。

两条轨道的职责不同：

1. 正式对话轨负责任务、剧情、信息推进。
2. 日常气泡轨负责生活感、存在感、轻反馈与关系预热。

这条拆分非常关键，因为 `Sunset` 的 NPC 目标不是“一靠近就弹长对白”，而是先让角色在世界里活起来，再在需要时进入重对话。

### 7.3 当前关系成长结构

当前关系成长线已明确预留最小四阶段：

1. 陌生
2. 认识
3. 熟悉
4. 亲近

当前正式要求是：

1. 同一句轻交互反馈不能在所有关系阶段都保持不变。
2. 同一种命中或工具行为，也不能在所有阶段得到相同反应。

这说明 NPC 的关系成长不是将来再补的数值表，而是已经开始影响文本、反馈和行为语气的结构层。

### 7.4 当前双气泡方向

当前文档已经明确玩家 / NPC 双气泡体系要保持：

1. 同一套节奏、尺寸与产品感。
2. 但玩家气泡与 NPC 气泡必须一眼可分。

这一步的意义，不只是 UI 好不好看，而是为了让“谁在说话、谁在回应、这是正式对话还是轻反馈”在视觉上被立刻读懂。

### 7.5 当前受击 / 工具命中兼容框架

NPC 当前还预留了更重的一层：

1. 区分普通接触、玩家攻击、工具挥击、误伤与特殊交互来源。
2. 把反应层级从无视到关系下降分层组织。
3. 保证这些反应未来能接到角色差异与关系阶段，而不推翻现有导航 / 漫游底座。

这说明 `Sunset` 的 NPC 设计已经开始考虑“玩家行为真正作用到角色身上”这件事，而不是只停留在可对话状态。

## 8. 当前核心 NPC 功能矩阵

当前 NPC 线已经不是单纯“给名字和性格”，而是形成了明确的系统分工：

| 角色 | 当前角色定位 | 当前常见落点 | 当前系统职责 | 当前叙事作用 |
|---|---|---|---|---|
| `001` 村长 | 秩序、判断、聚落运行感 | 村庄主通路、广场、能观察整体的位置 | 主说明、节奏推进、公共事务判断 | 把玩家正式接入聚落规则，承担 Day1 的权威说明与阶段推进 |
| `002` 村长女儿 | 生活感、照料感、情绪温度 | 庭院、采光舒适的生活区、住处周边 | 轻情绪承接、生活氛围反馈、柔性对话 | 让聚落不只像生存据点，而像有人在过日子的空间 |
| `003` 村长二儿子 / 研究型 | 观察、记录、推测、现象解释 | 研究台、路口、材料变化明显的位置 | 现象说明、制作 / 记忆 / 世界变化的解释节点 | 把主角的回忆、制作知识与世界现象连接起来 |

这三名角色共同完成三件事：

1. 让 Day1 的引导不只靠 UI 提示，而是通过角色关系自然推进。
2. 让聚落空间具备“公共秩序、生活温度、现象观察”三种不同气质。
3. 为后续正式对话轨、日常气泡轨与关系成长轨提供稳定承载点。

## 9. 聚落复兴当前的正式推进链

当前聚落复兴并不是一条空口号，而是已经通过 Day1 和基础系统落成了四段推进：

| 复兴阶段 | 当前剧情节点 | 当前系统锚点 | 当前玩家获得的东西 |
|---|---|---|---|
| 落脚与庇护 | 坠落、相遇、入村、疗伤 | 住处、血条、床铺、基础庇护 | 从“流落”转为“在聚落里暂时安身” |
| 生产能力恢复 | 工作台闪回 | 工作台、基础配方、记忆触发 | 主角第一次把回忆转成真正的制作能力 |
| 劳动能力恢复 | 农田教学、砍树、一次基础制作 | 农田、精力、木材、工作台制作 | 玩家开始用劳动推动聚落重新具备生产循环 |
| 日常节奏成形 | 晚餐、冲突、归途提醒、自由时段 | 关系事件、恢复节奏、睡觉收束 | 聚落从“临时避难点”转成“明天还会继续运转的生活空间” |

这条推进链说明：

1. 聚落复兴在当前版本首先不是大建筑解锁，而是“住下来、做起来、活起来”。
2. 剧情节点和经营节点已经共享同一条成长链，而不是各自平行。
3. 玩家的第一天并不是纯教程，而是聚落复兴的第一轮正式启动。

## 10. 当前正式收口

当前剧情与 NPC 卷的正式收口可以概括为四条：

1. `spring-day1` 已经从概念切片变成带固定阶段顺序、系统挂点和玩家目标的正式主入口。
2. 工作台闪回、血条 / 精力条首次出现、晚餐恢复与夜间提醒这些节点，已经证明叙事能直接发放玩法权限与状态认知。
3. `001 / 002 / 003` 三名核心 NPC 已形成明确角色职责，不再是没有系统价值的场景摆件。
4. 聚落复兴在当前版本已经有清晰的第一阶段推进链，主角与聚落的关系不再停留在抽象背景。
