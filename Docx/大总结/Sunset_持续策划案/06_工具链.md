# Sunset Unity 策划侧落地：工具链与数据资产

## 1. 文档目标

本卷用于统一 `Sunset` 当前策划侧落地链路的正式口径，明确数据资产如何组织、ID 与命名如何治理、Editor 工具如何支撑内容扩张，以及从设计数据到世界物体、场景与动画的生产路径。

## 2. 当前策划侧落地链路

### 2.1 当前统一链路

当前 `Sunset` 的策划侧落地链路可以概括为：

1. 确定物品、资源、配方、NPC、交互物的分类与字段结构。
2. 按 ID 规范与命名规范生产或修改基础 SO。
3. 将 SO 数据接入 `WorldPrefab`、`Recipe`、`Placeable`、`Storage` 等运行时载体。
4. 通过 Prefab、Tilemap、Sprite 与 Renderer 工具把资产转成可在场景中使用的玩法对象。
5. 通过动画与层级工具补齐三方向动作、排序、过渡与表现配置。
6. 通过验证类工具回查导航、放置、NPC、Story、存档等系统是否与数据口径一致。

### 2.2 当前链路的设计意义

这条链路说明 `Sunset` 当前并不是“少量手填资产 + 零散手摆场景”的开发方式，而是已经形成了面向批量扩张和重复维护的策划侧生产模型。

## 4. SO 规范、ID 体系与数据治理

### 4.1 当前正式治理入口

当前 SO 与工具规范的正式入口位于 `.kiro/specs/SO设计系统与工具`，已明确沉淀出以下主线：

- `ID分配规范.md`
- `0_字段优化重构`
- `1_ID规范和管理器修复`
- `2_批量生成物品工具优化`
- `3_ID规范全面固定`
- `4_WP生成`
- `5_批量修改SO参数`

### 4.2 当前 ID 体系

根据 `ID分配规范.md`，当前项目已明确把不同物品类型分入固定区段，例如：

- `0XXX`：工具与武器
- `1XXX`：种植与放置类
- `3XXX`：矿物与材料
- `4XXX`：药水
- `5XXX`：食品
- `8XXX`：装备

当前规范不只定义“哪类资产大概放哪里”，而是已经细到：

1. 各 SO 类型的起始 ID。
2. 各子类型的预留段。
3. 文件前缀命名规范。
4. 已使用 ID 与下一个可用 ID。

### 4.3 当前数据治理目标

这套治理当前主要解决三类问题：

1. 同类资产持续扩张时，如何避免 ID 冲突与命名漂移。
2. 当字段结构调整时，如何批量修正已有资产，而不是逐个手改。
3. 当运行时系统依赖统一数据口径时，如何保证设计、工具与资产实际一致。

| 工具组 | 核心工具 | 解决的痛点 | 直接支撑的业务模块 |

| 数据生产组 | `Tool_BatchItemSOGenerator`、`Tool_BatchItemSOModifier`、`Tool_BatchRecipeCreator`、`DatabaseSyncHelper` | 物品 SO 手填成本高、字段结构改动后批量返工困难、配方扩张效率低 | 工具、武器、材料、种子、作物、树苗、放置物、锁钥、食品、药水、配方 |
| 世界资产接线组 | `WorldPrefabGeneratorTool`、`BatchCreatePrefabs`、`BatchSpriteRendererSettings`、`PrefabDatabaseAutoScanner`、`PrefabDatabaseEditor`、`TilemapToSprite` | Sprite 资源难以快速变成世界玩法资产，`worldPrefab` 引用与数据库维护易漂移 | 世界掉落物、可放置物、箱子 / 工作台、资源点、场景素材转玩法资产 |

## 7. 当前工具链的正式使用方式

当前策划侧落地已经形成固定顺序：

1. 先用 ID 规范与命名规范定义数据边界，保证新增内容进入正确区段。
2. 再用 SO 批量工具生成或修正基础数据，避免内容扩张时退回纯手工维护。
3. 然后把数据接到 `WorldPrefab`、`Recipe`、`Placeable`、`Storage` 等运行时载体，确保“设计数据”变成“可运行资产”。
4. 最后再用场景、动画、验证工具把资产真正推到地图、表现层与验收链路里。

这条顺序意味着 `Sunset` 当前的工具链不是“做完功能后顺手写几个小脚本”，而是已经成为内容生产本身的一部分。

## 8. 当前正式收口

当前工具链卷的正式收口可以概括为四条：

1. 数据治理已经成为内容扩张底座，ID、命名、字段与批量生成工具是一起工作的。
2. `WorldPrefab`、`Placeable`、`Storage`、动画控制器、场景层级这些高频重复环节，已经不再依赖纯手工逐个配置。
3. 工具链输出直接服务经营、交互、剧情、NPC、表现层，而不是和业务脱节的独立技术实验。
4. 当前这卷已经能够真实体现“策划端如何把规则、数据、资产和验证串成一条生产链”。
