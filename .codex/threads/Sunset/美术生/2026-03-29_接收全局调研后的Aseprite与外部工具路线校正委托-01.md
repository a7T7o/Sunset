## 二、你必须吸收的新增信息

这些不是让你“参考一下”，而是本轮必须吸收进新判断的新增依据。

### 1. 外部候选的真实结论已经更清楚了

全局调研已经收口出一个硬判断：

- 市面上没有一个我愿意诚实称为“成熟、可信、Unity-aware 的现成美术总监 skill / MCP”
- 真正可关注的只有三类：
  - 参考与设计稿语义链：Figma MCP / Figma 官方设计实现工作流
  - 像素资产生产链：Aseprite / pixel-mcp / aseprite-mcp
  - 设计审稿流程链：`interface-design` 这类 prompt / skill 化设计规范

但它们都只补一部分：

- Figma 系列更偏“设计稿语义与对齐”
- pixel-mcp / aseprite-mcp 更偏“像素资产生产”
- interface-design 更偏“审稿与设计约束”

没有任何一个候选能单独解决 Sunset 当前“会动但丑、能摆但没叙事”的问题。

### 2. `pixel-mcp / aseprite-mcp` 的价值和边界

全局调研结论不是“不要看它们”，而是：

- 对小型像素资产、图标、提示气泡、血条边框、工作台小面板素材有真实帮助
- 对 UI 布局、层级、Prompt / Workbench 正式面、场景叙事、视觉重心帮助有限
- 安全面相对可控，主要风险来自本地执行 Aseprite 与临时 Lua 文件

所以如果你这轮还要继续谈 `pixel-mcp / aseprite-mcp`，正确口径只能是：

- 它们可能是“受限资产生产链”的局部执行器
- 不是 Sunset 的“审美总监”
- 不是 UI / Scene 方向问题的总解

### 3. Figma 方向当前不适配 Sunset 主链

全局调研已经确认：

- 当前 Sunset 更接近 `Canvas / RectTransform / TextMeshProUGUI`
- 不是 UI Toolkit 主链
- `UnityFigmaBridge` 和 `Unity.Package.FigmaToUnity` 都会引入平行 UI 流程
- 对当前像素风正式生产帮助有限，副作用偏大

所以本轮不要再把 Figma 方向写成主推荐路线。

### 4. Blender 方向当前价值极低

对 Sunset 当前 2D 像素 UI / 场景问题帮助极弱，会明显扩大工具面和复杂度，不值得。

### 5. 你当前对 Aseprite 的说法还要再收紧

你前一轮已经确认：

- 项目内有 `com.unity.2d.aseprite`
- 项目内也存在大量 `.aseprite` 资源

但你也已经确认本机当前：

- `aseprite` CLI 缺失
- Python 可用
- Node 可用
- `Pillow` 还未安装

所以你这轮对 Aseprite 的正确口径，不应该是“已经能直接串 CLI 产线”，而应是：

- V1 先走 `PNG-first + Python + Pillow`
- `.aseprite` 支持作为增强项或后续阶段
- 不把“当前项目里有 aseprite 包”误写成“当前本机已经有完整 Aseprite CLI 生产链”

### 6. 全局偏好里和你最相关的几条，必须纳入

你这轮不只是管流程，也在给后续“美术方向”定边界，所以必须吸收这些全局偏好：

- UI 不要“测试味”，要像真的给玩家游玩

- 场景搭建优先做可见构图，再补 invisible 语义
