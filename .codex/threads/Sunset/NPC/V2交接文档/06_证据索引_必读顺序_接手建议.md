## 3. 接手第一轮建议动作顺序

### 第一步：只读复核现场

至少复核：

1. `pwd`
2. `git branch --show-current`
3. `git rev-parse HEAD`
4. `git status --short -- 'Assets/000_Scenes/Primary.unity'`
5. `git diff --stat -- 'Assets/000_Scenes/Primary.unity'`
6. scene / lock / MCP 占用文档是否有新变化

### 第二步：只判断 scene 写窗口是否成立

只要下面任一项不成立，就继续只读：

1. `Primary.unity` dirty 归属已明确
2. 有独占写窗口或等价热区裁定
3. Unity 确认在 Edit Mode 且非 compile / reload
4. 没有新的 shared hot-zone 冲突

### 第三步：若窗口成立，只做一刀最小 scene 切片

推荐严格按这个顺序：

1. 先 audit `Primary.unity` 当前 001 / 002 / 003 实例状态
2. 只落 `HomeAnchor`
3. 不顺手扩活动路线和对话节奏
4. 做最小 live 回读后立即停

### 第四步：再决定是否继续第二刀

只有当 `HomeAnchor` 稳定后，再考虑：

1. roam 区域
2. 路线 / 停留点
3. 相遇节奏
4. 导航联调
