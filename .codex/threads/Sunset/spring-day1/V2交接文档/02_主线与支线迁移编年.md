# 主线与支线迁移编年

## 阶段 0：从规划走向主干落地（2026-03-22）

### 当时目标
把 spring-day1 从纯文档/设计推向可运行的最小业务骨架，尤其是 `0.0.2`。

### 当时动作
- 使用过 branch carrier 做基础脊柱迁入。
- 最终把 `0.0.2` 基础脊柱迁进 `main`。
- 关键基线提交：
  - `83d809a9` `spring-day1: migrate 0.0.2 story foundation`
  - `b64d4cdd` `spring-day1: organize 0.0 and handoff docs on main`

### 结果
- `StoryManager / StoryPhase / DialogueSequenceSO / DialogueManager / NPCDialogueInteractable` 这一层正式进入主干。
- spring-day1 不再只是规划线程，而是拥有真实代码面。

### 对后续的影响
- 这一步之后，后续所有 Day1 运行态和体验问题，才有真实接线入口。
- 也埋下一个长期风险：线程容易从“剧情主线”不断吞到 UI、工作台、任务提示、工作台交互、测试、导演层。

---

## 阶段 1：首段剧情与 Day1 最小闭环（2026-03-23）

### 当时目标
让 `NPC001` 交互能真正走 spring-day1 的首段剧情，并把状态推进跑起来。

### 当时动作
- 接了首段 / follow-up 对话。
- 接了 `LanguageDecoded`、首段推进、follow-up 分流。
- 接了对话调试入口、导演层骨架、测试骨架。

### 结果
- `0.0.2` 从“代码结构存在”变成“玩家可触发首段剧情”。
- Day1 开始有真实运行链，而不是只在 `SO` 和 `Manager` 之间空转。

### 对后续的影响
- 主线第一次从“纯剧情推进”延伸到“运行态可验”。
- 也开始出现“是不是需要顺手把 UI/体验一起做完”的倾向。

---

## 阶段 2：Day1 UI / 工作台 / 任务体验不断并入主线（2026-03-24）

### 当时目标
围绕 Day1 首日体验，把任务提示、工作台、任务页卡、工作台 UI 这些用户直接看到的部分落到项目里。

### 当时动作
- 开始制作 `SpringDay1PromptOverlay`。
- 开始制作 `SpringDay1WorkbenchCraftingOverlay`。
- 接入 `CraftingService / RecipeData` 和工作台配方。
- 尝试通过用户实时反馈，把工作台交互与 UI 迅速拉到“像个游戏”的状态。

### 结果
- Day1 主线从剧情链继续扩展为“剧情 + UI + 工作台 + 任务体验”的复合线程。
- 同时出现第一批严重反馈：不是不能用，而是体验和观感不合格。

### 对后续的影响
- 这一步之后，用户对 `spring-day1` 的要求不再只是“逻辑能跑”，而是“必须像正式可玩的内容”。
- 审美、层级、沉浸式、提示条、动画感开始变成硬验收项。

---

## 阶段 3：用户连续复测回拉，主线被强行改造成“体验验收驱动”（2026-03-24 ~ 2026-03-25）

### 当时目标
不再只做逻辑，而是围绕用户多轮实测反馈，一轮轮把体验拉正。

### 当时动作
多个委托逐步把要求收紧：
- `委托-01`：第一次明确把工作台/UI/任务体验当成 Day1 收口主线。
- `委托-02`：把用户最初 7 点要求与新增复测反馈一起钉死。
- `委托-03`：不再接受“代码差不多，等用户测”的停法。
- `委托-04`：在 farm 编译 blocker 解除后，要求做真实 Play 自验。

### 关键支线
1. **工作台 UI 支线**
   - 左列图标/滚动
   - 右侧详情/材料/数量
   - 进度条与制作按钮状态机
   - 离台小进度
   - 与工作台固定锚定
2. **任务卡 / Prompt 支线**
   - 日历撕页语义
   - 一任务一页
   - 完成动画
   - 对话期间隐藏 / 压制
3. **交互提示支线**
   - 工作台 `E`
   - NPC `E`
   - 教程态与常规态双语义
4. **运行态自验支线**
   - PlayMode 取证
   - farm 编译 blocker 排查
   - live 稳定性判断

### 结果
- 主线已经不再是传统意义上的“业务开发”，而是“业务 + 视觉 + 交互 + 验收回拉”的混合线程。
- 多个代码 checkpoint 在 `2026-03-24` 到 `2026-03-25` 持续进入 `main`：
  - `38dd71a9` ~ `84fc3818`
  - `dfba414d`

### 对后续的影响
- 这是当前 V2 必须继承的主线结构：Day1 不再只是一条剧情线，而是带着体验债和用户口径的综合线程。
- 也正是在这一阶段，线程最容易 scope 漂移、最容易误把“临时验证物/治理证据/他线尾账”吞到自己名下。

### 对后续的影响

1. 工作台 UI：已成为 Day1 主线的一部分，不再是支线噪音。
2. 任务卡 / Prompt 体验：已成为硬验收的一部分。
3. 运行态自验：已成为 Day1 线程不可回避的职责。

## 哪些支线只是噪声、误导、临时绕路
1. 把 `.kiro` 委托、样式快照当成可自删临时证据
2. 把 `Primary.unity` 当成 spring-day1 可以继续顺手写的 scene
3. 把“工作台 UI 运行时节点大致长得对”误判成“已经复刻了 prefab 样式”
4. 把“只有两份 UI 脚本”误判成当前最小提交闭包
