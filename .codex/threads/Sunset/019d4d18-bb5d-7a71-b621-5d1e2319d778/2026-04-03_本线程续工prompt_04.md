## 二、这轮唯一主刀

这轮只做一件事：

**把桥面、水边、地图边缘以及相关 tilemap props 的“能走 / 不能走”边界，收成尽量贴近玩家视觉的真实范围。**

说人话就是：

1. 不能再出现“脚已经站进水里才被拦住”
2. 不能再出现“桥明明还在脚下却像空气墙”
3. 不能再出现“看起来是实物却不挡人”或“看起来是空位却挡人”
4. 不能再让地图外边缘明显和绘制出来的场景边界错开

## 五、首轮必须先查清的三件事

### 1. 玩家脚底 probe 现在到底是偏宽还是偏松

重点回看：

1. `PlayerMovement.cs`
2. 当前脚底 probe 半径
3. 左 / 中 / 右 probe 的 support 判定
4. soft-pass 在桥面上到底是不是放宽过头

当前高嫌疑不是“桥完全没接上”，而是：

1. soft-pass 命中条件仍然过宽
2. 或脚底 query radius / lateral probe 仍然把桥边与水边混算

### 2. `NavGrid2D` 的 override / blocking 命中是不是仍然比视觉范围更粗

重点回看：

1. `HasWalkableOverrideAt(...)`
2. `IsWalkable(...)`
3. 当前对 tilemap 的面积命中 / 半径命中
4. 当前桥边、水边、地图边缘的命中是不是仍然按“整格语义”过宽或过窄

### 3. 当前 scene 里到底是哪一类源没有被正确纳入

重点核：

1. 哪些水边是被 blocking source 管住的
2. 哪些桥面是被 walkable override 放开的
3. 哪些视觉上像 `Props` 的 tilemap 其实根本没进 blocking source
4. 地图外边缘到底是 scene source 漏了，还是脚本侧 query 还在外溢
