# 2026-04-05 跨场景相机统一跟随：方向锚定与安全收口清单

## 1. 最终完成定义

这条线最终要做成的不是：

- `Primary` 勉强能跟
- `Town` 单独补锅
- 某些 UI 报错少一点

真正完成定义只有一个：

> 无论切到哪个场景，只要场景里存在合法的玩家根与合法的世界相机，镜头都必须自动接管、持续跟随、不中途漂移；同时所有世界锚点 UI 在拿不到合法相机时必须安全跳过，而不是继续硬投屏报红。

换成更具体的话，就是下面 4 条必须同时成立：

1. 切场后，镜头能自动重新绑定到当前场景真正的玩家。
2. 运行中如果 `Follow` 丢了、指错了、或相机缓存漂了，系统要自愈，而不是卡死在错误目标上。
3. `Primary` / `Town` / 后续新场景，不允许各自再写一套临时“补脑”逻辑。
4. 世界提示气泡、工作台浮层、世界锚点 UI 拿不到合法世界相机时，必须本帧安静返回，不允许打 `Screen position out of view frustum`。

## 2. 当前客观判断

当前问题不是“有两台主相机一直互抢”。

当前更像两个表面症状，共享同一个底层缺陷：

- 症状 A：`Screen position out of view frustum`
- 症状 B：镜头有时飘走，然后固定住，不再照玩家

共享的底层缺陷是：

> 项目里“世界相机解析 / 玩家解析 / 跟随自愈 / 世界点投屏”这一整条契约过松，到处都在使用 `Camera.main ?? first camera` 这种宽松兜底。

这会带来两类后果：

1. UI/浮层投屏链拿错相机时，容易直接打 frustum 红字。
2. 跟随链在运行时一旦跟错人，只要错误目标还“活着”，就可能不会触发重绑，表现成镜头漂走后卡死。

## 3. 这刀唯一正确修法

这刀不能继续“哪里报错修哪里”，而要按统一契约收。

### 3.1 统一世界相机解析

新增唯一可信的“世界相机解析器”，所有运行时代码统一从这里拿世界相机。

它必须满足：

- 优先锁定当前场景中真正输出世界画面的 `Main Camera`
- 必须是激活、启用、场景有效
- 优先要求带 `CinemachineBrain`
- 不能再退回“任意第一个 Camera”
- 如果当前场景没有合法世界相机，返回 `null`

### 3.2 统一玩家根解析

新增唯一可信的“当前玩家根解析器”，只认：

- 当前场景中真正带 `PlayerMovement` 的根物体

不再允许：

- 直接靠 `Player` tag 猜
- 把 `Tool` 这类子物体当成玩家

### 3.3 统一镜头自愈

`CameraDeadZoneSync` 的职责要改成：

- 场景加载时重新解析“当前世界相机 + 当前玩家根”
- 每帧或关键时机验证：
  - 当前 `Follow` 是否为空
  - 当前 `Follow` 是否仍然是当前最佳玩家根
  - 当前主相机是否仍然是合法世界相机
- 只要任一条件不成立，立即重绑，不允许“目标还活着就当正常”

### 3.4 统一世界锚点 UI 投屏守卫

所有调用 `WorldToScreenPoint` / `RectTransformUtility.WorldToScreenPoint` 的世界锚点 UI，都必须先做统一守卫：

- 没有合法世界相机：直接返回
- 世界点在相机后方：直接返回
- 世界点不在合法视口范围：直接返回

原则只有一句话：

> 宁可这帧不画，也不要硬投到错误相机上报红。
