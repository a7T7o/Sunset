## 2. 调用链导航图（文字版）

### 2.1 玩家遮挡主链
1. `OcclusionTransparency` 注册到 `OcclusionManager`。
2. `OcclusionManager.Update` 按 `detectionInterval` 调 `DetectOcclusion`。
3. 对每个 occluder 执行：距离过滤 -> 标签过滤 -> 同层过滤 -> 遮挡判定（像素/Bounds）-> 可选占比过滤。
4. 通过 `SetOccluding(true/false)` 触发渐变。

### 2.2 砍树高亮链
1. 工具命中树后，`TreeController` 调 `SetChoppingTree`。
2. `OcclusionManager` 保证同一时刻仅一个 `currentChoppingTree`。
3. 砍倒或超时后 `ClearChoppingHighlight` 清理。

### 2.3 预览遮挡链
1. `PlacementPreview` 每次更新预览区域时计算联合 bounds。
2. 调 `OcclusionManager.SetPreviewBounds(previewBounds)`。
3. 管理器执行 `DetectPreviewOcclusion`，只要 `Intersects` 即透明。
4. 预览结束时传 `null`，触发恢复。

### 2.4 树木状态 -> 导航链
1. `TreeController` 在阶段变化后更新碰撞启用状态。
2. 若碰撞状态有变化，延迟 0.2s 触发 `NavGrid2D.OnRequestGridRefresh`。
3. `NavGrid2D.RefreshGrid -> RebuildGrid` 全图重建导航可行走网格。

## 4. 结论摘要

- 遮挡不是单点脚本，而是“管理器调度 + 组件执行 + 树木/预览/导航多链联动”。
- 当前链路覆盖：玩家遮挡、树林整体透明、预览遮挡、砍伐高亮、导航刷新。
- 命中检测在不同链路存在策略差异（工具命中偏 Collider，点击交互偏 Sprite bounds），需在后续统一策略以减少体感差异。
