# 当前现场、高权重事项、风险与未竟问题

## 当前最高权重事项
- 当前最高权重事项只有一个：
  - 让 detour owner 真正稳定接管执行层
- 更具体地说，是让下面三处接成一条真实 runtime 闭环：
  - `PlayerAutoNavigator.HandleSharedDynamicBlocker / TryCreateDynamicDetour`
  - `NPCAutoRoamController.TryHandleSharedAvoidance / TryCreateSharedAvoidanceDetour`
  - `NavigationPathExecutor2D.TryCreateDetour / ClearOverrideWaypointIfChanged / TryClearDetourAndRecover`

## 现在最值得继续的工作
1. 先从代码事实继续，不先开新 live
2. 追一条完整链：
   - solver 产出 `ShouldRepath + SuggestedDetourDirection`
   - 玩家/NPC controller 接收后经过哪些 `if`
   - detour candidate 为什么没变成长期 `HasOverrideWaypoint`
   - 为什么又掉回 `BuildPath / TryRebuildPath / stuck`
3. 重点不是“多会算”，而是“谁在吞 detour owner”

## 当前已接受的结构性 checkpoint
1. 共享导航基础设施已经落地，不是空骨架：
   - `NavigationAgentSnapshot`
   - `NavigationAgentRegistry`
   - `NavigationAvoidanceRules`
   - `NavigationLocalAvoidanceSolver`
2. `NavigationPathExecutor2D` 已经承载了相当一部分共享执行职责：
   - 路径刷新
   - stuck/repath 状态
   - detour state / override waypoint state
3. 运行时当前仍以旧 solver 直出链为基线，而不是 `TrafficArbiter / MotionCommand`
4. 当前主叙事已经稳定收敛到：
   - `ShouldRepath` 已出现
   - detour owner 没有稳定接管执行层

## 当前未接受、未解决、未验证的部分
1. 用户体验没有通过：
   - 推着 NPC 走
   - 保护罩
   - 很远就停
   - 被围抽搐
2. detour lifecycle 底座虽有 API，但 runtime 并未完整接线
3. 玩家和 NPC 的 controller 侧仍残留旧 private loop：
   - `BuildPath / TryRebuildPath`
   - `CheckAndHandleStuck`
   - blocked steering / hard stop
4. `NavigationPathExecutor2D.TryClearDetourAndRecover()` 尚未进入现行运行链

## 当前 single first blocker
- 当前单一第一阻塞点已经固定为：
  - `PlayerAutoNavigator / NPCAutoRoamController / NavigationPathExecutor2D`
  - 在 detour owner 的创建、保活、释放上没有形成稳定闭环
- 现象等价描述：
  - solver 已经在说“该绕”
  - 但 controller 不是在真正绕，而是在：
    - 吞掉
    - 清掉
    - 冷却挡掉
    - 创建失败后退回旧重建路径

## 内部 blocker
1. 玩家侧：
   - `!avoidance.ShouldRepath`
   - repath cooldown
   - detour candidate 全灭后直接 `BuildPath()`
   - solver 某一帧丢 blocker 就可能清 override waypoint
2. NPC 侧：
   - `!avoidance.ShouldRepath`
   - cooldown
   - 先 `StopForSharedAvoidance()` 再 detour，失败就 `TryRebuildPath()`
3. 共享执行层：
   - candidate clearance 门槛严格
   - clear/recover API 没被 runtime controller 调用
