# 线程身份与职责

## 最初创建动机
- 线程最初不是为了一次性修一个小 bug，而是为了给 Sunset 导航系统建立长期可执行的重构主表与推进入口。
- 最早的核心判断是：
  - 当前导航不是统一系统
  - 而是静态路径层 + 玩家私有自动导航 + NPC 私有漫游补丁的分裂架构
- 因此线程初期职责是：
  - 建立统一导航重构阶段设计
  - 再逐步把共享导航契约、注册表、局部避让、路径执行层落地

## 历史上承担过的主要职责
1. 建文档基线：
   - 创建 `统一导航重构阶段设计与执行主表`
   - 把导航主线从零散 bug 修复升级为长期重构工作区
2. 建共享基础设施：
   - 扩展 `INavigationUnit`
   - 增加 `NavigationAgentSnapshot`
   - 增加 `NavigationAgentRegistry`
   - 增加 `NavigationAvoidanceRules`
   - 增加 `NavigationLocalAvoidanceSolver`
3. 把玩家与 NPC 逐步接到共享层：
   - 玩家 `PlayerAutoNavigator`
   - NPC `NPCAutoRoamController`
4. 把部分路径执行职责迁到 `NavigationPathExecutor2D`
5. 做 real-input 事故处置：
   - 不再拿结构进展顶体验结果
   - 直接围绕“推着 NPC 走 / 保护罩 / 很远就停 / 被围抽搐”做 fresh live
6. 最后把叙事收束到当前单一第一责任点：
   - `ShouldRepath` 已出现
   - detour owner 没有稳定接管执行层

## 当前真正身份
- 当前它已经不是“继续大范围设计导航架构”的线程。
- 它现在的真实身份是：
  - 导航回归事故末段收束线程
  - 负责把执行层握手链路压到足够窄、足够稳，能够无失真交给 `导航检查V2`

## 当前主要负责范围
1. 负责解释和修复这些文件之间的执行层断裂：
   - `Assets/YYY_Scripts/Service/Player/PlayerAutoNavigator.cs`
   - `Assets/YYY_Scripts/Controller/NPC/NPCAutoRoamController.cs`
   - `Assets/YYY_Scripts/Service/Navigation/NavigationPathExecutor2D.cs`
   - `Assets/YYY_Scripts/Service/Navigation/NavigationLocalAvoidanceSolver.cs`
2. 负责保持当前运行时基线不回漂：
   - 旧 solver 直出链仍是当前 runtime 基线
3. 负责把 live 症状和代码断点对应起来，而不是把 runner 结果和用户体感混验
4. 负责交代 hot-file 边界、dirty 边界、own/non-own 边界

## 当前明确不负责范围
1. 不负责重新开启 `TrafficArbiter / MotionCommand` runtime 接线实验
2. 不负责把导航主线扩回 `S0-S6` 整套宏大结构宣讲
3. 不负责顺手清理 `Primary.unity` mixed dirty
4. 不负责处理与导航主线无关的 `spring-day1`、Story UI、农田等外部问题
5. 不负责把所有 NPC 运动学、所有未来移动体统一到底层实现一次性做完

## 与其他线程的边界
- 与治理线程的边界：
  - 治理线程负责裁定口径、发 prompt、判 checkpoint
  - 本线程只负责按口径落地导航热区并给出真实代码/体验证据
- 与 NPC 线程的边界：
  - 本线程可以碰 `NPCAutoRoamController` 导航相关执行链
  - 但不应吞并 NPC 更大范围行为系统、气泡系统、聊天系统或 unrelated runtime
- 与 Scene/表现层线程的边界：
  - 本线程只做 `Primary.unity` 只读归属审计，不负责 scene 内容治理

## 哪些事项虽然相关，但不应再由本线继续吞并
1. “全面统一导航底座闭环是否已完成”的宏观裁定
2. “玩家与 NPC 最终运动学是否必须彻底同一实现”的总设计争论
3. “是否要重开先裁决后求解中轴”的结构重建
4. 与导航无关的 shared root hygiene 清扫

## 交给 `导航检查V2` 的一句话职责
- 接班后直接把 detour owner 的创建、保活、释放接成真正的 runtime 闭环，并用最小同轮 fresh live 验证它是否真的消灭了“推土机 / 保护罩 / 抽搐”。
