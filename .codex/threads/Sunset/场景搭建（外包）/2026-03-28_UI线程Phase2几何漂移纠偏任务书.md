## 五、必须写死的几何纪律

### A. PromptOverlay 的纪律

#### 1. 不准再改外壳横向几何

你不要再在 runtime 里重新改：

- `TaskCardRoot` 的宽度
- `TaskCardRoot` 的 X / Y 位置
- `Page / BackPage` 的宽度
- `Page / BackPage` 的横向位置关系

#### 2. 如果必须长高，只允许向下长

也就是：

- 顶边保持视觉锚定不乱
- 宽度不变
- X 不变
- 上面固定，下面往下延伸

#### 3. 优先让内层模块往下推，而不是先改整张卡

应该优先做：

- `Subtitle` 高了，`TaskList` 往下
- `TaskList` 高了，`FocusRibbon` 往下
- `FocusRibbon` 高了，`Footer` 往下

而不是先做：

- 整张卡先变形

#### 4. 如果 prefab 本身已有合适骨架，优先用布局组件和 `preferredHeight`

不要继续用手搓：

- `anchoredPosition`
- `sizeDelta`

去重新摆整页。

### B. WorkbenchOverlay 的纪律

#### 1. `PanelRoot` 不准再被 runtime 改尺寸

这条必须收死。

`PanelRoot` 的位置、宽高、左右栏比例，默认全部以 prefab 为真源。

你不要再在 Phase 2 里改：

- `panelRect.sizeDelta`
- 整个 panel 的目标高度
- 左右栏总宽度分配

#### 2. 右栏只允许做“内部纵向流动”

允许做的是：

- 说明文本变高
- 材料标题 / 材料区 / 数量区 / 提示语 / 进度区 依次下推或上移
- 空间不够时让 `materials viewport` 去滚

不允许做的是：

- 为了容纳内容，重新把整个 panel 拉高
- 重新算整个右栏几何
- 让按钮区、材料区、数量区位置脱离 prefab 的版式

#### 3. 左侧列表区不要再被“兼容修复”干扰视觉版式

如果列表项内容需要动态生成：

- 可以生成内容
- 可以在 viewport 内滚

但不要顺手去改：

- 左列外框位置
- 左列宽度
- 左列标题位置

### C. 通用纪律

#### 1. 动态布局只处理“内容区”

只允许改这些：

- 文本块高度
- 内容区子项的 Y 顺序
- viewport 内部内容高度
- 滚动内容高度

#### 2. 不允许再改“壳体区”

不允许再改这些：

- 卡片外壳
- 面板外壳
- 左右栏大框
- 顶层视觉容器
- 用户已经手调过的整体几何关系
