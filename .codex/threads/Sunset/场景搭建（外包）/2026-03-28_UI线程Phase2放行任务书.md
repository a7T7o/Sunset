## 五、本轮完成定义

这轮做完后，必须同时满足下面 6 条，才算第二步完成：

### 1. 自适应过线

当文本、描述、材料条目变多时：

- 下方内容会正确让位
- 不发生致命重叠
- 必要时转入滚动，而不是把 UI 撑坏

### 2. 撕页过线

`PromptOverlay` 的翻页必须具备：

- 前页被掀起 / 撕起的视觉语义
- 掀起时下面看到的是下一页，不是场景背景
- 前后页关系清楚，不是简单缩放假动作

### 3. 工作台可见性与滚动过线

至少包括：

- 左侧 `RecipeColumn` 内容能看见
- `Viewport / Content / ScrollRect / Mask` 真的工作
- 右侧材料区在内容多时能正确显示、让位或滚动

### 4. 制作按钮 / 进度条状态机过线

至少包括：

- 空闲态
- 可制作态
- 不可制作态
- 制作中悬停态
- 制作中非悬停态

并且：

- 进度显示要和制作状态一致
- 不能还是一个死按钮盖在那

### 5. 离台小进度过线

玩家离开工作台后：

- 仍有一个小型悬浮进度提示
- 有图标
- 有进度反馈
- 不会因为离开 UI 就完全丢失制作状态感知

### 6. 固定锚定过线

除了用户已经接受的那类切边 / 上下转换外：

- UI 和工作台的相对位置必须稳
- 不应再出现“移动一下就漂”的相对运动感

## 六、这轮建议的实现顺序

这轮虽然是一整步，但为了减少返工，我建议你内部按下面顺序落：

1. **先修壳体中的动态布局与滚动链**
   - 自适应
   - Recipe / Materials 的 viewport / content / scroll
2. **再修 Prompt 的撕页**
3. **再修 Workbench 的按钮 / 进度条状态机**

5. **最后收固定锚定**

原因很简单：

- 先把内容区稳定住，后面状态和定位才不容易反复打架

但这只是你的内部顺序。
