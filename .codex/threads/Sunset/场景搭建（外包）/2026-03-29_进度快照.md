## 六、当前可执行容量

### 最稳容量

- `1 条硬主线 + 1 条轻支撑线`

### 可接受上限

- 如果支撑线只是：
  - 文档
  - prompt
  - memory
  - 验收口径整理
- 那么可接受上限是：
  - `1 条硬主线 + 2 条轻支撑线`

### 当前不建议

- 不建议同一轮里同时推进：
  - `2 条硬支线`
  - 或 `1 主线 + 2 条重支线`

原因很简单：

- Sunset 当前有 Unity live
- 有 shared root
- 有场景锁
- 有用户观感验收

一旦并行太多，最容易先坏掉的是：

- 主线焦点
- 验收口径
- 恢复点清晰度

## 七、当前恢复动作

如果后续从这份快照恢复，默认顺序固定为：

1. 先看 UI 线程有没有交新回执
2. 如果已交回执，只审：
   - `Workbench 整面是否过线`
   - `Prompt 是否回正`
3. 如果还没交回执，不新开重支线
4. 只有当前主线过线后，才切下一刀：
   - 最近交互唯一提示 / 唯一 `E` 键仲裁 / NPC 优先级 / 气泡速度
