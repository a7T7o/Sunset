## 4. 接手第一轮建议动作顺序

### 建议顺序 A：如果先收 `013`

1. 做 shared root / Unity live preflight
2. 确认当前是否有并发 `NavValidation` 或其他 live 占用
3. 只重跑：
   - sapling-only
   - 必要时再补 1 个最小 sanity
4. 收口 runner 稳定性，不扩到 placeable 主链

### 建议顺序 B：如果先做新增 6 条需求

1. 从 chest / placeable 体验硬伤开始
2. 第一刀优先候选：
   - 右键箱子到不了可开启位置
   - 打开箱子不应退出放置模式
3. 在此之后再处理：
   - 幽灵占位
   - 工具损坏 / 精力耗尽反馈
   - 树第 2 阶段幽灵僵尸

## 5. 接手第一轮最容易踩的坑

1. 重新把 `012` 当成“还没恢复”
2. 把 `013` 的 runner 尾差误判成业务主链事故
3. 把 `GameInputManager.cs` 当 own 文件大改
4. 看到 `Props/Farm` 就以为 parent/container 规范已经最终敲定
5. 忘记新增 6 条需求已经是总入口的一部分
6. 一上来就大面重刷 second-blade / placeable 主链

## 6. 不建议优先读取的材料

以下材料不是没价值，而是不应排在第一波：

1. 旧的 `V1交接文档`
   - 可以后读做对照，但当前以这套 `V2交接文档` 为准
2. 污染分支 / cleanroom 取证材料
   - 除非要追历史来源，否则不必先读
3. 与农田无关的治理线程材料

- 我接手的不是“placeable 事故修复线程”
