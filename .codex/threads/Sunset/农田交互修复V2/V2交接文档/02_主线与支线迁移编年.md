# 主线与支线迁移编年

## 阶段 1：农田交互正确性修正期

### 当时目标

- 修掉农田工具、预览、队列、导航取消、手持保护这些会直接造成“不能玩”的错误。

### 当时动作

- 建立并推进 `1.0.2纠正001`
- 把用户对“正在交互”和“只是手持”的边界反复澄清进文档和代码

### 结果

- 农田工具链的基础交互边界被重新钉住；
- 但 placeable、箱子、Toolbar、Tooltip 等更高层问题还没被统一进去。

### 对后续的影响

- 这阶段形成了一个很关键的历史结论：
  - 不要把“只是手持”误判成“正在交互”。

## 阶段 2：污染分支 / cleanroom / continuation carrier 期

### 当时目标

- 从污染分支和 mixed 提交里把真正的 farm-only 内容剥出来，避免继续沿错误分支开发。

### 当时动作

- 对 `07ffe199 / 11e0b7b4 / 18f3a9e1` 做 farm / NPC / mixed 归类
- 以 `b9b6ac48` 为 cleanroom 推荐起点
- 最终确认 continuation carrier 不再沿污染分支开发

### 结果

- `codex/farm-1.0.2-correct001` 被降级成事故取证现场
- 有效 farm 成果被收束到 cleanroom continuation branch，再逐步迁回 `main`

### 对后续的影响

- 形成第二个关键历史结论：
  - 不要再把污染分支或旧 worktree 当成正常开发来源。

## 阶段 3：`1.0.3基础UI与交互统一改进`

### 当时目标

- 把 Tooltip、实例态、Toolbar 第一行同源、箱子 UI runtime 接线收起来，解决“信息出口”和“基础 UI 一致性”问题。

### 当时动作

- 补 Tooltip 入口
- 补 `InventoryItem` 实例态在掉落 / 拾取 / 背包中的传递
- 补食物 / 药水状态链
- 把背包第一行与 Toolbar 同源同步做实

### 结果

- `1.0.3` 成为了后续 `1.0.4` 的 UI 与实例态基线；
- 但 placeable 成功事务、工具统一消耗链、箱子单真源等系统问题仍然暴露。

### 对后续的影响

- 形成第三个关键历史结论：
  - `1.0.3` 之后，问题已经不能再当局部 UI 尾项修。

## 阶段 4：`1.0.4` docs-first 升级为 `全局交互V3`

### 当时目标

- 停止“边做边猜”，先把整条线升级成全局交互治理问题。

### 当时动作

- 创建 `001最后通牒.md`
- 生成两份主导文档：
  - `全面理解需求与分析.md`
  - `全面设计与详细任务执行清单.md`
- 把线程语义升级为：
  - `全局交互V3（原：农田交互修复V2）`

### 结果

- placeable、箱子、Toolbar、工具参数链被正式放到同一张桌子上讨论；
- 之后的所有实现都应以这两份文档为入口。

### 对后续的影响

- 形成第四个关键历史结论：
  - 后续任何实现都不能再脱离总入口文档单飞。

## 阶段 5：第一刀，箱子递归爆栈阻断清除

### 当时目标

- 先清掉最阻断整条交互验收的硬故障：
  - `ChestInventory / ChestInventoryV2 / ChestController` 的双向递归同步。

### 当时动作

- 把 `ChestInventoryV2` 定为 authoritative runtime/save source
- legacy `ChestInventory` 降为兼容 mirror
- 静默 bridge API + 一次性补发事件

### 结果

- 箱子 `StackOverflowException` 被压掉；
- 也为后面继续测箱子放置、箱子 UI 和 Sort 奠定了基础。

### 对后续的影响

- 形成第五个关键历史结论：
  - 箱子不能继续双真源互踢。

## 阶段 6：第二刀，placeable 主链与 second-blade live

### 当时目标

### 当时动作

  - 单轮 live 通过，不等于用户感知上的整体主链就真的恢复了。

## 当前最终迁移结论

到本次交接为止，这条线已经完成两次最重要的迁移：

1. 从“农田局部修补”迁移为“全局交互V3 语义”
2. 从“placeable 事故恢复期”迁移为“已恢复基线 + runner 尾差 + 新增需求待实现”的阶段
