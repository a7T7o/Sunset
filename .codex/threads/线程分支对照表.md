# Sunset 线程分支对照表

## 适用范围
- 本表只处理 `Sunset` 仓库内的长期线程映射。
- 本表解决的是“哪个线程默认进入哪个工作目录、对应哪条默认分支”。
- 本表不负责决定“什么时候把功能合并回 `main`”；那属于后续单独的 merge 决策。

## 核心原则
- 不依赖 UI 的历史分支提示判断真实现场。
- 真正有效的判断顺序固定为：
  1. 线程业务上属于哪条主线
  2. 当前默认工作目录是否正确
  3. `git branch --show-current` 是否正确
- `worktree` 解决的是“线程看哪份文件”。
- `merge` 解决的是“功能什么时候回主线联调”。
