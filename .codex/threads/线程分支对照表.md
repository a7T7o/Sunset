# Sunset 线程分支对照表

## 适用范围
- 本表只处理 `Sunset` 仓库内的长期线程入口映射。
- 本表解决的是“线程默认从哪里进入现场、先站在哪个基线看问题、真实任务实现时应怎样切换到提交位姿”。
- 本表不再把历史 worktree 当作默认现场，也不直接决定某条业务功能何时 merge 回 `main`。

## 核心原则
- 默认入口统一为 `D:\Unity\Unity_learning\Sunset @ main`。
- 真实任务改动不直接在 `main` 收尾；应从默认入口通过 `ensure-branch` 切到 `codex/xxx` 分支。
- `worktree` 现在只保留为高风险隔离、故障修复、特殊实验的例外工具链。
- 不依赖 UI 历史分支提示判断真实现场；真实判断顺序固定为：
  1. 当前线程是否先站在主项目根目录
  2. 当前 `git branch --show-current` 是否符合入口语义
  3. 本轮改动是治理类还是业务实现类
