- `.kiro/specs/农田系统/`
- `.kiro/specs/900_开篇/spring-day1-implementation/`

### 2.2 当前理解

- `spring-day1-implementation` 是当前剧情 / 对话验证主线之一，但当前更准确的阻塞描述是：验证场景的 UI 可见性 / 布局问题优先于纯脚本问题。
- `Codex迁移与规划` 当前新增了一条治理子主线：持续建设 `.kiro/about/` 三份项目认知文档。

> 以上为第一轮文档结论，后续仍需继续用代码、场景和最新 memory 交叉核对。

### 2.3 各主线当前恢复点

- `农田系统`
  - `10.2.1补丁001` 已整包实现，并用 Unity 自带 Roslyn 做过独立编译验证。
  - 当前恢复点是：按任务清单做编辑器内手动回归，重点看箱子、树苗、种子、`Hoe/WateringCan` 行为。
- `spring-day1-implementation`
  - 当前恢复点是：在 `DialogueValidation.unity` 里继续做 UI 可见性 / 布局 / 中文字体引用验收，但要先把它当作场景阻塞来理解，而不是直接假设脚本失效。
  - 当前已知原有配置证据是：`DialogueCanvas` 的 `RectTransform.m_LocalScale` 为 `0,0,0`，`DialogueUI` 组件 5 个关键引用为空。
  - 同时要记住空引用不是唯一问题，因为场景里仍有 `DialoguePanel`、`SpeakerNameText`、`DialogueText`、`ContinueButton`、`PortraitImage`、`Background` 这些命名对象，而 `DialogueUI.cs` 在验证场景里有按名字自动补线逻辑。
  - 重点看 `DialogueChinese SDF.asset` 的 Inspector 实际引用、Canvas 可见性和中文显示结果，不要误修成“只补 Inspector 引用”。
- `Codex迁移与规划`
  - 当前恢复点是：继续扩大 `.kiro/about/` 的覆盖面，让后来者只看这三份文档就能接手。

### 2.4 当前不是主线、但接手风险很高的系统

- `存档 / 箱子 / 放置`
  - 这三条线现在已经不是各自独立：`ChestController` 同时受 `StorageData`、`BoxPanelUI`、`SaveManager`、`DynamicObjectFactory`、`PlacementManager` 影响。
- `UI / 背包`
  - 当前 UI 结构不是旧设计里的静态小面板组合，而是 `InventoryPanelUI + PackagePanelTabsUI + InventoryInteractionManager + BoxPanelUI` 的协同系统，且经历过多轮回滚与重构。
- `制作台 / 配方`
  - 风险已经从“代码是否存在”转成“资产 / 场景接入是否真实落地”：`MasterItemDatabase.asset` 的 `allRecipes` 仍为空槽，当前也没找到清晰的 `RecipeData` / `WorkstationData` 资产落点或 prefab 对 `CraftingService` 的直接挂载。
  - 这意味着它不一定是“坏掉了”，但也绝不能被写成“已经完整闭环”。

## 3. 当前已知高风险漂移点

### 3.1 路径漂移

- 历史文档常写 `Assets/Scripts/`
- 当前真实仓库主要脚本目录是 `Assets/YYY_Scripts/`
- 但旧路径没有完全消失：`Assets/Scripts/Utils/ConvexHullCalculator.cs` 仍然存在

### 3.2 进度漂移

- `Docx/` 与 `.kiro/steering/archive/*.md` 中的进度描述只能视作阶段快照
- 当前真实状态必须以各工作区 `memory.md` 为准

### 3.3 工作区索引漂移

- `.kiro/specs/README.md` 仍保留大量旧英文目录映射
- 当前真实工作区已经以中文目录为主，必须用实际目录和 live `memory.md` 校准

### 3.4 存档体系漂移

- 文档和重构记忆都把 `PrefabDatabase` 视为主方案
- 但当前代码里 `PrefabRegistry` 仍保留为废弃兼容层，`SaveManager` / `DynamicObjectFactory` 仍有回退分支

### 3.5 工具链漂移

- `Packages/manifest.json` 已包含 `com.coplaydev.unity-mcp`
- 但“包已经在 manifest 中”不等于“Unity MCP 现在就稳定可用”；真实可用性仍取决于 Unity 侧 setup、依赖和连接状态

### 3.6 放置版本命名漂移

- 历史文档与日志仍常写 `PlacementManagerV3`
- 当前真实代码文件名已经是 `PlacementManager`、`PlacementPreview`、`PlacementValidator`
- 接手时不能把“V3”误认成仓库里另有一套并行新文件

### 3.7 背包容量漂移

- 旧 `Docx` 和部分早期交接仍会写主背包 `20` 格、Hotbar `8` 格
- 当前真实代码已经是背包 `36` 格，第一行 `12` 格和 Hotbar 同源
- 箱子 UI 的 `Down` 区当前显示完整背包 `0..35`

### 3.8 制作台状态漂移

- 历史设计文档会让人感觉“制作台系统已经完整闭环”
- 当前更准确的说法应是：制作台代码和 UI 骨架已在，但资产与场景接入证据仍明显不足
- `MasterItemDatabase.asset` 的 `allRecipes` 当前仍是 3 个 `{fileID: 0}` 空槽
- 当前未找到 `.asset` 对 `RecipeData.cs` / `WorkstationData.cs` 的脚本 GUID 引用，也未找到 prefab 对 `CraftingService.cs` 的直接挂载

### 3.9 对话验证阻塞漂移

- 旧描述很容易把 Day1 当前问题简化成“缺字体”或“只要补 Inspector 引用”
- 当前更准确的现场描述是：`DialogueCanvas` scale 为 `0,0,0`，`DialogueUI` 关键引用为空，但脚本又提供按对象名自动补线
- 因此这里不能误判成单纯脚本阻塞，也不能误修成只补 Inspector；更合理的口径是“视觉 / 布局阻塞优先，空引用需要和场景可见性一起看”

## 4. 当前接手顺序

### 4.1 如果只是想快速进入项目

1. 读 `00_项目总览与阅读地图.md`
2. 读 `.kiro/steering/archive/*.md`
3. 读对应活跃工作区 `memory.md`

### 4.2 如果要接代码任务

1. 读对应工作区 `memory.md`
2. 读对应历史完整文档
3. 读 `01_系统架构与代码全景.md`
4. 再进入代码和资源

### 4.3 如果你要直接接当前三条活跃主线

1. 接农田：先读 `.kiro/specs/农田系统/memory.md`，再看 `FarmTileManager.cs`、`PlacementManager.cs`、`PlacementValidator.cs`
2. 接剧情验证：先读 `.kiro/specs/900_开篇/spring-day1-implementation/memory.md`，再看 `DialogueValidation.unity` 与 `Story/` 目录，重点核对 `DialogueCanvas`、`DialogueUI`、`DialogueManagerRoot`
3. 接文档治理：先读 `Codex迁移与规划/tasks.md`，再继续维护 `.kiro/about/`

### 4.4 如果要继续维护这套 about 文档

1. 先对照 `tasks.md`
2. 先修正文档旧结论，再写新增结论
3. 每轮完成后同步更新工作区 memory

### 4.5 如果你接的是高耦合系统

1. 接 `存档 / 动态重建`
2. 先读 `存档系统/memory.md`
3. 再看 `SaveManager.cs`、`DynamicObjectFactory.cs`、`PrefabDatabase.asset`

1. 接 `箱子 / 背包 / 放置`
2. 先读 `箱子系统/memory.md`、`UI系统/memory.md`、`物品放置系统/memory.md`
3. 再看 `ChestController.cs`、`BoxPanelUI.cs`、`PackagePanelTabsUI.cs`、`PlacementManager.cs`，并把 `Primary.unity` 与 `PackagePanel.prefab` 一起核对

1. 接 `制作台 / 配方`
2. 先读 `制作台系统/memory.md`
3. 再确认 `CraftingService.cs`、`CraftingPanel.cs`、`RecipeData.cs` 是否能和 `MasterItemDatabase.asset`、实际配方资产目录、工作台挂载现场对得上
