## 2. 当前语义冻结

下面这些当前视为固定语义，不在导航线里再争论：

### 2.1 Day1 own

1. `opening / dinner` 的 staged contract
2. `001/002` 的 `house lead / Primary` 特殊剧情
3. 哪些 NPC 当前在剧情态
4. 何时 acquire / release
5. `19:30 / 20:00 / 21:00 / 26:00` 的时序语义
6. staged timeout 后的显式 snap
7. 对白期间冻结

### 2.2 导航 own

1. 所有真实 movement execution 的质量
2. 去 anchor / home / staged target 的路径执行
3. stuck / blocked / fail / abort / replan
4. static navigation / body clearance / detour / local avoidance
5. release 后如何稳定接回 free-roam

### 2.3 NPC body / facade own

1. resident state 自己如何切换
2. release 后进入哪种 resident state
3. 对外暴露哪层 facade，哪层低级接口必须收起来

### 2.4 当前新边界

当前采用这条边界：

`只要 NPC 发生真实位移，原则上都应优先走统一导航执行合同。`

明确例外只有 3 类：

1. staged timeout 后的显式 snap
2. load / repair 场景下的显式 snap
3. authored 摆位失败后的强制对位兜底

## 3. 当前总判断

### 3.1 当前三主根

用户压出来的 3 个主根成立：

1. `NPC 漫游走的都是坏点`
2. `NPC 的静态导航计算还是垃圾`
3. `NPC 阻塞卡死了不会自救`

### 3.2 当前第一主问题

当前导航第一主问题不是某个参数，而是：

`执行合同仍然分裂，导致坏点、clearance 和恢复逻辑始终被多套旁路口径重新污染。`

### 3.3 当前为什么还没过线

1. target acceptance 还不够硬
2. body clearance 还没推进到路径成立层
3. recovery 还是止血，不是完整脱困闭环
4. 导航 own 内部 still 有多套 movement contract
5. 外部线程 still 能通过低级 public API 绕过正确边界

## 4. 当前 movement contract 地图

## 4.1 当前至少并存的 5 套合同

### A. `autonomous roam contract`

含义：

- NPC 自己选点、自己建路、自己执行、自己恢复

当前入口：

- `StartRoam()`
- `TryBeginMove()`

### B. `resident scripted move contract`

含义：

- 外部 owner 指定目标点，但执行仍借导航

当前入口：

- `AcquireResidentScriptedControl(...)`
- `DriveResidentScriptedMoveTo(...)`

### C. `formal navigation debug / home-return contract`

含义：

- “去 home / 去 anchor / 去正式目标”的 point-to-point travel

当前入口：

- `DebugMoveTo(...)` + `debugMoveUsesFormalNavigationContract`

### D. `plain debug / manual fallback contract`

含义：

- 调试态、兜底态、临时性指令移动

当前入口：

- `DebugMoveTo(...)` 的非 formal 分支

### E. `Day1 staged playback 直推链`

含义：

- authored cue path 按导演路径直接推 transform

当前入口：

- `SpringDay1DirectorStaging` 内部 playback 驱动

### 4.2 当前最危险的分裂点

1. `B / C / D` 本质都在做 point-to-point travel，却没有共用单一真相
2. `A` 与 `B / C / D` 共享了不少底层状态，但目标来源和失败处理并不统一
3. `E` 完全绕开导航执行层，导致“剧情移动”和“导航移动”长期两张皮

---

## 5. 当前最值钱的统一方向

### 5.1 当前不做的统一

当前不直接做：

1. 不先把 `Day1 staged playback` 整体吞进 free-roam 合同
2. 不先把所有 movement 全塞进一个大状态机
3. 不先重写 Day1 phase / beat / cue 语义

### 5.2 当前先做的统一

当前第一优先统一的是：

`把导航 own 内部的 point-to-point movement contract 统一成一套正式 travel contract。`

先合并这 3 套：

1. `resident scripted move`
2. `formal navigation debug / home-return`
3. `plain debug / manual fallback`

暂时不强并：

1. `autonomous roam`
2. `Day1 staged playback`

### 5.3 为什么先统一这 3 套

因为如果这 3 套 point-to-point travel 还继续分裂：

1. return-home 会继续带着 debug / fallback 味
2. release 后 movement quality 仍会被旁路执行口径重新污染
3. 后面再做坏点和 clearance，仍会被旧入口绕开

## 7. 详细执行清单

## Package A｜统一 point-to-point navigation travel contract

### 目标

把“去一个明确目标点”的执行统一成一套正式导航 travel contract。

### 包含范围

1. `resident scripted move`
2. `formal home/anchor return`
3. `plain debug/manual fallback`

### 当前要做

1. 列出这 3 套当前所有入口
2. 列出它们当前的共有能力：
   - build path
   - move step
   - blocked / stuck
   - arrival
   - abort
3. 列出它们当前不一致的地方：
   - body clearance
   - arrival settle
   - restart behavior
   - failure handling
4. 设计一个统一的 point-to-point travel contract
5. 让这 3 套入口都转成这一个 contract

### 完成标准

1. return-home 不再走半 debug 半 fallback 口径
2. `resident scripted move` 和 `formal home-return` 不再各自带一套失败处理
3. `plain debug` 不再是独立执行真相，只是统一 contract 的 debug 入口

### 当前不做

1. 不在这一包吞 `autonomous roam`
2. 不在这一包吞 `Day1 staged playback`

---

## Package B｜destination hard contract

### 目标

把“坏点”从评分问题升级成硬合同问题。

### 覆盖对象

1. autonomous roam 目标点
2. detour 点
3. retarget 点
4. rebuild 后的新目标点
5. final destination

### 当前要做

1. 定义统一 acceptance 条件：
   - body-shape overlap 不通过即拒绝
   - 静态边界最小距离不通过即拒绝
   - 最小可通行走廊宽度不通过即拒绝
2. 去掉“先抽坏点，再靠大纠偏拉回”的残留口径
3. 不再接受贴墙、贴树、贴桶、贴围栏的勉强点

### 完成标准

1. 坏点不是“候选里挑好一点”，而是直接进不来
2. detour / retarget 不再比主目标合同更松

---

## Package C｜path-level body clearance

### 目标

让身体宽度从“最后一步别撞”推进到“整条路径从成立开始就别贴边”。

### 当前要做

1. path build acceptance 吃 clearance
2. waypoint acceptance 吃 clearance
3. detour acceptance 吃 clearance
4. 不再只靠 `next-step constraint` 和 static steering 做后处理

### 完成标准

1. 贴墙路径不再先成立后补救
2. 树根 / 桶边 / 围栏边的危险路径在成立层就被拒绝

---

## Package D｜完整脱困闭环

### 目标

把 recovery 从“止血”升级成“稳定脱困”。

### 当前要分成两条

1. `静态卡死`
2. `动态互卡`

### 静态卡死至少要有

1. 短后退
2. 侧移采样
3. 坏边 / 坏点临时黑名单
4. 失败后重规划

### 动态互卡至少要有

1. 限时等待
2. 短让路
3. 失败后 reroute

### 完成标准

1. 不再原地磨、原地抽、反复修同一小段
2. `Stuck / SharedAvoidance / MoveCommandNoProgress` 明显下降

---

## Package E｜NPC facade 与外部 call-site 收口

### 目标

不再让外部线程裸调低级 locomotion 写口。

### 当前要做

1. 明确哪些 API 只能 internal/runtime 用
2. 明确哪些 API 升为 facade
3. 让 Day1 不再直接深碰：
   - `StartRoam / StopRoam / RestartRoamFromCurrentContext`
   - `SetHomeAnchor / ApplyProfile / RefreshRoamCenterFromCurrentContext`
   - `SetFacingDirection / SetExternalVelocity / SetExternalFacingDirection`
   - resident scripted control 原语

### 完成标准

1. Day1 只发语义命令，不再拼导航/NPC body 的低级状态机
2. 导航执行层的统一合同不会再被 call-site 绕开
