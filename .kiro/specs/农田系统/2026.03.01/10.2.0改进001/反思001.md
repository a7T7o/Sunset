  第二章：核心需求澄清与规则定义

  2.1 V键放置模式的本质

  核心定义：V键是一个"建设/破坏"模式的隔离开关，而非"放置范围"或"UI面板"。

  两种状态的职责划分：

  ┌───────────────────────┬────────────────────────┬──────────┬──────────────────────┬────────────────────────────┬───────────────────────────┐
  │         状态          │          锄头          │   水壶   │      可放置物品      │ 其他工具（斧头/镐子/镰刀） │ 其他交互（NPC/箱子/拾取） │
  ├───────────────────────┼────────────────────────┼──────────┼──────────────────────┼────────────────────────────┼───────────────────────────┤
  │ 放置模式关闭（默认）  │ 除草、挖树苗、挖农作物 │ 无功能   │ 不显示预览、不能放置 │ 正常使用（砍树/挖矿/割草） │ 正常交互                  │
  ├───────────────────────┼────────────────────────┼──────────┼──────────────────────┼────────────────────────────┼───────────────────────────┤
  │ 放置模式开启（V键后） │ 只能耕地               │ 只能浇水 │ 显示预览、可以放置   │ 正常使用（不受影响）       │ 正常交互                  │
  └───────────────────────┴────────────────────────┴──────────┴──────────────────────┴────────────────────────────┴───────────────────────────┘

  关键澄清：
  1. V键不影响移动：WASD移动与V键状态完全独立，移动不会退出放置模式
  2. V键不影响UI：打开背包、地图等UI不会退出放置模式
  3. V键不影响其他工具：斧头、镐子、镰刀等工具的使用不受V键影响
  4. V键只隔离"建设类"交互：耕地、浇水、放置物品这三类操作被V键隔离，其他所有交互（除草、砍树、挖矿、NPC对话、箱子交互、拾取物品）都不受影响

  2.2 锄头的双重职责

  默认状态（V键关闭）：
  - 左键点击 → 除草（清除杂草物体）
  - 左键点击树苗 → 挖掉树苗
  - 左键点击农作物 → 挖掉农作物
  - 不能耕地

  放置模式（V键开启）：
  - 左键点击 → 耕地（创建耕地Tile）
  - 不能除草、不能挖树苗、不能挖农作物

  设计意图：彻底隔离"破坏"和"建设"，避免误操作（站在树苗边上耕地时把树苗挖掉）。

  2.3 ESC/WASD的退出逻辑（需要修正）

  用户明确要求：
  - ESC：清除队列预览，但不退出放置模式
  - WASD：清除队列预览，但不退出放置模式

  当前实现的错误：
  - 代码中可能存在"WASD移动时退出放置模式"的逻辑，这是错误的
  - V键状态应该是持久的，只有再次按V键才能切换

  第四章：Bug根因总结

  Bug 1：可放置物品按V键不显示预览

  根因：
  1. GameInputManager切换V键状态时，没有主动触发预览刷新
  2. PlacementManager没有订阅V键状态变化事件，只能被动轮询
  3. PlacementManager的状态机没有"从Idle进入Preview"的触发逻辑（当V键从关闭变为开启时）

  修复方案：
  1. 在GameInputManager中添加V键状态变化事件：OnPlacementModeChanged
  2. PlacementManager订阅该事件，当V键开启时，检查手持物品并进入Preview状态
  3. 或者在GameInputManager的V键处理中，主动调用UpdatePreviews()强制刷新

  Bug 2：队列与工具切换的冲突

  根因：
  1. 切换工具时，OnHotbarSelectionChanged事件触发了队列清空
  2. 队列清空时，没有正确重置PlacementManager的状态机
  3. 导致PlacementManager卡在中间状态，拒绝新的点击
  4. 预览对象的透明度没有被正确恢复

  修复方案：
  1. 切换工具时，不应该清空队列（除非新工具与队列中的操作不兼容）
  2. 如果必须清空队列，必须同时重置PlacementManager的状态机为Idle
  3. 清空队列时，必须清理所有预览对象（包括恢复透明度）
  4. 或者：将队列系统与工具类型绑定，切换工具时只清空"不兼容"的队列

  Bug 3：ESC/WASD退出放置模式（如果存在）

  根因：
  - 代码中可能有"WASD移动时退出放置模式"的逻辑
  - 这与用户需求不符

  修复方案：
  - 移除所有"WASD移动时退出放置模式"的代码
  - ESC和WASD只清空队列预览，不改变V键状态

  public static event System.Action<bool> OnPlacementModeChanged;

  private void OnHotbarSelectionChanged(int newIndex)
  {
      // 获取新工具类型
      var newSlot = inventory.GetSlot(newIndex);
      ToolType? newToolType = GetToolType(newSlot);

      // 只有当新工具与队列中的操作不兼容时，才清空队列
      if (_farmActionQueue.Count > 0)

          var firstAction = _farmActionQueue.Peek();
          bool isCompatible = IsToolCompatibleWithAction(newToolType, firstAction.type);

          if (!isCompatible)
          {
              // 清空队列，并重置所有状态
              ClearActionQueue();
              ResetAllStates();
          }
      }

      // V键状态不变，只是切换工具
  }

  private bool IsToolCompatibleWithAction(ToolType? toolType, FarmActionType actionType)
  {
      if (toolType == null) return false;

      switch (actionType)
      {
          case FarmActionType.Till:
              return toolType == ToolType.Hoe;
          case FarmActionType.Water:
              return toolType == ToolType.WateringCan;
          case FarmActionType.PlantSeed:
              // 种子不是工具，需要特殊判断
              return false;
          default:
              return false;
      }
  }

  private void ResetAllStates()

      // 重置PlacementManager状态
      if (PlacementManager.Instance != null)

          PlacementManager.Instance.ForceResetState();

      HideAllPreviews();

      // 重置导航状态
      _farmNavState = FarmNavState.Idle;
      _isExecutingFarming = false;
  }

  ## V键放置模式规范（2026-03-03新增）

  ### 核心定义

  V键是"建设/破坏"模式的隔离开关，用于彻底隔离耕地工具和放置物品的交互，避免误操作。

  ### 两种状态的职责划分

  | 状态 | 锄头 | 水壶 | 可放置物品 | 其他工具 | 其他交互 |
  |------|------|------|-----------|---------|---------|
  | **放置模式关闭（默认）** | 除草、挖树苗、挖农作物 | 无功能 | 不显示预览、不能放置 | 正常使用 | 正常交互 |
  | **放置模式开启（V键后）** | 只能耕地 | 只能浇水 | 显示预览、可以放置 | 正常使用 | 正常交互 |

  ### V键不影响的操作

  - ✅ WASD移动：移动不会退出放置模式
  - ✅ UI交互：打开背包、地图等UI不会退出放置模式
  - ✅ 其他工具：斧头、镐子、镰刀等工具的使用不受V键影响
  - ✅ 其他交互：NPC对话、箱子交互、拾取物品不受V键影响

  ### V键影响的操作

  - 🔴 耕地：只有在放置模式下才能耕地
  - 🔴 浇水：只有在放置模式下才能浇水
  - 🔴 放置物品：只有在放置模式下才能放置树苗、箱子、种子等

  ### ESC/WASD的行为

  - ESC：清除队列预览，**但不退出放置模式**
  - WASD：清除队列预览，**但不退出放置模式**
  - 只有再次按V键才能切换放置模式状态

  ### 工具切换行为

  - 切换工具不影响放置模式状态
  - 例如：在放置模式下从锄头切换到树苗，仍然保持放置模式
  - 切换工具时，只清理"与新工具不兼容"的队列，不清空所有队列

  ### 实现要点

  1. **事件化V键状态**：使用`OnPlacementModeChanged`事件通知各模块
  2. **预览刷新**：V键切换时，主动触发预览刷新
  3. **状态同步**：PlacementManager订阅V键事件，立即响应状态变化
  4. **队列清理**：工具切换时，只清理不兼容的队列，不清空所有队列
  5. **透明度恢复**：清理预览时，必须恢复预览对象的透明度

  8.1 Bug 1：可放置物品按V键不显示预览

  测试步骤：
  1. 手持种子，此时不显示预览（V键默认关闭）
  2. 按V键，立即显示种子预览
  3. 再按V键，立即隐藏种子预览
  4. 重复步骤2-3，确认预览响应及时

  预期结果：
  - ✅ 按V键后，预览立即显示，无需切换物品
  - ✅ 再按V键后，预览立即隐藏

  3. 切换到种子
  4. 观察队列是否被清空
  5. 尝试点击播种，观察是否能正常交互
  6. 观察作物预览是否有透明度异常

  预期结果：
  - ✅ 切换到种子后，耕地队列被清空（因为种子与耕地不兼容）
  - ✅ PlacementManager状态正确重置为Idle
  - ✅ 点击播种能正常交互
  - ✅ 作物预览
