# 10.2.2补丁002 需求文档

版本: v1.0  
日期: 2026-03-10  
状态: 待用户审核

## 1. 背景

本补丁来自用户对 `10.2.1补丁001` 的现场纠偏。用户明确指出：上一轮把“农田施工链的障碍规则”和“普通 placeable 的放置预览规则”混在了一起，也把 `SeedData`、`SaplingData`、普通 `PlaceableItemData` 三条本来就不同的验证链误合并了。

用户这轮给出的核心纠偏包括：

> 虽然在耕地上左键箱子确实不可以放置，但是你只做了基础逻辑，没有和预览进行联动适配，不可交互应该是需要严格处理的，走红色判断的需要，是哪些格子不可放置才标红。

> 我说不能放置的是除了种子的其他内容。

> 种子播种就是有耕地就行啊，不会被其他影响啊，因为耕地上不可以有任何除了种子的东西放置。

> 树苗和种子在可放置物品里面是比较特殊的，其他的比如箱子就是比较规范的，放置检测就是格子检测，很原始。

## 2. 本轮目标

本轮 `10.2.2补丁002` 的目标不是直接改代码，而是先重新梳理项目真实设计与当前实现，形成一套新的补丁文档，作为后续实现的唯一依据。

本轮文档必须回答清楚四件事：

1. 普通 placeable 的预览与放置到底应该如何联动。
2. 为什么 `SeedData` 不能被套进普通 placeable 的耕地封禁规则。
3. 为什么 `SaplingData` 也不能被当作普通 placeable 处理。
4. `FarmToolPreview`、`PlacementPreview`、`PlacementValidator` 三者各自应该承担什么职责。

## 3. 正式需求

### 3.1 普通 placeable 的耕地禁放规则

普通 placeable 指本项目中走 `PlaceableItemData` 的常规放置物，例如箱子、工作台、展示台、简单事件物。

正式要求：

1. 普通 placeable 不允许落在已耕作的农田格上。
2. 这个禁放规则必须反映到 `PlacementPreview` 的红/绿格子预览，而不是只在执行时失败。
3. 红色判定必须是“哪些占用格不可放置，就把哪些格标红”，不能把整组格子无差别全部染红。
4. 普通 placeable 仍然使用标准放置预览体系：
   - 方框格子预览；
   - 物品半透明预览；
   - 不切换到 `FarmToolPreview` 的耕地 ghost tile 体系。

### 3.2 种子的特殊规则

`SeedData` 虽然在当前代码中 `isPlaceable = true`，但它不是普通 placeable。

正式要求：

1. 种子继续走 `PlacementManager + PlacementPreview` 这条标准放置预览链。
2. 种子的预览样式继续是“方框 + 作物第一阶段半透明 sprite”，不回退到旧的独立种子预览系统。
3. 种子的有效性判断只围绕“该格是否为可种植耕地”展开，核心依据应是农田格数据本身，而不是普通 placeable 的耕地封禁规则。
4. 种子不能因为“其他普通 placeable 不可放在耕地上”而被一并挡掉。

### 3.3 树苗的特殊规则

`SaplingData` 也是特殊放置物，不能简单等同于普通 placeable。

正式要求：

1. 树苗继续走 `ValidateSaplingPlacement()` 这条专用验证链。
2. 树苗是否可放，应继续受冬季、耕地、水域、树距、树苗成长边距等专用规则约束。
3. 树苗不能被重写成“和箱子一样只做原始格子检测”的普通放置物。
4. 树苗与普通 placeable 的共同点仅限于都使用 `PlacementPreview`，但验证来源不同。

### 3.4 农具预览与放置预览的职责分离

正式要求：

1. `FarmToolPreview` 只服务于 `Hoe` / `WateringCan` 的施工预览。
2. `PlacementPreview` 继续服务于 `SeedData`、`SaplingData`、普通 `PlaceableItemData`。
3. 不能把农田施工链的 `1.5 x 1.5 footprint` 障碍规则直接上提成所有放置物的统一规则。
4. 不能为了修箱子压耕地问题，把种子和树苗一起拖进普通 placeable 的规则里。
