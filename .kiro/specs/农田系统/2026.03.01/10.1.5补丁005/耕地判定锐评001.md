### 🔴 第二部分：原报告解决方案的致命缺陷（核心锐评）

请代码执行 Agent **绝对不要**照搬原报告中“第 5.1 节 方案 A”的伪代码逻辑，其中存在以下三个严重的系统性级缺陷：

#### 🧨 缺陷一：极其恐怖的性能黑洞（O(N) 嵌套查询）

* **报告原案**：在 `HasTreeInArea` 和 `HasChestInArea` 中，设计了嵌套的 3x3 `for` 循环（遍历周围 8 格），并在循环内部调用 `Object.FindObjectsByType<TreeController>(...)`。
* **锐评与依据**：
* **依据**：《FarmToolPreview.cs》第 225 行 `UpdateHoePreview` 显示，预览更新是高频触发的（随着鼠标移动或状态每帧刷新）。
* **分析**：`FindObjectsByType` 是 Unity 中极其昂贵的全场景遍历 API。如果按照报告的设计，鼠标每移动一格，系统将执行 `1次物理碰撞检测 + 9次全场景树木遍历 + 9次全场景箱子遍历`。在一个种植了大量作物和树木的后期农场中，这种 O(N) 的嵌套查询会立刻引发断崖式的掉帧，导致游戏卡死。


* **重构指令**：**严禁在循环内调用全局查找。** 必须在循环外一次性抓取全场景树木/箱子列表，然后遍历该列表，通过比较物体坐标与目标中心点 `Vector2.Distance` 或构建 1.5 单位的 `Rect/Bounds` 来判定是否在范围内。

#### 🧨 缺陷二：无视现有架构，重复造轮子（楼层隔离检测）

* **报告原案**：报告手写了一个 `IsSameLayer` 方法，试图通过解析 `SortingLayer.IDToName` 并硬编码字符串比较（如 `layerName == "Layer 1" ? 0 : 1`）来映射楼层。
* **锐评与依据**：
* **依据 1**：《PlacementLayerDetector.cs》中已经完整实现了 `GetSortingLayerAtPosition` 和 `GetPlayerSortingLayer` 方法。
* **依据 2**：《FarmToolPreview.cs》第 120 行已经定义了 `private string currentSortingLayer = "Layer 1";`，并在第 795 行 `UpdateSortingLayer` 中实时跟随玩家更新。
* **分析**：报告作者没有仔细查阅现有的放置和楼层工具类，试图用魔法数字和硬编码字符串来建立楼层映射。如果项目后续添加 "LAYER 4"，这个硬编码逻辑将直接崩溃。


* **重构指令**：**直接复用现有状态！** 修改 `HasFarmingObstacle` 的签名，让上游（《FarmToolPreview.cs》）直接把 `currentSortingLayer` 这个 string 传进来。在检测碰撞体或树木/箱子时，只需获取其 `SpriteRenderer.sortingLayerName` 并使用 `string.Equals` 对比即可，完全不需要转成 int 索引。

#### 🧨 缺陷三：忽略底层物理对齐机制（坐标判定漂移）

* **报告原案**：在判断箱子和树木位置时，作为 fallback 使用了 `chest.transform.position` 和 `tree.transform.position`。
* **锐评与依据**：
* **依据 1**：《PlacementGridCalculator.cs》第 185 行明确注释：“TreeController 和 ChestController 在 Awake/Start 时会执行底部对齐... 这会改变 Collider 相对于物品原点的位置。”
* **依据 2**：《ChestController.cs》第 559 行包含 `GetColliderBounds()` 方法。
* **分析**：项目中的特殊物体（树、箱子）为了实现 2.5D 遮挡效果，Sprite 底部被大幅度下移了，真实的物理占用（Collider）与 `transform.position` 存在严重偏差。如果直接取 Transform 坐标进行 1.5 单位的绝对范围判定，会导致判定框在 Y 轴上发生错位（视觉上没碰到的判定碰到了，碰到的判定没碰到）。


* **重构指令**：**必须使用 Collider 边界坐标！** 获取特殊物体位置时，必须调用各自类中已有的 `GetColliderBounds().center`，或者依赖 `PlacementGridCalculator.GetColliderCenterAfterBottomAlign()`，绝对不能直接使用原始的 Transform 坐标点。
