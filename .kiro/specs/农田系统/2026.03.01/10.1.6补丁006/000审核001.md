# 耕地判定 Bug 全面审核报告

## 问题概述

**严重性**：🔴 高危 - 核心判定逻辑失效

**现象描述**：
- 在树苗附近创建耕地时，预览显示绿色可交互，实际也能创建成功
- 树苗在某些情况下会直接消失（疑似锄头交互冲突）
- 判定规则应该是：以耕地中心为中心，边长 1.5 的正方形内不允许有特定标签物体

**测试案例**：
```
树苗位置 A: Vector3(-21.98, 9.97, 0)
树苗位置 B: Vector3(-21.98, 12.08, 0)

测试耕地位置：
- Pos=(-22, 9, 0)  → 显示红色 ✅ 判定正确
- Pos=(-22, 10, 0) → 显示绿色 ❌ 判定错误（应该红色）
- Pos=(-23, 10, 0) → 显示绿色 ❌ 判定错误（应该红色）
- Pos=(-23, 9, 0)  → 显示绿色 ❌ 判定错误（应该红色）
```

**预期行为**：
- 树苗 A 在 (-21.98, 9.97, 0)，应该阻止在 (-22, 9, 0)、(-22, 10, 0)、(-23, 10, 0)、(-23, 9, 0) 创建耕地
- 只有 (-22, 9, 0) 判定正确，其他三个位置都错误显示绿色

## 一、坐标系统分析

### 1.1 格子中心计算

**PlacementGridCalculator.GetCellCenter()**：
```csharp
public static Vector3 GetCellCenter(Vector3 worldPosition)
{
    return new Vector3(
        Mathf.Floor(worldPosition.x) + 0.5f,
        Mathf.Floor(worldPosition.y) + 0.5f,
        worldPosition.z
    );
}
```

**测试验证**：
```
输入 (-22, 10, 0) → 输出 (-22 + 0.5, 10 + 0.5, 0) = (-21.5, 10.5, 0)
输入 (-22, 9, 0)  → 输出 (-22 + 0.5, 9 + 0.5, 0)  = (-21.5, 9.5, 0)
输入 (-23, 10, 0) → 输出 (-23 + 0.5, 10 + 0.5, 0) = (-22.5, 10.5, 0)
输入 (-23, 9, 0)  → 输出 (-23 + 0.5, 9 + 0.5, 0)  = (-22.5, 9.5, 0)
```

**结论**：格子中心计算逻辑正确，使用标准的 `Floor + 0.5` 公式。

### 1.2 FarmTileManager 的格子中心获取

**FarmTileManager 使用 LayerTilemaps.GetCellCenterWorld()**：
```csharp
// FarmToolPreview.cs Line 1456
Vector3 cellCenter = GetCellCenterWorld(layerIndex, cellPos);

private Vector3 GetCellCenterWorld(int layerIndex, Vector3Int cellPos)
{
    var tilemaps = FarmTileManager.Instance?.GetLayerTilemaps(layerIndex);
    if (tilemaps != null)
    {
        return tilemaps.GetCellCenterWorld(cellPos);
    }
    
    // 回退：使用 PlacementGridCalculator
    return PlacementGridCalculator.GetCellCenter(new Vector3(cellPos.x + 0.5f, cellPos.y + 0.5f, 0));
}
```

**问题点**：
- `tilemaps.GetCellCenterWorld(cellPos)` 依赖 Unity Tilemap 的 Grid 组件
- 如果 Grid 的 cellSize 不是 (1, 1, 0)，会导致坐标偏移
- 回退逻辑中 `cellPos.x + 0.5f` 是错误的，应该直接传 `cellPos`

**修正建议**：
```csharp
// 回退逻辑应该是：
return PlacementGridCalculator.GetCellCenter(new Vector3(cellPos.x, cellPos.y, 0));
// 而不是：
return PlacementGridCalculator.GetCellCenter(new Vector3(cellPos.x + 0.5f, cellPos.y + 0.5f, 0));
```

## 二、障碍物检测分析

### 2.1 HasFarmingObstacle 方法

**PlacementValidator.HasFarmingObstacle()**：
```csharp
public static bool HasFarmingObstacle(Vector3 cellCenter)
{
    // 从 FarmTileManager 读取配置
    float radius = FarmTileManager.FarmingObstacleCheckRadius;  // 1.5
    string[] obstacleTags = FarmTileManager.FarmingObstacleTags;
    string[] whitelistTags = FarmTileManager.FarmingWhitelistTags;

    // 1. 碰撞体检测（使用 AABB 正方形检测）
    Vector2 boxSize = new Vector2(radius, radius);
    Collider2D[] hits = Physics2D.OverlapBoxAll(cellCenter, boxSize, 0f);
    
    // 2. 检测无碰撞体的树苗（Stage 0）
    if (HasTreeAtPositionStatic(cellCenter))
        return true;
    
    // 3. 检测无碰撞体的箱子
    if (HasChestAtPositionStatic(cellCenter))
        return true;
    
    return false;
}
```

### 2.2 关键问题：OverlapBoxAll 的参数理解

**Physics2D.OverlapBoxAll 文档**：
```csharp
public static Collider2D[] OverlapBoxAll(
    Vector2 point,      // 盒子中心点
    Vector2 size,       // 盒子大小（不是半径！）
    float angle         // 旋转角度
);
```

**当前代码**：
```csharp
Vector2 boxSize = new Vector2(radius, radius);  // radius = 1.5
Collider2D[] hits = Physics2D.OverlapBoxAll(cellCenter, boxSize, 0f);
```

**实际检测范围**：
- 盒子大小 = (1.5, 1.5)
- 盒子中心 = cellCenter
- 检测范围 = [cellCenter.x - 0.75, cellCenter.x + 0.75] × [cellCenter.y - 0.75, cellCenter.y + 0.75]

**预期检测范围**：
- 以耕地中心为中心，边长 1.5 的正方形
- 检测范围应该是 = [cellCenter.x - 0.75, cellCenter.x + 0.75] × [cellCenter.y - 0.75, cellCenter.y + 0.75]

**结论**：OverlapBoxAll 的 size 参数是盒子的完整大小，不是半径！当前代码实际检测范围是 1.5×1.5 的正方形，符合预期。

### 2.3 树苗检测逻辑

**HasTreeAtPositionStatic()**：
```csharp
private static bool HasTreeAtPositionStatic(Vector3 cellCenter)
{
    Vector2Int checkCellIndex = PlacementGridCalculator.GetCellIndex(cellCenter);
    
    // 遍历场景中所有 TreeController
    var allTrees = Object.FindObjectsByType<TreeController>(FindObjectsSortMode.None);
    foreach (var tree in allTrees)
    {
        Vector3 treeRootPos = tree.transform.parent != null 
            ? tree.transform.parent.position 
            : tree.transform.position;
        
        Vector2Int treeCellIndex = PlacementGridCalculator.GetCellIndex(treeRootPos);
        
        if (checkCellIndex == treeCellIndex)
            return true;
    }
    
    return false;
}
```

**问题点**：
- 这个方法只检查树苗是否在同一个格子内
- 不检查树苗是否在 1.5×1.5 的正方形范围内
- 只有当树苗和耕地在完全相同的格子时才返回 true

**测试验证**：
```
树苗位置 A: (-21.98, 9.97, 0)
树苗格子索引: Floor(-21.98, 9.97) = (-22, 9)

耕地位置 (-22, 10, 0)
耕地中心: (-21.5, 10.5, 0)
耕地格子索引: Floor(-21.5, 10.5) = (-22, 10)  ❌ 不匹配

耕地位置 (-22, 9, 0)
耕地中心: (-21.5, 9.5, 0)
耕地格子索引: Floor(-21.5, 9.5) = (-22, 9)  ✅ 匹配
```

**根本原因**：
- `HasTreeAtPositionStatic` 只检查格子索引是否相同
- 不检查树苗是否在 1.5×1.5 的正方形范围内
- 导致只有树苗和耕地在同一格子时才会阻止，相邻格子不会阻止

## 四、根本原因总结

### 4.1 核心问题

**HasTreeAtPositionStatic 的逻辑错误**：
```csharp
// 当前逻辑：只检查格子索引是否相同
Vector2Int checkCellIndex = PlacementGridCalculator.GetCellIndex(cellCenter);
Vector2Int treeCellIndex = PlacementGridCalculator.GetCellIndex(treeRootPos);
if (checkCellIndex == treeCellIndex)
    return true;
```

**应该的逻辑**：检查树苗是否在 1.5×1.5 的正方形范围内
```csharp
// 应该检查距离
float dx = Mathf.Abs(cellCenter.x - treeRootPos.x);
float dy = Mathf.Abs(cellCenter.y - treeRootPos.y);
if (dx <= 0.75f && dy <= 0.75f)  // 1.5 / 2 = 0.75
    return true;
```

### 4.2 为什么 OverlapBoxAll 没有检测到树苗

**树苗 Stage 0 没有碰撞体**：
- TreeController 在 Stage 0（树苗阶段）不添加碰撞体
- 只有成长到 Stage 1+ 才会添加 PolygonCollider2D
- 因此 `Physics2D.OverlapBoxAll` 无法检测到 Stage 0 的树苗

**代码证据**（TreeController.cs）：
```csharp
private void UpdateColliderState()
{
    if (currentStageIndex == 0)
    {
        // Stage 0: 树苗阶段，禁用碰撞体
        if (polyCollider != null)
            polyCollider.enabled = false;
    }
    else
    {
        // Stage 1+: 启用碰撞体
        if (polyCollider != null)
            polyCollider.enabled = true;
    }
}
```

### 4.3 为什么只有一个位置判定正确

**(-22, 9, 0) 判定正确的原因**：
- 树苗位置：(-21.98, 9.97, 0)
- 树苗格子索引：Floor(-21.98, 9.97) = (-22, 9)
- 耕地中心：(-21.5, 9.5, 0)
- 耕地格子索引：Floor(-21.5, 9.5) = (-22, 9)
- 格子索引相同 → `HasTreeAtPositionStatic` 返回 true → 显示红色

**其他三个位置判定错误的原因**：
- 耕地格子索引与树苗格子索引不同
- `HasTreeAtPositionStatic` 返回 false
- 即使树苗在 1.5×1.5 范围内，也显示绿色

### 5.1 可能原因

**锄头和树苗交互冲突**：
- 用户提到"多次测试中有绝大多数情况下这两个位置的树苗会出现直接消失"

```csharp
    // ... 挖出树苗逻辑

```

### 5.2 交互优先级问题

**可能的执行顺序**：
1. 用户左键点击耕地位置
2. 锄头工具的射线检测命中了树苗的 Collider（如果有）或 Sprite
3. TreeController.OnHit 被触发
4. HandleSaplingDigOut 执行，树苗被销毁
5. 耕地创建逻辑执行，因为树苗已经不存在，判定通过

**建议**：
- 这个问题需要单独处理，不在本次审核范围内
- 记入代办：解决锄头和树苗交互冲突问题

## 六、修复方案

### 6.1 修复 HasTreeAtPositionStatic

**方案 A：修改为距离检测（推荐）**

```csharp
private static bool HasTreeAtPositionStatic(Vector3 cellCenter)
{
    float checkRadius = 0.75f;  // 1.5 / 2 = 0.75
    
    // 遍历场景中所有 TreeController
    var allTrees = Object.FindObjectsByType<TreeController>(FindObjectsSortMode.None);
    foreach (var tree in allTrees)
    {
        Vector3 treeRootPos = tree.transform.parent != null 
            ? tree.transform.parent.position 
            : tree.transform.position;
        
        // 使用 AABB 正方形距离判断（与 OverlapBoxAll 一致）
        float dx = Mathf.Abs(cellCenter.x - treeRootPos.x);
        float dy = Mathf.Abs(cellCenter.y - treeRootPos.y);
        
        if (dx <= checkRadius && dy <= checkRadius)
            return true;
    }
    
    return false;
}
```

**优点**：
- 与配置的 `FarmingObstacleCheckRadius = 1.5` 一致
- 与 `OverlapBoxAll` 的检测逻辑一致
- 数学上正确

**缺点**：
- 需要遍历所有树木，性能略低（但树木数量通常不多）

### 6.2 修复 GetCellCenterWorld 的回退逻辑

**当前代码**：
```csharp
return PlacementGridCalculator.GetCellCenter(new Vector3(cellPos.x + 0.5f, cellPos.y + 0.5f, 0));
```

**修正后**：
```csharp
return PlacementGridCalculator.GetCellCenter(new Vector3(cellPos.x, cellPos.y, 0));
```

**原因**：
- `PlacementGridCalculator.GetCellCenter` 内部已经做了 `Floor + 0.5` 的计算
- 传入 `cellPos.x + 0.5f` 会导致重复加 0.5，造成坐标偏移

### 6.3 同步修复 HasChestAtPositionStatic

**当前代码**：
```csharp
private static bool HasChestAtPositionStatic(Vector3 cellCenter)
{
    Vector2Int checkCellIndex = PlacementGridCalculator.GetCellIndex(cellCenter);
    
    var allChests = Object.FindObjectsByType<ChestController>(FindObjectsSortMode.None);
    foreach (var chest in allChests)
    {
        var collider = chest.GetComponentInChildren<Collider2D>();
        if (collider != null)
        {
            Vector2Int chestCellIndex = PlacementGridCalculator.GetCellIndex(collider.bounds.center);
            if (checkCellIndex == chestCellIndex)
                return true;
        }
        else
        {
            Vector2Int chestCellIndex = PlacementGridCalculator.GetCellIndex(chest.transform.position);
            if (checkCellIndex == chestCellIndex)
                return true;
        }
    }
    
    return false;
}
```

**修正后**：
```csharp
private static bool HasChestAtPositionStatic(Vector3 cellCenter)
{
    float checkRadius = 0.75f;  // 1.5 / 2 = 0.75
    
    var allChests = Object.FindObjectsByType<ChestController>(FindObjectsSortMode.None);
    foreach (var chest in allChests)
    {
        Vector3 chestPos;
        var collider = chest.GetComponentInChildren<Collider2D>();
        if (collider != null)
        {
            chestPos = collider.bounds.center;
        }
        else
        {
            chestPos = chest.transform.position;
        }
        
        // 使用 AABB 正方形距离判断
        float dx = Mathf.Abs(cellCenter.x - chestPos.x);
        float dy = Mathf.Abs(cellCenter.y - chestPos.y);
        
        if (dx <= checkRadius && dy <= checkRadius)
            return true;
    }
    
    return false;
}
```

## 十、总结

### 10.1 根本原因

**HasTreeAtPositionStatic 的逻辑错误**：
- 只检查格子索引是否相同，不检查距离
- 导致只有树苗和耕地在同一格子时才会阻止
- 相邻格子内的树苗不会被检测到

### 10.2 修复方案

**修改为距离检测**：
- 使用 AABB 正方形距离判断
- 检测范围：以耕地中心为中心，半径 0.75（1.5/2）的正方形
- 与 `OverlapBoxAll` 的检测逻辑一致

### 10.3 预期效果
