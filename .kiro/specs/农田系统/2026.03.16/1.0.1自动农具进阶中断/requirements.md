# 1.0.1 自动农具进阶中断 - 需求文档

## 背景

当前自动农具 FIFO 队列已经存在，但中断语义仍偏“允许缓存切换”，与用户要求不一致。用户明确要求：
- 自动队列进行中，不能把当前工具从背包拖走；
- 自动队列进行中，不能通过 Toolbar 点击、滚轮、数字键切换工具；
- 被拒绝时要给失败音效和图标/槽位抖动；
- 系统要知道玩家想中断，后续队列与预览要正确收束；
- 手动长按连续动作不能被这套拒绝逻辑误伤。

## 目标

1. 自动农具队列进行中，工具切换与拖走都被严格拒绝。
2. 拒绝行为必须给用户明确反馈。
3. 移动中断仍保留，但不触发抖动。
4. 手动长按连续动作保持原有体验。

## 需求范围

### R1 自动队列期间拒绝工具切换
- 适用入口：
  - Toolbar 左键点击
  - 滚轮切换
  - 数字键切换
- 自动队列期间不再缓存 hotbar 切换意图，也不允许动作结束后再偷偷应用。
- 拒绝后需要：
  - 播放失败音效
  - 对目标槽位或当前工具图标做抖动反馈
  - 按既定口径处理中断预览与后续等待队列

### R2 自动队列期间拒绝拖走当前工具
- 打开背包后，如果试图拖走当前用于自动队列的工具，必须直接拒绝拖拽开始。
- 拒绝后需要：
  - 播放失败音效
  - 对源槽位图标做抖动反馈
- 不允许出现“背包里已经拿走了工具，但场景里仍继续自动耕地/浇水”的情况。

### R3 中断口径
- 自动队列期间：
  - 工具切换 / 拖走工具：拒绝 + 失败反馈 + 收束后续队列/预览
  - 手动移动：仍然中断，但不抖动
- 手动长按连续动作：
  - 玩家持续按住鼠标时，不因为这套自动队列拒绝逻辑被错误中断

## 非目标

- 本子工作区不处理作物排序和浇水样式问题。
- 本子工作区不改普通 placeable 放置规则。

## 验收标准

1. 自动锄地/浇水队列进行中，滚轮、数字键、Toolbar 点击都不能切换当前工具。
2. 自动队列进行中，打开背包尝试拖走当前工具时，拖拽不会开始，并立即出现失败反馈。
3. 自动队列进行中，移动仍会中断，但不会播放抖动。
4. 手动长按使用农具时，不会被误当成自动队列拒绝场景。
