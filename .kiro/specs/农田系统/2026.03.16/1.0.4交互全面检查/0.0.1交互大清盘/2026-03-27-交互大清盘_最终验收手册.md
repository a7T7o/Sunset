## 5.1 `A1` 树苗连续放置事务

### 测试前提

- 快捷栏中准备足够数量的树苗
- 玩家站在树苗可放置的农田或允许放置区域附近

### 入口路径

- 选中树苗进入放置预览

### 操作步骤

1. 把鼠标停在玩家前方 1 到 2 格的位置，不要移动鼠标。
2. 连续点击放置，让玩家向前走并连续种下 3 到 5 格树苗。
3. 在刚种下一棵后继续保持鼠标不动，观察 preview 是否会自动刷新到下一格。
4. 把鼠标压在已种下树苗边缘，继续快速连点，观察是否出现“看起来能放、实际又不对”的幽灵格。

### 预期结果

- 鼠标不动时，随着玩家走近，preview 会自己刷新到新的候选格
- 刚放下一棵后，不会继续拿上一格做判断
- 不会出现新树苗已落地，但逻辑仍像没占位或还在占旧位的幽灵状态
- 连续放置时，整体体验应是“玩家一路往前走一路种”，而不是“必须抖一下鼠标才刷新”

### 失败判读

- 如果刚种下一棵后 preview 仍钉在上一格，判定为 `A1-刷新未闭环`
- 如果视觉上可放但逻辑又失败，或出现新旧占位打架，判定为 `A1-幽灵占位仍在`
- 如果必须靠鼠标挪一下才刷新下一格，判定为 `A1-候选格刷新仍绑定像素位移`

## 5.2 `A2` 树木倒下事务

### 测试前提

- 准备可砍树的斧头
- 场景中可稳定找到 1 阶段树、2 阶段树，以及 0 阶段树苗

### 入口路径

- 对 1 阶段树和 2 阶段树分别进行砍树测试

### 操作步骤

1. 先砍 1 阶段树，直到树进入倒下动画。
2. 在树倒下动画快结束时继续连续挥砍。
3. 再对 2 阶段树重复同样操作。
4. 最后补跑一次 0 阶段树苗正常流程，确认没有被这轮改坏。

### 预期结果

- 1 阶段树和 2 阶段树在倒下后半程继续挥砍时，不会重新“立起来”
- 倒下期间不会再次吃命中
- 不会重复掉落、重复经验、重复砍倒反馈
- 树桩路径和无树桩路径都应表现一致

### 失败判读

- 如果树倒下快结束时又重新竖起来，判定为 `A2-死亡状态未锁死`
- 如果倒下期间还能重复触发掉落或经验，判定为 `A2-重复提交仍在`
- 如果只有某一阶段正常、另一阶段异常，判定为 `A2-阶段分支仍不一致`

## 5.3 `A3` 工具失效强制收尾事务

### 测试前提

- 准备只剩最后一次耐久的斧头
- 准备只剩最后一次耐久的锄头
- 准备只剩最后一次有效浇水次数的水壶
- 同时准备“完全没耐久 / 空水壶 / 精力不足”的场景

### 入口路径

- 分别对斧头、锄头、水壶做最后一次有效动作测试

### 操作步骤

1. 使用只剩最后一点耐久的斧头挥砍一次。
2. 观察这一次动作是否完整播完，以及播完后是否立刻损坏、清槽、出反馈。
3. 在工具已经损坏消失后，立刻再按一次挥砍。
4. 对只剩最后一点耐久的锄头重复同样验证。
5. 对只剩最后一次有效浇水的水壶重复同样验证。
6. 再额外测试空水壶重复尝试，以及精力不足时尝试动作。

### 预期结果

- 耐久大于 0 时，最后一次动作允许完整成立
- 这次动作完成后才扣掉最后一点耐久
- 若本次扣到 0，应立即触发损坏特效、气泡和清槽
- 工具消失后，当前动作不会继续假播尾巴
- 下一次尝试时，会在动作起播前被拦下
- 空水壶和精力不足不会被误判成“工具损坏”那套反馈

### 失败判读

- 如果耐久已经没了还能继续起播，判定为 `A3-动作前检查未闭环`
- 如果最后一次动作还没播完就提前扣坏，判定为 `A3-扣账时机错误`
- 如果工具都消失了但动画还继续，判定为 `A3-动作链未强制收尾`
- 如果空水壶或精力不足走成了损坏反馈，判定为 `A3-前置阻断语义串线`

## 5.4 `B1` hover 遮挡最终口径

### 测试前提

- 农田旁边准备树木、箱子或其他可透明化障碍

### 入口路径

- 进入锄头或播种等会出现农田 hover 预览的状态

### 操作步骤

1. 让农田中心格正对一个障碍，观察该障碍是否透明。
2. 让只有侧边格靠近障碍，但中心格不被挡住。
3. 让箱子紧邻农田 hover 区域，观察箱子是否被误透明。
4. 把障碍移开或让 hover 离开，观察透明是否正确恢复。

### 预期结果

- 只有中心格被挡时才透明
- 侧边格蹭到不应触发整片透明
- 箱子不会因为农田 hover 一开就远距离透明
- 障碍移出后透明应及时恢复，不残留

### 失败判读

- 如果边上蹭一下就整片透明，判定为 `B1-范围仍大于中心格`
- 如果箱子离得并不近也被一起透明，判定为 `B1-外扩仍在`
- 如果透明不恢复，判定为 `B1-恢复时机异常`

## 5.5 `B2` 箱子交互链

### 测试前提

- 准备一个可开启的箱子
- 玩家背包中同时准备普通可堆叠物和带耐久工具
- 最好带着一个 placeable 预览去开箱

### 入口路径

- 在普通移动状态和放置模式下都测试一次箱子

### 操作步骤

1. 站在稍远位置右键箱子，观察玩家是否会走到合理距离再开启。
2. 带着 placeable 预览再次右键箱子，确认开箱后放置模式是否还在。
3. 在箱子中放多组普通同物品，点击 `Sort`。
4. 在箱子中混入带耐久工具，再点击 `Sort`。

### 预期结果

- 玩家会走到真正可开启的位置再开箱
- 开箱不应强制退出当前放置模式
- 普通同物品排序后会正确堆叠
- 带耐久或动态属性的实例物品不会被错误合并

### 失败判读

- 如果人还没走到合适位置就开，或总差一点打不开，判定为 `B2-开启距离不对`
- 如果开箱直接把放置模式清掉，判定为 `B2-状态链被错误打断`
- 如果 `Sort` 把耐久工具合并了，判定为 `B2-排序真源错误`

## 5.6 `B3` Toolbar / Hotbar 选中态真源

### 测试前提

- 快捷栏至少准备多个可区分槽位
- 最好同时覆盖滚轮、数字键、开箱、放置模式切换几种场景

### 入口路径

- 先在正常移动状态测试，再在开箱和放置状态测试

### 操作步骤

1. 用滚轮切换多个快捷栏槽位。
2. 用数字键切换前几格槽位。
3. 打开箱子前后反复切槽。
4. 在放置模式中切槽。

### 预期结果

- 任一时刻只有一个槽位是当前选中态
- UI 亮的槽位与实际手持一致
- 开箱前后、放置前后都不会出现双选中或错槽

### 失败判读

- 如果同时有两个槽都亮，判定为 `B3-双选中未消失`
- 如果 UI 亮的是一个，实际拿的是另一个，判定为 `B3-事实源仍分叉`

## 5.7 `B4` 高树冷却输入层前置拦截

### 测试前提

- 准备低等级斧头和可砍高等级树的高等级斧头
- 场景中准备高等级树

### 入口路径

- 先用低等级斧头测试，再换高等级斧头测试

### 操作步骤

1. 用低等级斧头第一次尝试砍高等级树。
2. 30 秒内继续重复尝试。
3. 换成正确等级斧头后再次尝试。

### 预期结果

- 低等级斧头不会成功起到正常挥砍效果
- 失败时会给出正确提示，但不会反复假挥砍
- 30 秒内重复尝试应在输入层被拦下
- 换成高等级斧头后能恢复正常砍树

### 失败判读

- 如果低等级斧头还能持续空挥砍，判定为 `B4-输入层未拦住`
- 如果 30 秒内还能不断播动作，判定为 `B4-冷却前置拦截未闭环`
- 如果换高等级斧头后仍被错误挡住，判定为 `B4-等级切换恢复异常`
