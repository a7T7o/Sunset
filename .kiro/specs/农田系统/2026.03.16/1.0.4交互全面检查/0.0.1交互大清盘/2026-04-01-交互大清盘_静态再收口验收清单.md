## 4. 逐项验收步骤

### A1 连续放置边界与近身直放手感

**入口**

- 选中树苗或种子，进入正常放置模式

**步骤 A1-1：近身 9 宫格直放**

1. 让玩家站在一个格子中间附近。
2. 鼠标分别指向玩家当前格与周围 8 格，连续左键放置。

**预期结果**

- 玩家当前主占格明显在本格时，当前格和周围 8 格应允许直接放下，不需要先导航走一步。
- 预览格和真正落点必须一致。

**失败判读**

- 如果玩家明明就站在中心附近，仍频繁先走路再放，说明近身直放判定还不够稳。
- 如果预览显示能放，落地却跑到别的格，说明 preview / click 仍未完全同源。

**步骤 A1-2：边界 10% 顺延**

1. 在已经成功放下一个树苗或已播种的格子边上停住鼠标。
2. 不把鼠标挪出当前格，只把鼠标压到当前格靠左 / 右 / 上 / 下边界约 10% 的位置。
3. 连续点击，观察预览格与实际落点。

**预期结果**

- 只有当鼠标真的贴近当前格边界时，预览才应顺着该方向延伸到邻格。
- 鼠标回到格子内部时，不应提前乱跳邻格。

**失败判读**

- 如果离边界还远就开始跳邻格，说明顺延范围还是过大。
- 如果鼠标已经明显贴边，但仍死卡当前格，说明顺延来源或边界窄带还不够对。

**步骤 A1-3：鼠标不动、玩家轻微移动后的重判**

1. 把鼠标停在连续放置链边界位置，不要再挪鼠标。
2. 让玩家自己因为连续放置或短距离移动稍微换位。
3. 继续点击，观察预览是否会自己跟着重判。

**预期结果**

- 不需要再抖一下鼠标，预览应在玩家换位后自动重新判断当前候选格。

**失败判读**

- 如果必须再动一下鼠标，预览才刷新，说明“放后 hold 释放”仍残留旧问题。

### B1 农田 hover 与 placeable hover 遮挡

**入口**

- 一次用锄头 / 水壶做农田 hover
- 一次用树苗或箱子做 placeable preview hover

**步骤 B1-1：农田 hover 只看中心格**

1. 让农田预览贴近一棵树、箱子或其他遮挡物边缘。
2. 缓慢移动鼠标，让中心格从“未被挡”过渡到“正好被挡”。

**预期结果**

- 农田 hover 只应在中心格真正被挡时触发透明。
- 旁边格子擦边、不在中心格上的遮挡，不该让整片提前透明。

**失败判读**

- 如果离得还远、只是外围碰到就触发，说明农田 hover footprint 仍然过大。

**步骤 B1-2：placeable hover 不应被农田链误清空**

1. 切到树苗或箱子预览。
2. 把预览放到树、建筑或岩石后面。
3. 缓慢进出遮挡边界。

**预期结果**

- placeable preview 应独立触发遮挡，不应出现“农田能透明、摆放预览完全不触发”的情况。
- 农田 preview 与 placeable preview 不应互相清空对方的遮挡状态。

**失败判读**

- 如果 placeable preview 仍几乎从不触发透明，说明 preview 来源隔离还没真正站住。

### B2 箱子走近停下与到位开启

**入口**

- 选一个玩家可直接交互的箱子，优先从多个方向测试

**步骤**

1. 分别从正面、侧面、斜角对箱子点交互。
2. 让玩家从略远位置自动走近。
3. 观察导航停止时机与箱子 UI 打开时机。

**预期结果**

- 玩家应该继续走到真正可开箱的距离，再停下并立刻开箱。
- 不应靠扩大箱子判定来“远远开箱”。
- 也不应出现已经停下但 UI 没开、过一会儿也不补开的情况。

**失败判读**

- 如果还需要比箱子判定更近的“玄学角度”才能开，说明 stop radius / pending auto interaction 仍未完全并口。
- 如果站住了却不开，说明“到位即开箱”链还有断点。

### B3 背包 / Toolbar 点击与拖拽选中真源

**入口**

- 打开背包
- 准备同时观察第一行 hotbar 映射格、普通背包格、箱子格

**步骤 B3-1：背包单击选中**

1. 打开背包后，先确认第一行映射当前 hotbar 选中格。
2. 依次左键点击第一行其他格、普通背包格。

**预期结果**

- 被点击的格子应立刻转成选中样式。
- 如果点击的是第一行 hotbar 映射格，Toolbar 也应同步到对应格。

**失败判读**

- 如果点了别的格却像没点，说明选中真源仍有分叉。

**步骤 B3-2：箱子格单击选中**

1. 打开箱子。
2. 单击箱子里的普通物品格。

**预期结果**

- 箱子格单击后也应有明确选中样式，而不是只有拖拽时才像被选中。

**失败判读**

- 如果箱子格单击仍没有任何选中反馈，说明箱子槽位选中链还没补全。

**步骤 B3-3：拖拽起手与落点选中**

1. 从背包格长按拖出一个物品。
2. 观察起手格是否先变成选中态。
3. 把物品放到新格或箱子格。

**预期结果**

- 拖拽开始时，源格应先进入选中态。
- 松手后的最终落点应成为新的选中态。
- 源格不应残留错误选中影子。

**失败判读**

- 如果起手格不亮、落点不亮，或跨区域后旧格残留选中，说明选中真源仍未收成一条链。

### C2 Tooltip 与工具状态条运行时入口

**入口**

- 准备一个有耐久的工具、一个水壶、几个普通物品

**步骤 C2-1：Toolbar tooltip**

1. 鼠标悬浮到 hotbar 里的工具和普通物品上。

**预期结果**

- tooltip 应在短延迟后稳定出现。
- 从一个格子挪到另一个格子时，应立即刷新内容，而不是重新卡长延迟。

**失败判读**

- 如果完全不出现，或频繁出现“逻辑触发了但看不到”，说明 tooltip 运行时画布挂载仍有问题。

**步骤 C2-2：背包 tooltip**

1. 打开背包。
2. 鼠标悬浮到背包物品格、箱子物品格。

**预期结果**

- 背包和箱子里的 tooltip 都应可见，且应跟着当前 UI 画布正确显示。

**失败判读**

- 如果 Toolbar 有、背包没有，或箱子里没有，说明 tooltip 入口还没有完全统一。

**步骤 C2-3：工具状态条显示策略**

1. 不开背包，只切换选中一个有耐久工具、一个水壶。
2. 悬浮到 hotbar 工具格。
3. 打开背包，再观察所有工具格。

**预期结果**

- 不开背包时：选中工具格或鼠标悬浮格应显示状态条；最近刚使用过的工具格也可短时间保留状态条。
- 打开背包后：背包里相关工具格应显示状态条。
- 水壶应按水量显示，非水壶工具应按耐久显示。

**失败判读**

- 如果水壶仍没有任何状态条，说明运行时水量链还没真正接通。
- 如果显示逻辑与“选中 / hover / 打开背包全显”明显不符，说明状态条可见性仍未收口。

### B5 玩家气泡样式语言

**入口**

- 优先用以下任一方式触发玩家气泡：
  - 让水壶空水后继续尝试使用
  - 让工具损坏触发提示
  - 用低级斧头去砍高等级树触发提示

**步骤**

1. 连续触发 2 到 3 次玩家气泡。
2. 观察颜色、对比度、宽度、换行和整体风格。
3. 同时对照当前 NPC 气泡风格。

**预期结果**

- 玩家气泡应明显更接近 NPC 当前暖色纸片气泡语言。
- 文字应比上一版更清楚，不应再因为深底浅字或过窄宽度导致难读。
- 自然换行应比上一版更顺，不应把短句切得太碎。

**失败判读**

- 如果仍然一眼像另一套系统的气泡，不像 NPC 同语言，说明样式只修到一半。
- 如果文字依旧难看清、短句被切得很怪，说明颜色或宽度口径仍需再收。
