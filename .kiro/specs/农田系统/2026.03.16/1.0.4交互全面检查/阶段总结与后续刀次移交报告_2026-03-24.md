## 3. 真实剩余问题总表

当前第二刀之外，真正剩余且应进入后续刀次的问题是：

### A. `Toolbar 左键后 A/D 独立框选`

这是用户最新补充的明确问题。

当前判断：

- 更像是 `Toolbar` 鼠标点击后的选中态 / 输入焦点 / 键盘导航状态没有正确隔离
- 它不属于 second-blade 的放置链问题
- 应进入后续刀次作为核心问题，而不是回头污染第二刀

### B. 更高层的 `Toolbar / 背包 / 锁定态` 边界问题

这里包括：

- `Toolbar` 与背包第一行的同源选中态
- 鼠标点击、滚轮、数字键、UI 打开关闭、锁定态之间的输入优先级
- 选中态、可交换态、可拖拽态、锁定态的统一规则

当前判断：

- 这是后续刀次最该优先打的主主题
- 因为它直接决定玩家的核心输入正确性和 UI 体感

### C. 工具耐久 / 精力 / Tooltip / SO 参数统一口径

这里包括：

- 工具运行时参数与 UI 展示的统一
- 精力 / 生命 / 耐久度等文本与数据口径的一致性
- ScriptableObject 参数定义、运行时消耗、Tooltip 展示三层的统一

当前判断：

- 这是更高层的统一问题
- 重要，但不比 `Toolbar / 背包 / 锁定态` 的直接交互正确性更急
- 适合后移到后续刀次，而不是抢占第三刀入口

## 4. 后续刀次建议排序

### 第三刀最该先打什么

建议第三刀主题为：

- `Toolbar / 背包 / 锁定态输入边界统一`

### 为什么不是别的

因为它同时覆盖：

- 用户最新补充的 `Toolbar 左键后 A/D 独立框选`
- `Toolbar` / 背包 / 锁定态的输入焦点和选中态问题
- 鼠标、键盘、滚轮、UI 打开态之间的真实交互边界

这是一组直接影响“现在用起来对不对”的核心交互问题，优先级高于更上层的数据展示统一。

### 哪些问题可以后移

可以后移到后续刀次的主题：

- 工具耐久 / 精力 / Tooltip / SO 参数统一口径
- 更高层的数据展示和参数治理

原因：

- 这些问题重要，但当前不会像输入边界错误那样立即破坏基础操作正确性

## 5. 当前是否建议继续进入第三刀
