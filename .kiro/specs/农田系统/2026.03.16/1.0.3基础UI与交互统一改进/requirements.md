# 1.0.3 基础 UI 与交互统一改进 - 需求文档

## 背景

`1.0.0图层与浇水修正`、`1.0.1自动农具进阶中断`、`1.0.2纠正001` 解决的是农田交互正确性与自动链路稳定性问题。当前这一轮的方向已经从“农田逻辑纠错”切换为“基础 UI 与物品交互统一改进”。

本轮用户先提出了一个收尾问题：背包中的 reject shake 手感明显比 Toolbar 更黏、更卡。live 代码复核后已确认根因不是 shake 协程，而是背包槽位 `Toggle` 仍保留默认视觉过渡，与自定义 shake 叠加冲突。该问题已在 `InventorySlotUI.cs` 中做最小修正。

用户随后要求把下一阶段真正要做的内容完整沉淀下来，不是为了堆文档，而是为了让后续进入“全面交互改进阶段”时，方向、边界、文件落点和验收标准都已经清楚。

## 当前阶段定位

`1.0.3` 不是继续修农田放置正确性，而是作为新的 UI / 交互统一阶段入口，重点面向：

1. 物品 Tooltip 真实接线与展示能力重建。
2. 种子袋保质期、开袋状态、剩余数量、工具实例耐久等“实例态信息”的 UI 出口建立。
3. 食物 / 药水使用效果从“只记日志”改为真正接入玩家状态系统。
4. 精力 / 体力 / stamina / vitality 等术语与显示口径统一。

## 真实问题定义

### R1 背包 reject shake 已完成收尾，但只算阶段前置结论

- 背包 reject shake 手感发黏的根因已确认并完成修复。
- 这不是 `1.0.3` 的主体任务，而是新阶段开始前的收尾闭环。
- 后续文档必须把这条结论写实，不再把 shake 问题描述成未查明。

### R2 Tooltip 现在不是“内容不够”，而是“接线不成立”

- `ItemTooltip.cs` 当前只显示 `itemData.description`。
- 项目中大量 `ItemData` 子类已经实现 `GetTooltipText()`，但 UI 完全没有消费这条能力。
- 更关键的是，代码侧几乎没有真正的 Tooltip 调用入口，当前更接近“有脚本、无接线”状态。

### R3 仅改 `GetTooltipText()` 仍无法解决种子袋和耐久显示

- 运行时需要展示的很多信息不在 `ItemData` 静态配置里，而在 `InventoryItem` 实例态里。
- 典型例子包括：
  - 种子袋是否已开袋
  - 剩余种子数量
  - 剩余保质期天数
  - 当前耐久 / 最大耐久
- 因此下一阶段必须补“实例态 Tooltip 入口”，而不是只改静态描述文案。

### R4 掉落链仍会丢失实例态信息

- 当前背包 / Toolbar 已经能显示工具耐久条，但掉落辅助链还只接受 `ItemStack`。
- `ItemDropHelper.cs` 已明确留有 `TODO`，说明掉落链尚不支持 `InventoryItem` 实例数据。
- 这意味着即使 Tooltip 和背包显示补齐，只要掉落再拾取仍丢实例数据，整条体验链就还没闭合。

### R5 食物 / 药水效果只做了一半

- `FoodData`、`PotionData`、`EnergySystem` 等底层能力已经存在。
- 但 `ItemUseConfirmDialog.cs` 当前仍只 `Debug.Log`，没有真正把食物 / 药水效果接到玩家状态。
- 所以下一阶段既包含“显示”，也包含“使用效果真正落地”。

### R6 术语口径需要统一

- 当前项目里同时存在“精力 / stamina / vitality / 体力”等口径。
- 如果不统一，后续 Tooltip、确认弹窗、状态 UI 会继续各说各话。

## 范围

### 本阶段包含

- 物品 Tooltip 静态与实例态展示链。
- 背包 / Toolbar / 箱子 / 装备等物品槽位的悬浮入口梳理。
- 实例态信息在 UI 中的展示与掉落链保持。
- 食物 / 药水使用效果与状态系统接线。
- 文案和术语统一。

### 本阶段不包含

- 农田放置规则、耕地判定、树苗距离、导航收尾等 `1.0.2` 主体逻辑。
- NPC、共享治理、Scene / Prefab 表现层大改。
- 重新发明已有 runtime 底层能力。

## 目标

1. 用户把鼠标悬停在物品槽位上时，看到的是已经接线成功、口径统一的 Tooltip，而不是一套闲置脚本。
2. Tooltip 至少能区分“静态配置”与“实例态运行时数据”。
3. 种子袋、工具、食物、药水这四类关键物品的交互说明与运行时状态都能被正确表达。
4. 食物 / 药水使用后，玩家状态产生真实变化，而不是只输出日志。
5. 后续阶段进入实现时，修改文件范围、优先级、验证路径都已经明确。

## 验收标准

1. `ItemTooltip` 不再只显示 `itemData.description`，而是真正消费统一后的 Tooltip 文本出口。
2. 悬浮到背包 / Toolbar / 箱子 / 装备槽位时，Tooltip 能稳定显示，不依赖手工临时调用。
3. 工具 Tooltip 能显示当前实例耐久；种子袋 Tooltip 能显示保质期与实例态种子信息。
4. 掉落再拾取后，工具耐久等实例态信息不丢失。
5. 食物 / 药水确认后，玩家精力 / 生命产生真实变化。
6. UI 文案中不再混用未定义的“精力 / 体力 / stamina / vitality”口径。
