# 1.0.0 图层与浇水修正 - 需求文档

## 背景

当前农田系统已经完成种子接入放置链，但现场验收暴露出两个体验缺口：
1. 作物创建后没有对齐玩家当前正确的遮挡/前后关系，导致角色与作物并排时视觉层级错误。
2. 浇水预览的水渍样式随机时机仍不对，用户要求必须在“成功入队后，并且鼠标移出当前农田格”的那一刻，才为下一次预览刷新随机样式。

## 目标

1. 种植成功后创建的作物实例必须与玩家现有排序体系一致。
2. 浇水预览必须稳定复现“当前格保持当前样式，移出后下一格才出现新的随机样式”的行为。

## 需求范围

### R1 作物创建后的排序修正
- `PlacementManager.ExecuteSeedPlacement(...)` 创建作物实例后，必须补齐 sorting layer、sorting order、必要的动态排序组件。
- 修正后作物的前后遮挡关系要与玩家当前正确的遮挡标准一致。
- 不允许破坏现有作物 prefab 的底部对齐、作物状态切换与持久化流程。

### R2 浇水样式随机时机修正
- 水壶预览进入模式时可初始化一个当前样式。
- 只有在浇水入队或执行成功之后，系统才标记“下一次需要新样式”。
- 标记生效后，只有鼠标离开刚刚成功入队/执行的那个格子时，才真正刷新下一次随机样式。
- 鼠标仍停留在当前格时，预览样式必须保持不变。

## 非目标

- 本子工作区不处理自动队列切换/拖拽拒绝，也不处理 UI 失败反馈。
- 本子工作区不调整普通 placeable、树苗或背包系统的其他行为。

## 验收标准

1. 在 `Primary` 中播种后，玩家与作物并排或穿行时，作物前后遮挡关系与玩家标准一致。
2. 连续进行两次浇水入队时：
   - 第一次成功后，当前格样式不立即跳变；
   - 鼠标离开当前格后，下一次预览才出现新随机样式。
3. 本轮修改不能引入新的编译错误或新的 farm 相关 warning。
