# 1.0.2纠正001 - 需求文档

## 背景

`1.0.1自动农具进阶中断` 在用户现场验收后暴露出新的理解偏差与严重边界问题。当前不是继续“补丁式追加判断”，而是要重新核对现有 UI、手持、动画、队列、导航、预览、种植显示链路的真实规则，再纠正错误口径。

用户明确指出：
1. 背包 UI 打开时，世界交互应总体禁止，只允许背包内部更新。
2. 背包交换限制不应只针对“自动农具队列”，而应覆盖所有“当前正在使用且不应被 UI 打断的手持内容/动作/流程”。
3. 当前实现出现“第一次交换被拒绝、第二次交换成功、关闭 UI 后原动作却继续完成”的错误，这是严重逻辑漏洞。
4. Toolbar / 背包抖动反馈位置也存在错误，未对齐项目既有 UI 关系。
5. 导航、队列、预览、中断在打开背包、WASD、右键导航、高速点击等边界下存在多处残留与状态错乱。
6. 种植后的作物偶发出现“类似预览幽灵”的半透明显示，但 Inspector 常规参数正常，说明问题不只在 `SpriteRenderer.color`。

## 当前轮次约束

1. 本轮只允许：
   - 读取代码
   - 读取 Unity / MCP 现场
   - 创建与更新文档
   - 输出分析结论与修正方案
2. 本轮禁止：
   - 修改任何业务代码
   - 修改场景 / Prefab / ScriptableObject
   - 直接进入修复实现
3. 必须先完成用户审核，再进入后续实现阶段。

## 目标

1. 彻底厘清当前农田交互与 UI/手持系统的实际规则边界。
2. 对用户指出的每个问题给出“已证实事实、影响范围、候选根因、推荐修正口径”。
3. 把新的稳定口径落盘为 `1.0.2纠正001` 文档，作为后续实现唯一依据。

## 需求范围

### R1 UI 打开时的“手持交换限制”规则重建
- 必须重新界定“正在使用”的统一语义。
- 该语义不能只覆盖自动农具队列，还要覆盖：
  - 动画播放中的工具动作
  - 正在执行中的逻辑/任务
  - placeable 放置流程
  - 后续会接入的食物与其他手持消耗流程
- 必须查清当前背包交换、Toolbar、Hotbar 选中、装备刷新、手持表现之间的真实调用链。

### R2 队列 / 导航 / 预览 / 中断的边界缺陷取证
- 必须查清以下问题的代码入口与状态机缺口：
  - 打开背包会中断导航，但保留预览与异常队列
  - WASD 会清理，右键导航却不会
  - 高速点击 + 持续移动会制造残留队列与残留预览
- 必须区分：
  - 玩家主动移动
  - 自动导航
  - UI 打开导致的交互冻结
  - 手持切换 / 交换

### R3 反馈表现规则纠正
- 滚轮切换被拒绝时，应抖动当前正在使用的工具，而不是滚轮目标槽位。
- 背包打开时，Toolbar 作为背包第一行的 UI 语义必须纳入口径说明。
- 需要明确：
  - 背包打开时哪个 UI 可以响应
  - 哪个 UI 只做视觉同步
  - 哪个 UI 不允许交互

### R4 种植后作物“幽灵透明态”问题取证
- 必须从作物创建、预览对象、材质、排序、父子层级、Ghost Tilemap / 预览状态残留等角度全面排查。
- 不能把 `SpriteRenderer.color` 正常就当成“透明问题已排除”。
- 必须结合用户提供的截图，给出最可能的调查路径和验证手段。

## 非目标

- 本轮不直接修复任何问题。
- 本轮不新增新的交互功能。
- 本轮不重新定义与当前问题无关的农田规则。

## 验收标准

1. 形成一份可审核的分析文档，覆盖用户指出的全部问题，不遗漏边界。
2. 每个问题都明确区分：
   - 已证实事实
   - 当前推断
   - 需要后续验证的点
3. 文档中必须给出后续实现阶段的最小修改范围与验收口径。
