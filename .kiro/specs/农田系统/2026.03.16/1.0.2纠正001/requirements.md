# 1.0.2纠正001 - 需求文档

## 背景

`1.0.0图层与浇水修正` 和 `1.0.1自动农具进阶中断` 已经回到 `Sunset/main`，但用户后续验收明确指出，这两轮之后仍有一组更深层的交互问题没有被真正统一：

1. 背包 UI 打开时，世界交互应整体冻结，但当前活跃手持槽位始终不可在背包内部被交换。
2. “不可交换”不能只覆盖自动农具队列，还必须覆盖所有当前正在执行、不会被 UI 切换打断的手持流程。
3. Toolbar / 背包 / Hotbar 的失败反馈要对齐到“当前活跃槽位”，不能抖错对象。
4. 打开 UI、`WASD`、右键导航、高频点击等边界下，导航、队列、预览、执行预览必须收口到同一套中断规则。
5. 种植后的作物偶发出现“像预览幽灵”的半透明显示，需要在种植显示链上做防守性闭环。

当前用户口径已经切到 `main-only`：
- 唯一开发现场：`D:\Unity\Unity_learning\Sunset @ main`
- 不再默认先回到 cleanroom / worktree
- 用户最终要在 `main` 的 Unity 场景里直接验收

## 目标

1. 在 `main` 上彻底收口这组农田交互升级，不再把 `1.0.2` 停留在历史分支语义里。
2. 统一“当前活跃手持流程”“UI 冻结”“移动中断”“预览所有权”的实现口径。
3. 让用户可以直接在 `main` 里验收这 6 条场景闭环，而不是继续追问“哪些还散落在外面”。

## 需求范围

### R1 活跃手持槽位保护
- 必须统一建模“当前正在使用且不应被 UI 切换破坏的手持槽位”。
- 保护范围至少覆盖：
  - 自动农具队列 / 导航 / 执行态
  - placeable 放置导航与执行态
  - UI 打开时被冻结、但关闭后还会继续恢复的手持流程
- 背包内部的拖拽、点击交换、整理、快速装备都必须受该保护约束。

### R2 UI 冻结与恢复
- UI 打开时，世界输入必须被冻结：
  - 不接受新的世界点击
  - 不接受新的农田队列创建
  - 不接受新的右键导航改点
- 自动链当前动作可以自然收尾，但 UI 内的交换尝试不能把它“中断后换掉”或“偷偷补做完毕”。
- UI 关闭后只能恢复到一个自洽状态，不能留下“预览还在 / 队列还在 / 导航没了”的错乱组合态。

### R3 移动中断统一
- `WASD` 与右键导航都必须被视为“移动意图”。
- 当农田自动链存在时，右键导航和 `WASD` 对队列、导航、预览的收尾口径必须一致。
- 高频点击与持续移动同时发生时，系统必须拒绝继续制造不可清理的残留。

### R4 反馈表现统一
- 被拒绝时，抖动的是当前活跃槽位，不是目标槽位。
- 背包打开时，Toolbar 作为背包第一行的镜像语义，需要与背包槽位同步反馈。
- 背包打开时 Toolbar 不承担新的交互入口，只承担镜像反馈。

### R5 透明幽灵作物防守闭环
- 种植完成后，预览对象与真实作物对象的生命周期必须彻底分离。
- 作物创建后要补齐图层 / 排序 / 动态排序。
- 如果作物或交互碰撞体曾被隐藏，初始化和加载时必须恢复。
- 预览隐藏时必须清掉遗留 sprite / color / localPosition，避免后续视觉串台。

## 非目标

- 本轮不处理 NPC 专属 Editor 编译问题。
- 本轮不处理 Unity MCP 网关异常本身。
- 本轮不扩写与农田交互无关的 NPC / 场景 / Dialogue 逻辑。

## 验收标准

1. 背包打开期间，活跃手持槽位始终不可交换，多次尝试结果一致。
2. 滚轮、数字键、快捷键、背包交换的失败反馈都落在当前活跃槽位。
3. UI 打开/关闭、右键导航、`WASD`、高速点击不再制造不可清理的农田残留。
4. 种植后不再出现由预览残留或显示链串台导致的幽灵透明态。
5. `Assembly-CSharp` 运行时代码编译通过，且不新增 farm 相关 warning。
