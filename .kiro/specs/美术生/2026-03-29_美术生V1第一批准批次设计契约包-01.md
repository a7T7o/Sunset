# 美术生 V1 第一批准批次设计契约包

## 0. 批次边界

当前批准批次固定为：

1. `Gold_Standard` 目录规范
2. `ArtBatch` schema
3. DNA extractor 输入/输出字段草案

不在当前批次里的内容：

- Python DNA extractor 原型实现
- Unity Scene / Prefab / Importer 改动
- 外部 MCP / skill 安装

## 1. 闭环总图

这三部分不是并列散文，而是一条闭环：

1. 人先把合格参考资产按 `Gold_Standard` 目录规范入库
2. 新建 `ArtBatch` manifest，明确这批资产引用了哪些 `Gold_Standard` 参考、计划产出什么、去哪里 Review、Reject 时删哪里
3. DNA extractor 后续实现时，严格读取 `ArtBatch.source` 与 `dna.inputProfile`
4. DNA extractor 输出分成两类：
   - 写回 `ArtBatch.dna.authoritative` 的硬约束
   - 写回 `ArtBatch.dna.candidates` 的候选值
5. Unity Review 链只认 `ArtBatch.outputs.reviewRoot` 和 `ArtBatch.review.scenePath`
6. Reject / cleanup 只认 `ArtBatch.cleanup.deleteTargets`

当前批次先把这条闭环的**目录、数据结构和字段契约**写死，下一刀才允许进入实现。

## 2. Gold_Standard 目录规范

## 2.1 根目录与分层

`Gold_Standard` 根目录固定为：

`Assets/Art_Reference/Gold_Standard/`

当前建议目录骨架：

```text
Assets/Art_Reference/Gold_Standard/
├─ _Intake/
├─ UI/
│  ├─ Frames/
│  ├─ Panels/
│  └─ Buttons/
├─ Environment/
│  ├─ Buildings/
│  └─ Tiles/
└─ Props/
```

目录职责：

- `_Intake/`
  - 暂存待审参考，不参与 DNA 提取
- `UI/Frames|Panels|Buttons/`
  - UI 底板、框体、按钮类 DNA 参考
- `Environment/Buildings|Tiles/`
  - 建筑白盒、Tile / 地表类 DNA 参考
- `Props/`
  - 小型道具、提示气泡或局部边框素材参考

## 2.2 入库单元

每个参考资产以“一个参考包目录”为最小单元，不直接散放平铺文件。

示例：

```text
Assets/Art_Reference/Gold_Standard/UI/Frames/gs_ui_frame_wood_01/
├─ gs_ui_frame_wood_01.png
├─ gs_ui_frame_wood_01.aseprite        (可选)
├─ gs_ui_frame_wood_01.meta.json       (必需)
└─ gs_ui_frame_wood_01.notes.md        (可选)
```

规则：

- `png`
  - 必需
  - 作为当前 V1 的 canonical DNA 输入
- `aseprite` / `ase`
  - 可选
  - 仅作为后续增强项和人工源文件保留
- `meta.json`
  - 必需
  - 记录批准、来源和用途信息
- `notes.md`
  - 可选
  - 记录边框说明、特殊禁忌、人工备注

## 2.3 接受的源资产类型

当前允许入库的源资产类型：

1. `*.png`
   - 必须是 lossless 导出
   - 必须能作为当前 V1 的 DNA 输入
2. `*.aseprite` / `*.ase`
   - 可选伴随源文件
   - 不作为当前 V1 的强依赖
3. `*.meta.json`
   - 必需 sidecar
4. `*.notes.md`
   - 可选 sidecar

当前必须拒收的类型：

- `*.jpg` / `*.jpeg`
- 带不可控压缩痕迹的 `webp`
- 带编辑器 UI、浏览器 UI 或拼贴边框的截图
- 无法追溯来源的网络图片
- 只存在源文件、没有对应 canonical `png` 的资产包

## 2.4 允许入库的参考图标准

能入库的参考图必须同时满足：

1. 已经被人明确认可为 Sunset 当前可继承的风格样本
2. 主要职责单一：
   - 一个框体
   - 一个底板
   - 一组同家族 tile
   - 一类建筑外轮廓 / 立面底图
3. 像素干净：
   - 没有模糊缩放
   - 没有脏色抗锯齿边
   - 透明边界清晰
4. 不混多种风格语言
5. 没有把文本、复杂 icon、角色立绘、整屏 UI 组合强行塞进同一 DNA 包
6. 能明确落到某个子类：
   - `ui_frame`
   - `ui_panel`
   - `building_base`
   - `tile_base`
   - `prop_small`

## 2.5 必须拒收的情况

下面情况必须拒收，不进入 `Gold_Standard`：

1. 只是整屏截图，不是单一资产参考包
2. 明显是运行时组合结果，拆不出基础资产责任
3. 有缩放模糊、半透明脏边、压缩噪点
4. 同一包里混了多个风格或多个不相干职责
5. 只有“好看感觉”，但没有可追溯来源或批准信息
6. 含文本排版、复杂 icon 或角色信息，导致 DNA 提取会混入不该继承的元素

## 2.6 命名规则

参考包 basename 固定使用 ASCII 小写蛇形：

`gs_<domain>_<family>_<variant>_<nn>`

示例：

- `gs_ui_frame_wood_01`
- `gs_ui_panel_stone_01`
- `gs_building_base_farmhouse_01`
- `gs_tile_ground_dirt_01`

命名字段说明：

- `domain`
  - `ui`
  - `building`
  - `tile`
  - `prop`
- `family`
  - `frame`
  - `panel`
  - `button`
  - `base`
  - `ground`
- `variant`
  - 材质或家族特征
- `nn`
  - 两位流水号

所有 sidecar 必须与 basename 完全同名。

## 2.7 `meta.json` 最小字段

每个参考包的 `meta.json` 最小字段固定为：

```json
{
  "refId": "gs_ui_frame_wood_01",
  "category": "ui_frame",
  "approved": true,
  "approvedBy": "human",
  "canonicalPng": "gs_ui_frame_wood_01.png",
  "sourceFiles": [
    "gs_ui_frame_wood_01.png",
    "gs_ui_frame_wood_01.aseprite"
  ],
  "notes": "9-slice border only as hint; text/icon not part of DNA"
}
```

## 3. ArtBatch schema

## 3.1 Manifest 路径

后续每个 batch 的 manifest 固定路径为：

`Assets/ArtPipeline/Metadata/Batches/<batchId>/ArtBatch.json`

当前 schema 文件路径：

`D:\Unity\Unity_learning\Sunset_worktrees\scene-build-5.0.0-001\.kiro\specs\美术生\ArtBatch.schema.json`

## 3.2 生命周期

当前 `ArtBatch.status` 固定允许值：

- `planned`
- `dna_contract_ready`
- `review_pending`
- `approved`
- `rejected`
- `cleaned`

## 3.3 最小字段集

`ArtBatch` 最小字段集固定为：

- `schemaVersion`
- `batchId`
- `batchType`
- `status`
- `source`
- `dna`
- `outputs`
- `review`
- `cleanup`

## 3.4 挂接关系

### source

`source` 负责回答：

- 这批引用了哪些 `Gold_Standard` 参考包
- 本批请求摘要是什么

最小字段：

- `goldStandardRefs[]`
- `requestSummary`

### dna

`dna` 负责回答：

- DNA extractor 未来该按什么输入契约读取
- 哪些结果写成硬约束
- 哪些结果只写成候选值

最小字段：

- `contractVersion`
- `inputProfile`
- `authoritative`
- `candidates`

### outputs

`outputs` 负责回答：

- 这一批计划产出哪些 review 文件
- review 文件落在哪个目录
- 通过后目标目录在哪里

最小字段：

- `reviewRoot`
- `approvedRoot`
- `plannedFiles[]`

### review

`review` 负责回答：

- 这一批进哪个 Unity scene 阅兵
- 按什么 layout profile 摆放
- 当前 review 状态是什么

最小字段：

- `scenePath`
- `layoutProfile`
- `status`

### cleanup

`cleanup` 负责回答：

- Reject 后删哪些文件
- 保留哪些证据

最小字段：

- `rejectPolicy`
- `deleteTargets[]`
- `keepEvidence[]`

## 4. DNA extractor 输入 / 输出字段草案

## 4.1 输入契约

DNA extractor 后续实现时，最小输入固定为：

| 字段 | 必需 | 来源 | 说明 |
| --- | --- | --- | --- |
| `schemaVersion` | 是 | 调用方 | 输入契约版本 |
| `batchId` | 是 | `ArtBatch.batchId` | 与批次绑定 |
| `assetKind` | 是 | `ArtBatch.batchType` | 当前提取目标种类 |
| `goldStandardRefs[]` | 是 | `ArtBatch.source.goldStandardRefs` | 参与提取的参考包路径 |
| `expectedPPU` | 是 | `ArtBatch.dna.inputProfile.expectedPPU` | 项目期望 PPU |
| `expectedGridSize` | 是 | `ArtBatch.dna.inputProfile.expectedGridSize` | 项目期望网格 |
| `analysisTargets[]` | 是 | `ArtBatch.dna.inputProfile.analysisTargets` | 要提取哪些项目 |
| `outputPath` | 是 | 调用方 | DNA 输出 JSON 落点 |

最小输入示例：

```json
{
  "schemaVersion": "1.0.0-draft",
  "batchId": "artbatch_ui_frame_20260329_001",
  "assetKind": "ui_frame",
  "goldStandardRefs": [
    "Assets/Art_Reference/Gold_Standard/UI/Frames/gs_ui_frame_wood_01"
  ],
  "expectedPPU": 16,
  "expectedGridSize": [16, 16],
  "analysisTargets": ["palette", "alpha_policy", "border_candidates"],
  "outputPath": "Assets/ArtPipeline/Metadata/Batches/artbatch_ui_frame_20260329_001/dna-output.json"
}
```

## 4.2 输出契约

DNA extractor 后续实现时，最小输出固定为：

| 字段 | 必需 | 写回位置 | 权威级别 |
| --- | --- | --- | --- |
| `batchId` | 是 | 审计 / 对账 | 权威 |
| `strictPalette.colors[]` | 是 | `ArtBatch.dna.authoritative.strictPalette` | 权威 |
| `alphaPolicy.allowedAlpha[]` | 是 | `ArtBatch.dna.authoritative.alphaPolicy` | 权威 |
| `sourceMetrics.primarySizes[]` | 是 | `ArtBatch.dna.authoritative.sourceMetrics` | 权威 |
| `ppuCandidate` | 否 | `ArtBatch.dna.candidates.ppuCandidate` | 候选 |
| `gridSizeCandidate` | 否 | `ArtBatch.dna.candidates.gridSizeCandidate` | 候选 |
| `borderCandidates[]` | 否 | `ArtBatch.dna.candidates.borderCandidates` | 候选 |
| `rampGroups[]` | 否 | `ArtBatch.dna.candidates.rampGroups` | 候选 |
| `componentStats` | 否 | `ArtBatch.dna.candidates.componentStats` | 候选 |

最小输出示例：

```json
{
  "batchId": "artbatch_ui_frame_20260329_001",
  "strictPalette": {
    "colors": ["#1F140F", "#5E3B23", "#8F6642", "#C9A67A"]
  },
  "alphaPolicy": {
    "allowedAlpha": [0, 255]
  },
  "sourceMetrics": {
    "primarySizes": [
      { "width": 48, "height": 48 }
    ]
  },
  "ppuCandidate": 16,
  "gridSizeCandidate": [16, 16],
  "borderCandidates": [
    { "left": 6, "right": 6, "top": 6, "bottom": 6 }
  ],
  "rampGroups": [
    ["#1F140F", "#5E3B23", "#8F6642", "#C9A67A"]
  ],
  "componentStats": {
    "outerBorderThickness": 2,
    "innerFillMargin": 4
  }
}
```

## 4.3 哪些字段写回 `ArtBatch`

### 写回 `ArtBatch.dna.authoritative`

这些字段一旦写回，就成为本批后续生成链必须 obey 的硬边界：

- `strictPalette`
- `alphaPolicy`
- `sourceMetrics`

### 写回 `ArtBatch.dna.candidates`

这些字段只作为提示或候选值，不冒充最终权威：

- `ppuCandidate`
- `gridSizeCandidate`
- `borderCandidates`
- `rampGroups`
- `componentStats`

## 4.4 明确不冒充最终权威的字段

下面这些字段必须始终按候选处理，不能被 DNA extractor 自己写成最终权威：

1. `borderCandidates`
   - 只给 9-slice 候选，不直接宣布最终 border
2. `ppuCandidate`
   - 只说明观察结果，不覆盖项目标准
3. `gridSizeCandidate`
   - 只说明观察结果，不直接变成硬标准
4. `rampGroups`
   - 仅用于后续辅助审查，不直接变成生成硬约束
5. `componentStats`
   - 仅作为描述性几何统计，不直接代替人工审稿判断

## 5. 三部分如何对齐

这三部分的对齐关系固定如下：

| 设计项 | 输出给谁 | 被谁消费 |
| --- | --- | --- |
| `Gold_Standard` 目录规范 | 参考包路径、命名和批准规则 | `ArtBatch.source.goldStandardRefs` |
| `ArtBatch` schema | 批次生命周期、review / cleanup 路径 | DNA extractor、Unity Review 链 |
| DNA extractor I/O 草案 | 输入字段、输出字段、authoritative / candidate 分层 | 后续 Python DNA extractor 实现 |

简化理解：

- `Gold_Standard` 定义“能吃什么”
- `ArtBatch` 定义“怎么挂一整批”
- DNA I/O 定义“中间数据怎么流”

三者缺一不可。
