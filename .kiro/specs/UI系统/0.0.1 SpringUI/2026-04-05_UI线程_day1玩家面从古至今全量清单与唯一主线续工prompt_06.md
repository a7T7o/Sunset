## 从古至今，你这条线的完整 backlog 清单

下面这些全部都还是你线程 own backlog。  
不是让你一口气乱打一锅，而是明确告诉你：这些都在你账上，不要再误判范围。

### A. 任务栏 / Prompt / continue / 缺字链

1. 开局左侧任务栏必须稳定显示，不允许空壳无字。  
2. 任务栏必须按用户语义只显示：
- `1 个主线`
- `1 个当前小任务`
做完再往后刷，不是整列堆一排。
3. 任务内容超长时，必须自适应或滚动，不允许直接冲出容器。
4. 如果背包里已经有满足条件的物品，例如木头已经 `>=3`，任务出现时就应该立即按已有库存判完成，不是还等玩家再捡。
5. 中间任务卡、`PromptOverlay`、村长对话右下角 `继续`、工作台左列、工作台右侧标题/简介/材料、提示条、按钮文案，都必须进入同一条“缺字链彻底收死”的范围，不允许只修一半。

### B. 工作台正式面

1. 左列 recipe 必须稳定有字，不能再出现“可点但没字”。  
2. 右侧正式面必须按用户语义重新整理：
- 物品名称往上
- 简介往上
- 所需材料区域获得更多空间
- 不允许重叠
- 不允许继续塞一堆多余提示语
3. 删除用户明确说不要的冗余文案：
- `先选择数量再开始制作`
- `正在打造……当前单件……`
只保留真正有价值的进度语义。
4. 数量从 `0` 开始；`1` 不代表“静止等待”，而是进度条已经开始走第一件。
5. 进度条语义必须是：
- 当前单件进度条
- 过完一次完成一件
- 还有剩余数量就重新从 0 走
- 剩余数量常驻显示

### C. 工作台制作链 / 交互矩阵

1. 工作台解锁后就要一直允许制作，不能被后续错误状态重新禁用。
2. `E` 必须是 toggle：
- 打开工作台
- 再按关闭工作台
3. 打开大面板时显示正式工作台 UI；关闭后如果还有在制或可领取产物，则显示悬浮小框。
4. 工作台 UI 和悬浮小框都必须相对工作台稳定，不允许漂、不允许晃。
5. 上下翻面要弹性移动，不是瞬间硬跳。
6. 交互距离与 teaser 范围必须合理，提示位置不能只剩一个孤零零的 `E` 在左边。

### D. 工作台运行时语义

1. 产物不自动进背包，而是留在工作台。  
2. 领取 / 取消的交互应集中到制作条本体，语义必须统一。  
3. 取消后材料返还；背包装不下则掉地。  
4. 制作中允许继续追加数量。  
5. 如果没有选择制作数量，鼠标到条上是中断语义；如果当前条目还可追加，则 hover 要体现追加语义。  
6. 这整套逻辑如果需要继续动：
- `CraftingService.cs`
- `CraftingStationInteractable.cs`
- `SpringDay1ProximityInteractionService.cs`
你可以继续收，但不要漂去 day1 正式剧情链。

### E. 工作台悬浮小框

1. 小框里物品图标尽量占满。  
2. 进度条在下方，跟单件进度增长。  
3. 剩余数量就一个数字，放在条中间或同等清晰位置。  
4. 相同物品追加只叠数量，不重复开卡。  
5. 不同物品分卡显示：
- 一行 3 个
- 最多两行
6. 完成态、制作态、可领取态的颜色/文案语义必须和正式大面统一。
