# 剧情开发初步规划文档

> 目的：帮助你理清"从剧本到实现"的路径，解决"没做过剧情和UI开发，不知道怎么下手"的迷茫。

---

## 一、你现在在哪里

你已经有了：
- Deepseek-2 输出的八篇章宏观剧本（覆盖春夏秋冬四季）
- Claude 的事件编排重构V1（时间压缩+事件密度框架）
- 春1日融合版剧本（时间轴+分镜+对话+UI+代码提示，已确认为标准格式）
- 审核报告（Deepseek 的理解偏差已记录）

你还没有的：
- 春2日及后续天数的剧本
- 对话系统的代码实现
- 事件触发系统的代码实现
- CG/过场动画的实现方案
- UI 教学提示系统

---

## 二、你的方向是否正确

你说"先完成第一天的剧情编排然后直接开始实现"——这个方向是对的，而且是最务实的做法。原因：

1. 剧本写100天再开始实现 = 大量返工。因为实现过程中你会发现"这个镜头做不了"、"这个UI太复杂"、"这个事件触发逻辑不对"，然后要回头改剧本。
2. 先做1天 = 建立管线。第一天的实现会帮你搭建出：对话系统、事件触发系统、UI教学系统、CG播放系统。这些管线搭好后，后续天数就是"填内容"。
3. 边做边写 = 剧本和代码互相校准。你写春2日剧本时，已经知道"对话系统能做什么、不能做什么"，不会写出实现不了的东西。

---

## 三、推荐的工作节奏

不要试图一口气把所有系统都做完。按"纵向切片"来推进：

**第一阶段：春1日完整实现（建立管线）**

这是最重要的阶段，目标是把春1日从剧本变成可玩的游戏内容。完成后你会拥有所有基础管线。

需要搭建的系统（按实现顺序）：

1. 对话系统 — 支持对话框显示、乱码→正常文字转换、NPC头像、对话选项
2. 事件触发系统 — 支持时间触发、位置触发、交互触发、条件触发
3. CG/过场系统 — 支持黑屏淡入淡出、简单图片序列、文字叠加
4. UI教学提示 — 支持按键提示弹出（"按WASD移动"等）
5. NPC跟随/引导 — 支持马库斯带路、玩家跟随

每个系统做到"能跑春1日"就够了，不需要做到完美。

**第二阶段：春2-5日剧本+实现（验证管线）**

用已有管线填充春2-5日的内容。这个阶段会暴露管线的不足，按需补充。同时写春2-5日的融合版剧本。

**第三阶段：春6-25日（批量填充）**

管线成熟后，后续天数主要是"写剧本→填数据→测试"的循环，速度会快很多。

**第四阶段：夏秋冬（按季推进）**

每个季节开头可能需要新增少量系统（比如夏季的钓鱼、秋季的收获节），但大部分管线已经就绪。

## 四、春1日实现的具体拆解

基于融合版剧本，春1日需要实现的内容按场景拆分：

**开场CG（约30秒）**
- 最简方案：黑屏 + 文字描述 + 简单粒子（流星）+ 屏幕震动
- 不需要做精美动画，文字+特效就够了
- 用 Timeline（时间线编辑器）或简单协程控制序列

**14:00 坠落与相遇**
- 需要：对话系统（乱码→正常转换）、移动教学提示、NPC引导移动
- 骷髅兵：放一个静态/简单巡逻的敌人在远处，不需要战斗逻辑

**14:20 初入村庄**
- 需要：NPC气泡对话（村民围观）、场景过渡（黑屏转场）
- 村民：用已有NPC素材，站在固定位置，朝向玩家

**14:30 疗伤与血条UI**
- 需要：血条UI组件、治疗动画（简单粒子）、血量变化补间动画
- 精力条此时不显示

**14:45 工作台与记忆闪回**
- 需要：交互触发（靠近工作台按E）、闪回特效（白闪+图片）、配方解锁逻辑
- 制作系统应该已有基础，只需要触发解锁

**15:10 耕种教学**
- 需要：精力条UI组件（此时才显示）、教学步骤提示
- 农田系统已有，只需要接入教学引导

**16:00 晚餐与冲突**
- 需要：场景切换到饭馆、对话系统（多NPC对话）、精力恢复逻辑

**17:00 归途**
- 需要：时间系统（天色变化）、睡觉交互、日期推进

## 五、你不需要担心的事情

1. 剧本不需要一次写完 — 写一天做一天，边做边调整
2. CG不需要很精美 — 黑屏+文字+简单特效就是合格的独立游戏CG
3. NPC不需要复杂AI — 第一天的NPC只需要站在固定位置、说固定台词
4. 对话系统不需要很复杂 — 先做最基础的"显示文字+下一句"，后续再加分支
5. 28个NPC不需要同时做 — 春1日只需要马库斯、艾拉、卡尔、2-3个路人村民

---

## 六、建议的下一步

1. 审阅春1日融合版剧本，确认内容无误
2. 开始搭建对话系统（这是所有剧情实现的基础）
3. 搭建事件触发系统（控制"什么时候触发什么"）
4. 用这两个系统实现春1日的第一个场景（14:00 坠落与相遇）
5. 验证可行后，继续实现剩余场景
6. 春1日全部可玩后，写春2日剧本并实现

## 七、已确定的设计决策（不可更改）

这些是你在和 Deepseek 的对话中明确确认的，后续所有剧本和实现都必须遵守：

- 剧本格式：融合版（时间轴+分镜+对话+UI+代码提示）
- 坠落地点：矿洞口（后续地牢入口）
- 遭遇怪物：骷髅兵（徘徊，未发现玩家）
- 语言转换：乱码→正常中文（一次性转换）
- UI顺序：血条（治疗时）→ 精力条（耕地时）
- 第一天作物：花菜
- 房屋：废弃闲置房
- 村落技术：全村统一基础水平
- 冬季：不种作物（靠存粮+钓鱼）
- 地牢：无限层，自动生成（RuleTile）
- 作物总数：现有8种，不新增
