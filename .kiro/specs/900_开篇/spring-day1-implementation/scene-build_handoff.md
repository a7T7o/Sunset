## 4. `SceneBuild_01` 的正式身份

### 4.1 它是什么
- `SceneBuild_01` 是春1日 `14:20 进入村庄` 之后，到 `15:10 耕种 / 砍树教学` 前后的主承载场景。
- 它的正式叙事身份是：**“废弃小屋 + 院落 + 小块农田 + 教学承接”的生活启动场景**。
- 它要承接的不是纯环境展示，而是“被带到住处 → 屋内治疗后出屋 → 工作台闪回 → 院内站位对话 → 农田教学 → 砍树教学 → 回屋衔接”。

### 4.2 它不是什么
- 不是整座落日村总图
- 不是矿洞口 / 骷髅兵威胁区
- 不是晚餐 / 饭馆 / 冲突区
- 不是纯美术试摆场
- 不是为了替 UI、字幕、对话实现兜问题的临时验证场

## 5. `SceneBuild_01` 的强制承载动作

### 5.1 动线与入口
- 东侧进入：玩家与 NPC 要能从东侧被引入这张场景。
- NPC 带入：进入后要有明确“被带到住处”的主导动线，而不是一进来就散。

### 5.2 院落核心
- 院落中心站位对话：必须有一个可读、留白、适合对话和镜头停驻的中心区。
- 屋前生活区：建筑右前或侧前区域应承担“安置感”和“有人生活过”的视觉逻辑，但不能抢走院心焦点。

### 5.3 关键交互点
- 工作台交互与闪回：工作台必须是明确可达、可读、可停驻的焦点点位，不应埋在散物中。
- 回屋 / 室内衔接：主场景必须明确“屋外 → 屋内治疗 / 床铺”的空间关系，哪怕当前先做最小版本。

### 5.4 教学落点
- 农田教学落点：必须是清晰、可读、便于教学镜头和站位的区域，不能只是抽象地块。
- 砍树教学落点：应与农田区和院落保持邻接关系，但不要侵占院落核心对话区。

## 6. `SceneBuild_01` 的禁止误扩边界
- 不要继续扩成整村总览图
- 不要把矿洞口骷髅兵威胁段塞进来
- 不要把饭馆 / 晚餐冲突强行塞进来
- 不要为了“更丰富”把院落中心和入口动线堆满
- 不要把它做成“好看但不服务剧情动作”的环境孤岛

## 7. 给 `scene-build` 的精修优先级
1. 先收主入口动线：东侧进入、NPC 带路、到住处的抵达感
2. 再收院落留白与视线焦点：院心站位、工作台焦点、生活区不抢戏
3. 再收工作台与教学区关系：从住处到工作台、到农田、到砍树点的顺序感
4. 再收室内外衔接：治疗 / 床铺语义与屋外生活启动的关系
5. 最后才是泛装饰扩张：树石、小物、边角氛围等

## 8. 给 `scene-build` 的落地提示

### 8.1 关键站位
- 入口抵达站位：用于 NPC 把玩家带进来后的第一停点
- 院落核心站位：用于住处介绍与院内对话
- 工作台交互站位：用于闪回触发
- 教学观察站位：用于 NPC 指向农田 / 树木教学

### 8.2 入口 / 出口
- 主入口明确保持东侧语义
- 屋内入口要能支撑“治疗后出屋 / 晚些时候回屋 / 睡觉”这条叙事链
- 教学区出入口应可读，但不需要现在就扩成开放大地图

### 8.3 互动点
- 工作台
- 床铺 / 室内休息点
- 农田教学点
- 砍树教学点

### 8.4 需要预留的空间
- 对话停驻空间
- 教学演示空间
- 触发区与镜头停留空间
- 后续室内外切换 / 回屋衔接空间
