## 1. 用户语义真值

这部分以用户最新明确裁定为准，后续代码必须服从这里，不允许再被旧 prompt、旧文档、旧测试带偏。

## 1.1 总体原则

1. `Day1` 只负责剧情语义、时序、staged contract、release intent。
2. `NPC` 负责 resident 自己的状态切换。
3. `导航` 负责所有真实位移执行。
4. 只要 NPC 发生真实移动，原则上都应走导航。
5. 允许的 snap 例外只有：
   - staged 超时对位
   - load / repair 对位
   - authored 点位失效时的明确兜底
6. `001/002` 只在剧情段是特殊角色。
7. `003` 在剧情外就是普通 NPC。
8. 所有 NPC，白天都不该因为 Day1 私房逻辑回 anchor / 回 home。
9. 所有 NPC 统一在 `20:00` 之后才开始回家。

## 1.2 Opening 真值

`opening` 的正确语义只有一套：

1. `001~203` 全员都先直接传到各自起点。
2. 再统一走一次“起点 -> 终点”。
3. 最多等 `5` 秒。
4. 超时就直接 snap 到终点。
5. 对话开始后不再有任何二次走位。
6. 不允许 `001~003` 和 `101~203` 吃两套不同 owner。

## 1.3 Opening 结束后的真值

1. `001/002` 继续吃自己的 Day1 剧情合同。
2. `003` 与 `101~203` 一样，回到普通 resident。
3. ordinary resident 不需要“先回 anchor 再 roam”。
4. opening 结束后到 `20:00` 之前，ordinary resident 直接原地解散，恢复全图自由活动。
5. 不允许因为 Day1 runtime 再把 ordinary resident 传回 baseline / anchor / home。

## 1.4 Primary 真值

1. `Primary` 期间时间应正常流逝。
2. 在 `0.0.5` 任务全部完成前，时间最晚只能推进到 `16:00`，此时才允许锁住。
3. `靠近艾拉触发回血` 只需要改一个数值：
   - `1.6`
4. 触发判定应以艾拉为中心的圆范围计算，不要求玩家必须面朝艾拉。
5. 触发回血后，允许艾拉做最小朝向玩家处理。
6. 这里不要碰 NPC 的 `E` 交互。
7. `primary` 未完成前，不能自由去 `Town`，这一条保持不变。

## 1.5 0.0.6 回 Town 真值

1. 玩家在完成 `Primary` 后回到 `Town`，ordinary resident 必须是正常自由活动。
2. `001/002` 若此时需要为晚饭或后续剧情做准备，只允许吃明确剧情合同。
3. 不允许出现：
   - 群体原地卡死
   - 一秒一帧
   - 聊一下天才恢复正常
4. 这段如果有位移恢复，必须通过正确 owner 和正确导航合同完成。

## 1.6 Dinner 真值

`dinner` 现在采用最简、最硬的同构语义：

1. 晚饭从 `18:00` 开始。
2. `001~203` 全员都视为普通 dinner 参与 NPC。
3. 全员都有自己手摆的起点和终点。
4. 晚饭入口只允许：
   - 直接传到起点
   - 走一次到终点
   - 最多等 `5` 秒
   - 超时 snap 到终点
5. 从 `18:00` 到 `19:30` 整个剧情窗口内，不允许再有任何额外走位。
6. `19:00` 的提醒段仍属于这整段晚饭剧情窗口，不是放人时刻。
7. `19:30` 之前，NPC 不应该进入普通自由漫游。

## 1.7 19:30 / 20:00 / 21:00 / 次日 真值

### `19:30`

1. 晚饭剧情结束。

## 5. 施工总原则与红线

## 5.1 总原则

1. 先语义真值，后代码手段。
2. 先清 owner，后调参数。
3. 先清错误 runtime 合同，后谈体验优化。
4. `Day1` 只收 Day1 自己的错 owner，不继续深碰 NPC 内核。
5. 所有结论都必须回到代码和真实体验，不允许靠长文档自嗨。

## 5.2 红线

1. 不再把结构修复包装成体验过线。
2. 不再让 `003` 保留特殊 runtime。
3. 不再用 `Town_Day1Residents / Town_Day1Carriers / EnterVillageCrowdRoot` 这类脏中间层回流。
4. 不再接受 ordinary resident 在白天被 Day1 私房逻辑传回 baseline / anchor / home。
5. 不再接受 dinner/opening 有二次走位。
6. 不再继续保留全 NPC 的夜间 split contract。
7. 不允许靠隐藏 Town、冻结 Town、屏蔽玩家回 Town 来掩盖 Day1 runtime 问题。

## 5.3 完成定义

Day1 真正过线，至少要同时满足：

1. `opening` 正确
2. `primary` 时间正确
3. `0.0.6` 回 Town 不再卡爆
4. `dinner 18:00~19:30` 正确
5. `19:30` 放人正确
6. `20:00/21:00/次日` 夜间合同正确
7. `001/002/003/101~203` 身份与行为符合语义

少任何一条，都不能说“Day1 已彻底完成”。
