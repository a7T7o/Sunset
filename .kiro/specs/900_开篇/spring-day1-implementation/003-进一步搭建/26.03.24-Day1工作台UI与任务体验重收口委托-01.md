## 你这轮真正要完成的事

### 1. 工作台 UI 必须重做到真正可验收
- 左侧必须是真正的可选制作列表，不允许再出现“空框”。
- 左侧至少明确承载：斧头、镐子、锄头，以及对应名字。
- 右侧比例必须明显收紧，不允许继续喧宾夺主。
- `+ / -` 必须显著缩小，不能再像大按钮占版。
- 图标、描述、留白、字号、层级关系都要重新收拾，不能维持当前测试味布局。
- 背景透明度、悬浮感、上下切换过渡可以做，但必须克制，不要为了动而动。

### 2. 第一优先级里必须包含“用户可手动精调”
- 这不是附带项，而是硬要求。
- 你必须告诉用户：
  - 为什么当前这些 UI 无法拖动 / 精调。
  - 是运行时动态生成、`LayoutGroup` / `ContentSizeFitter` / 锚点驱动、还是别的机制把它锁死了。
  - 你改了什么之后，用户现在可以在什么对象上、通过什么方式手动拖动或微调。
- 不允许只把结果做成“你代码里能调，但用户手上还是不能拖”。

### 3. Day1 任务检查必须补成严格约束
- 砍树拿木材不能再被玩家在别的任务流程里提前旁路完成。
- 你必须把任务完成判定改成真正符合 Day1 教学顺序的机制，不允许继续靠“背包里正好有木头”这种松判定混过去。
- 任务 UI 也要一起重做，不准继续用粗糙、大块、测试味的表现。

### 4. 血量条与精力条这轮要按新审美重做
- 用户明确允许你先停用旧的那套。
- 这轮不是“修修旧条子”，而是你自己重新给出一套更像游戏正式表现的版本。
- 需要和 Day1 当前整体观感一致，不要像调试条。

### 5. 工作台制作过程必须有真正的“做东西”感
- 你要把工作台动画、制作耗时、进度条、UI 表现、人物干活状态整合成一个流程。
- 用户已经明确给了工作台动画资源路径：`Assets/ZZZ_999_Package/Pixel Crawler/Environment/Structures/Stations/Anvil/Anvil_01.aseprite`
- 人物表现可以创新，但不能偷懒复用一套明显不合适的现有动作然后硬说完成。
- 这部分必须和工作台交互流程整成一条体验链，不是单独飘一个进度条了事。

### 6. 距离判断、自动关闭、上下翻转、提示气泡，这四项必须一起收口
- 打开距离不再允许继续用“中心点到中心点 0.5m”这种粗糙算法顶替。
- 你必须认真实现或至少严格逼近用户要的判定：
  - 以玩家 Collider 中心到工作台中心连线
  - 求该连线与工作台 Collider 边界的交点
  - 用这个边界距离作为交互触发与关闭判断的重要依据
- 关闭阈值 `1.5m` 也必须和玩家肉眼观感一致，不能继续“看起来没走远却突然关了”。
- 上下翻转不能只靠粗糙的 `y` 值想当然比较；工作台是不规则、双 Collider、`PolygonCollider2D`，你必须给出一个稳的方法。
- 首次使用的 `E` 气泡提示必须只出现一次，而且样式要像游戏里真的会有的引导气泡，不是开发提示框。
