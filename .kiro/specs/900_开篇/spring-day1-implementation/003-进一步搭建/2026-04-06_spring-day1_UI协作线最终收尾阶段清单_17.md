## 第一部分：把 Workbench 彻底收成玩家可用、可读、可判定的正式面

### 当前主战场

`Workbench` 仍是当前 UI 最核心未闭环项。

### 当前必须收穿的点

1. 左列 recipe generated row 的真实显示链
   - icon
   - 名称
   - 摘要
   - 高亮状态
2. 右侧详情区
   - 名称
   - 描述
   - 材料
3. 制作条与制作按钮语义
   - 开始 / 追加
   - 领取 / 取消
   - hover / click 提示
4. 悬浮信息与进度提示
5. 面板与工作台的相对稳定表现
6. 上下移动 / 切换时的体感，不要再硬跳或乱飘

### 完成定义

不是“代码像是对了”，而是：

1. 玩家一眼能看清左列和右列
2. 当前能做什么、能领什么、能取消什么，一眼可判
3. 没有再出现“有壳无字 / 能点但像死面板”的假活状态
