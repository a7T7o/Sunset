## 你这轮唯一主刀

继续只做玩家面 `UI/UE` 收口，不接剧情 owner。

重点按用户最新裁定收下面这些：

### A. Prompt / 任务栏 / 正规对话文字问题

1. 开场左侧任务栏不该空。
2. 任务显示应维持：
   - 一次一个主任务
   - 一个小任务
3. 内容超出时要能：
   - 自适应
   - 或滚动
   不能直接溢出。
4. 仍有部分任务列表文字缺失。
5. 村长正规对话框右下角“继续对话”提示文字仍会空白。

### B. Workbench 左栏与正式面排版

1. 工作台左侧 recipe 栏仍可能空白但可点击。
2. `先选择数量再开始制作` 这类文案可以删。
3. `正在打造……当前单件……` 这类冗长提示也删。
4. 进度与剩余数量要常驻显示，不是 hover 才显示。
5. 物品名称、简介、材料区整体要上移，但绝不能和图标或其他区块重叠。

### C. Workbench 制作条与交互态

用户当前语义固定为：

1. 制作条本体就是单件进度条。
2. 有背景条，进度随当前单件增长。
3. `数量=0` 且制作中时：
   - hover 制作条 = 中断制作态
4. `数量>0` 且制作中时：
   - hover 制作条 = 追加制作态
5. 取消逻辑必须闭环：
   - 材料返还
   - 背包满则溢出掉地上

### D. Workbench 悬浮框

用户当前语义固定为：

1. 物品图标尽量占满方框主体。
2. 进度条在下方。
3. 中间只保留一个数字/简洁状态，不要再堆 verbose 文案。
4. 做完一个就从 0 重新开始下一件。
5. 多个不同物品并行时：
   - 一行 3 个
   - 最多两行
6. 相同物品追加制作只叠数量，不重复开新框。

### E. Workbench 打开/关闭与静止锚定

1. 工作台一旦正式激活，就应一直允许打开。
2. `E` 是大 UI 的 toggle。
3. 大 UI 关闭时，如果有制作中的内容，显示悬浮小框。
4. 大 UI 和悬浮小框都必须和工作台保持相对静止。
5. 上下翻面保留弹性，但常态不能漂。
