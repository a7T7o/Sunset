## 6. 对 UI 的稳定合同

为了不把 UI 正在做的内容再次带偏，这刀必须守住下面 6 条：

### 6.1 保留 9 个大 phase 外壳

对外仍保留：

1. `CrashAndMeet`
2. `EnterVillage`
3. `HealingAndHP`
4. `WorkbenchFlashback`
5. `FarmingTutorial`
6. `DinnerConflict`
7. `ReturnAndReminder`
8. `FreeTime`
9. `DayEnd`

### 6.2 优先做加法，不做破坏性改名

如果必须加剧情源，优先：

1. 新增对白资产
2. 新增 phase 内部 beat
3. 新增最小状态字段

不要先：

1. 改现有外层 phase 名
2. 删掉 UI 正在读取的旧入口
3. 把旧字段直接改名导致 UI 读空

### 6.3 不用 UI 文件来补剧情

剧情补回必须发生在剧情源本身，不准借：

1. `PromptOverlay`
2. `Dialogue UI`
3. `WorkbenchOverlay`
4. 左下角提示卡

去“假装剧情补回了”。

### 6.4 如果剧情源合同变化，必须同步文档

如果这刀不只是改字，而是改了 `UI` 正在读取的稳定合同，本轮必须同轮更新：

1. `2026-04-04_UI线程_剧情源协同开发提醒_03.md`
2. 当前子工作区 `memory.md`

不能只改代码，不补协同说明。

### 6.5 把“字会变长”视为常态

这刀默认会让：

1. 早期任务文案更长
2. 村长早期正式对话更多
3. Prompt/Focus/Progress 更细

所以任何实现都不应默认这些字还会保持今天的长度。

### 6.6 不碰玩家面体验 owner

这刀不承担：

1. 对话框 continue 提示为什么空
2. 任务栏为什么溢出
3. Workbench 左栏为什么无字
4. 悬浮框为什么排版不好
5. 左下角提示怎么长才像正式游戏

这些继续都归 `UI`。

## 8. 正式剧情约束

这刀必须守住下面这些原案硬约束：

1. 玩家是在矿洞口附近醒来，不是直接在村里开始
2. 村长最初说话先是语言错位，之后才逐渐能听懂
3. 矿洞附近必须有怪物威胁感
4. 村长必须承担撤离带路人
5. 进入村庄后要有围观感
6. 被安置的是闲置废屋，不是大儿子的房子
7. 正式主承载仍是：
   - 马库斯
   - 艾拉
   - 卡尔
8. `101~301` 仍只当匿名 crowd / 过桥壳
