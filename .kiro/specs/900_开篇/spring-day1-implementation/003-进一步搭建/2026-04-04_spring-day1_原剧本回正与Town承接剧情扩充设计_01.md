# spring-day1 原剧本回正与 Town 承接剧情扩充设计

## 1. 这份文档解决什么问题

当前 `spring-day1` 的 Day1 主链已经有“首段对话 -> 疗伤 -> 工作台 -> 耕种 -> 晚餐 -> 回屋 -> 睡觉”的最小跑通链，但它和最初春一日原案之间还有 4 个明显落差：

1. 开场缺了“矿洞口危险感 + 怪物逼近 + 跟随村长撤离”的前半段。
2. 进村后缺了“村民围观、异乡人被注视、被安置到闲置小屋”的承接。
3. 当前很多 NPC 是在流程推进后突然冒出来，不像一个原本就在运转的村庄。
4. 当前 crowd 资源里有一批后来追加的名字与设定，但它们不能直接当成春一日原案正式角色。

这轮不碰 UI，不回头重做已经能跑的玩家面结果层，只做三件事：

1. 把最初春一日 Day1 原案重新钉实。
2. 把当前实现和原案之间的缺口说清楚。
3. 给出一份“在现有实现基础上继续扩充”的剧情走向、剧情脚本和 Town 承接方案。

## 2. 已查实现状

### 2.1 当前代码真实主链

当前 `SpringDay1Director` 的正式阶段只有 9 段：

1. `CrashAndMeet`
2. `EnterVillage`
3. `HealingAndHP`
4. `WorkbenchFlashback`
5. `FarmingTutorial`
6. `DinnerConflict`
7. `ReturnAndReminder`
8. `FreeTime`
9. `DayEnd`

当前真正落成资产的正式对话，只有两份：

1. `SpringDay1_FirstDialogue.asset`
2. `SpringDay1_FirstDialogue_Followup.asset`

疗伤、工作台、晚餐、归途提醒这 4 段，现在仍主要是 `SpringDay1Director` 在运行时临时组装出来的对白序列，不是完整的原案分场剧情包。

### 2.2 当前最稳的现有骨架

当前可以直接保留、不需要推翻的骨架有 5 条：

1. `矿洞口首遇 -> 语言乱码 -> 解码 -> 跟村长走` 这条正式开场已经成立。
2. `HP 在疗伤时出现` 这条语义已经成立。
3. `工作台闪回 -> 基础制作` 这条功能骨架已经成立。
4. `耕种 / 木材 / 制作 / 晚餐 / 夜间睡觉规则` 的流程闸门已经成立。
5. `FreeTime -> 两点前必须睡` 的压力链已经成立。

### 2.3 当前最需要纠偏的点

当前最需要纠偏的不是“完全重做 Day1”，而是以下 6 点：

1. 把 `CrashAndMeet` 内部补成“醒来 -> 危险 -> 跟随撤离”，而不是只剩首段对话。
2. 把 `EnterVillage` 内部补成“进村 -> 围观 -> 安置到闲置小屋 -> 交给艾拉”。
3. 让 `HealingAndHP` 真正承担“被村庄接住”的情绪段，而不只是一个功能播报点。
4. 让工作台闪回和“这个村子当前的技术水平”连起来，而不是只剩一句自我回忆。
5. 让晚餐、回屋和夜间规则有村庄氛围与人物关系，而不是纯功能过桥。
6. 把 NPC 的出现位置收回到 `Town` 语义，temp scene 只保留必要过场。

## 3. 原剧本硬约束

以下是这轮必须守住的原案硬约束：

### 3.1 开场硬约束

1. 主角是在 **矿洞口附近坠落并醒来**，不是直接在村里开始。
2. 村长最初说话时，玩家看到的是 **乱码**，之后才逐渐听懂。
3. 这里的语言差异是“玩家先听不懂，回忆后逐渐能听懂”，不是要把文本做成文言文。
4. 矿洞附近有 **魔物 / 骷髅 / 怪物威胁**，村长必须带着玩家先离开危险区域。
5. 跟随村长离开这段，要承担 **移动 / 奔跑教学**。

### 3.2 进村硬约束

1. 进入村子后，村民会对主角产生明显围观和警惕。
2. 村长带主角去的是 **闲置多年、没人住、没有打扫的废屋**，不是大儿子的房子。
3. 村长的女儿 **艾拉** 会来疗伤，她是 Day1 的正式关键角色。
4. **血条只在疗伤时出现**，此时精力条还不该出现。

### 3.3 工作台与教学硬约束

1. 小屋里的工作台是 **村里现有技术水平里最好的基础工作台**，但整体仍然很落后。
2. 玩家触摸工作台后触发 **回闪 / 回忆起基础配方**，而不是听别人夸外面的技术更先进。
3. 工作台的意义是把玩家从“受伤外来者”接到“能开始自己活下去的人”。
4. **精力条要等到开始耕地后才出现**，不是和血条同一拍出现。
5. Day1 的基础种子是 **花菜种子**，不是大蒜。

### 3.4 晚间硬约束

1. 晚餐事件要引出 **卡尔** 的敌意和后续冲突伏笔。
2. 村长会嘱咐玩家 **两点前必须睡觉**，这是这片土地的夜间规则。
3. Day1 结束前，玩家应该已经感到：
   - 自己被村庄暂时接纳了
   - 但并没有真正被完全信任
   - 且这个地方夜里有不对劲的东西

### 3.5 角色承载硬约束

1. Day1 正式主承载角色，仍以 **马库斯 / 艾拉 / 卡尔** 为主。
2. **老杰克 / 老乔治 / 老汤姆 / 小米 / 围观村民 / 饭馆村民 / 小孩** 是原案 Day1 村庄肌理的一部分。
3. `101~301` 这一批后补 crowd 槽位，当前只能视为 **临时 crowd 外壳或次级背景位**，不能直接当作原案正式人设真名。
4. 后续 NPC 的主要出现面应在 `Town`，不是长期卡在 `Primary` 这条 temp scene 上。
5. 只有村长在 Day1 前半段承担“从矿洞到村庄、从安置到教学”的跟随段。

## 5.2 各段具体语义

### 段 1：矿洞口醒来

必须保留的体验：

1. 玩家醒来时先看不懂村长的话。
2. 这是“自己掉进了一个根本不认识的世界”，不是“在村里慢悠悠聊天”。
3. 村长此时是救助者，不是教程提示器。

推荐脚本口径：

- 马库斯第一句仍先以乱码呈现。
- 主角内心独白指出“这些音节很像我学过的古语，但我一时抓不住”。
- 第二拍以后，玩家才逐渐能把马库斯的话听成正常中文。

### 段 2：怪物逼近

这一段的职责不是打斗，而是建立：

1. 矿洞附近真的危险。
2. 村长知道怎么活着离开这里。
3. 玩家第一天还不是英雄，只能先跟着跑。

推荐脚本口径：

- 马库斯先看向矿洞口远处，表情一变。
- 远景给出骷髅 / 怪物晃动的威胁感。
- 马库斯只说一句“先走，别在这里停”，不在这里讲太多世界观。

### 段 3：林间撤离

这一段承担 3 个功能：

1. 移动教学
2. 奔跑教学
3. 把“矿洞危险区”切换到“村庄安全区”

这里不需要再塞对话密度，只要：

1. 马库斯在前
2. 玩家在后
3. 怪物的威胁在身后
4. 到达安全路径后黑屏转场进村

### 段 4：进村围观

这里要补的是“村庄不是空壳”。

推荐处理：

1. 玩家跟着马库斯走进村口时，至少出现一小组围观村民和一个小孩视线。
2. 村民不必长谈，但一定要让玩家感到“大家都在看我”。
3. 这段只需要营造异样感，不抢艾拉和后面疗伤的戏。

### 段 5：闲置小屋安置

这段要明确 3 件事：

1. 屋子是空置多年的闲置屋，不是某个家庭成员的专属房。
2. 村长暂时给了你一个落脚点，但不是完全接纳。
3. 艾拉会在这里正式接住你。

推荐脚本口径：

- 马库斯：`这屋子空了很多年，先收拾收拾，你今晚就先住这儿。`
- 马库斯：`人先稳下来，别的事后面再说。`

### 段 6：艾拉疗伤

这段是 Day1 第一次真正的“柔软段”。

必须保证：

1. HP 只在这时出现。
2. 艾拉的态度是谨慎中的善意，不是无条件亲近。
3. 玩家会第一次感觉“自己不是马上就要死在这儿了”。

推荐脚本口径：

- 艾拉先检查伤口，再使用治疗魔法。
- HP 条升起并常驻。
- 艾拉对主角的服饰和来历感到好奇，但会因为主角的怪异话语而后退半步。

### 段 7：工作台闪回

这段不能只是“我会做东西”，而要补上语义：

1. 这是这个村子当前最好的基础工作台。
2. 玩家对这种基础工具有熟悉感。
3. 这不是炫技，而是“我也许能在这里活下去”的第一步。

推荐脚本口径：

- 马库斯：`村里能用的东西都在这儿了。够不够好不好说，但先活下来总是够的。`
- 玩家触碰工作台后闪回。
- 玩家低声说出一串自己也没完全想明白的材料 / 配方碎片。
- 艾拉被吓到，往村长身后缩一点。
- 村长打断：`先别想太多，先把手边最简单的东西做出来。`

### 段 8：耕种教学

这段已经有现成功能骨架，但语义上要补 4 件事：

1. 花菜种子是 Day1 的正解。
2. 精力条要在第一次真正开垦后再升起。
3. 村长是在教“怎么自己养活自己”，不是在刷系统流程。
4. 木材与基础制作，是为了让玩家理解“村里现在就靠这些活着”。

推荐口径：

- 马库斯先给花菜种子。
- 第一次开垦后，才让 EP 出现。
- 木材 / 基础制作应和“这村子当前的基础劳动结构”连起来。
- 老乔治可以被提及为“村里最懂这些的人”，但 Day1 不强制出场。

### 段 9：晚餐冲突

这段是 Day1 的第一处对人不对事的冲突。

必须做到：

1. 卡尔首次出场就带敌意。
2. 他的敌意来自对外来者的不信任，而不是无缘无故发疯。
3. 马库斯能暂时压住场面，但压不掉后患。

推荐口径：

- 饭馆里是简陋但真实有人气的村庄空间。
- 卡尔质疑主角为什么能吃这顿饭、为什么能被带回村里。
- 马库斯压下冲突，但主角不会因此真正安心。

### 段 10：回屋提醒

这段要承担：

1. 白天结束
2. 夜晚规则建立
3. 玩家第一次对“这个地方夜里不对劲”产生明确印象

推荐口径：

- 马库斯送到小屋前或回屋路上提醒：两点之前必须睡。
- 不是威胁式讲述，而是村里人真的怕这条规则失效。

### 段 11：自由活动与睡觉

这段不需要塞满任务，但要让玩家感受到：

1. 村子夜里还活着
2. 但大家都在收口
3. 你现在只是暂住者

因此这段的重点不在再开新大剧情，而在：

1. 允许玩家看见 Town 的 3-4 个夜间存在感
2. 再由回屋与睡觉把 Day1 收住

## 6. Day1 NPC 出场与走位方案

## 6.1 核心原则

1. **村长是唯一跟着玩家跑完整个前半段的人。**
2. 其他 NPC 原则上都应该在 `Town` 里“本来就在那儿”。
3. temp scene 只保留必须发生在当前链条里的少量承载点。

## 6.2 角色走位与时机

| 角色 | 首次出现 | Day1 主要活动区域 | Day1 是否强制交互 | 说明 |
| --- | --- | --- | --- | --- |
| 马库斯 | 矿洞口 | 矿洞口 -> 林路 -> 村口 -> 小屋 -> 农地 -> 饭馆 -> 回屋路 | 是 | Day1 前半段唯一持续跟随者 |
| 艾拉 | 小屋疗伤段 | 小屋 -> 之后回到村长家/教堂/村内安全点 | 疗伤段强制，之后可选 | 她是“被村庄接住”的第一层情感承载 |
| 卡尔 | 饭馆冲突段 | 饭馆 -> 夜间离场 | 是 | 只需短时高压出现即可 |
| 老杰克 | 进村远景或自由时段 | 农田区 | 否 | 应作为 Town 生活感的一部分，后续再接主线 |
| 老乔治 | 工作台被提及时或自由时段 | 铁匠铺外 / 工作间附近 | 否 | Day1 可先做“被提及 + 可远看” |
| 老汤姆 | 傍晚或自由时段 | 码头 / 村边水域 | 否 | 承担村边缘感和暮色气氛 |
| 小米 | 进村围观 / 自由时段 | 村口树下 / 路边 | 否 | 承担“孩子看着外来者”的视线 |
| 围观村民 | 进村 | 村口 / 主路 | 否 | 必须有，但不必做复杂剧情 |
| 饭馆村民 | 晚餐 | 饭馆 | 否 | 只为让冲突不是空房间争吵 |

## 6.3 当前阶段的 temp scene 保留口径

在 `Town` 场景未完整接管前，当前 temp 承载最多只保留这些内容：

1. 矿洞口首遇
2. 跟随撤离
3. 村口转场入口
4. 小屋与工作台
5. 小屋外农地
6. 饭馆事件
7. 回屋与睡觉

也就是说：

1. **村长必须留在 temp 链里。**
2. **艾拉必须留在 temp 链里。**
3. **卡尔至少要在晚餐时短时进入 temp 链。**
4. 其余 NPC 尽量作为 `Town` 语义内容，不再在 temp 里长期驻留。

## 7. temp scene 与 Town 的承接边界

## 7.1 temp scene 负责什么

temp scene 只负责 Day1 的“受伤、被带走、被安置、被教会第一天怎么活下来”。

它负责：

1. 首段危险
2. 安置与疗伤
3. 工作台闪回
4. 基础生存教学
5. 晚餐冲突
6. 夜间规则
7. 睡觉收尾

## 7.2 Town 负责什么

Town 负责“真正的村庄存在感”和“后续 NPC 的主要生活面”。

它负责：

1. 村民平时各自站在哪里
2. 正式 NPC 的日常活动点
3. 非强制对话与群像氛围
4. 未来 Day2 以后继续扩展的主承载

## 7.3 当前过渡策略

当前在 `Town` 未完全可用前，采用下面的过渡策略：

1. Day1 正式剧情仍先在 temp 链里跑通。
2. 只把必须在 Day1 内见到的人拉进 temp 承载。
3. 其余角色先用“进村远景 / 自由时段远景 / 被提及”来保存在语义里。
4. 等 `Town` 到位后，再把这些角色的主要出现面整体迁过去。

## 8. 给后续实现的框架结论

这轮不直接写实现，但后续框架必须遵守下面 6 条：

1. 不推翻当前 `StoryPhase` 主链，只在每个大阶段内部补“更细的剧情步”。
2. 把运行时临时组装的关键对白，逐步改成可审可改的正式剧情资产。
3. 马库斯 / 艾拉 / 卡尔先守住正式主承载，别再被 crowd 壳稀释掉。
4. `101~301` 在重新映射前，只能做匿名 crowd 或过桥位。
5. Day1 扩充优先做 **前半段补齐 + Town 入口语义 + 夜间收束氛围**，不先扩 Day2。
6. UI 线程只负责“怎么显示”，这里负责“到底发生什么、什么时候发生、谁说话、谁出现”。

## 9. 本轮结论

当前最合理的方向不是推翻现有 Day1，而是：

1. 保留已经跑通的 9 段主链骨架。
2. 把原案中缺掉的“矿洞危险、撤离、进村围观、闲置小屋安置、村庄情绪层”补回。
3. 把 NPC 的主出现面明确交还给 `Town`。
4. 把 `spring-day1` 后续实现收束成“非 UI 的剧情与阶段承载”，等 UI 收口后再对齐显示层。
