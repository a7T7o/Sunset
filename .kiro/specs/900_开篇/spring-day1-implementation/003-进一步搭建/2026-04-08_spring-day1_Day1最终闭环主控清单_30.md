## 我自己这轮必须收掉的部分

### 第一部分：Town 开场到进 `Primary`

1. 游戏从 `Town` 开场，不再默认沿旧的 `Primary` 开场路线
2. `001` 村长是引路主角，`002` 艾拉必须跟着一起走，不允许掉队
3. 村长的引路必须具备真实等待逻辑：
   - 玩家落后太多就停
   - 停下后给出明确提示
   - 玩家跟上再继续
4. 围观群众必须吃用户已经在场景里摆好的点位，不允许再从很远的 home 出生点硬跑进来冒充“排练”
5. 走位期间的对话时机要和移动完成点对齐，不能边乱走边开关键对白

### 第二部分：`Primary` 主链闭环

1. `001/002` 在 `Primary` 的剧情控制期间，必须关掉漫游、自动打招呼、附近闲聊等会抢戏的逻辑
2. 治疗段改成“玩家走到艾拉身边触发”，不再要求艾拉硬挤到玩家身上
3. 村长带玩家去工作台时，必须先走到位，再开桥接对白
4. 工作台交互前新增一段强制对白：
   - 村长说明旁边箱子里给玩家准备了东西
   - 只有这段对白发生后，工作台交互和后续推进才允许打开
5. `Primary` 阶段结束后：
   - `001/002` 不再继续作为 `Primary` 常驻剧情入口
   - 后续主链回到 `Town`

### 第三部分：回村、晚饭、自由活动、睡觉

1. 回村晚饭段的围观演出直接复用第一次进村的那套导演语言
2. 回村后的正式段仍然要守住：
   - 晚饭 formal
   - 回屋提醒 formal
   - 自由活动 opening formal
3. 自由活动后必须能在 `Town` 里通过床进入睡觉收尾
4. 睡觉后要把 `DayEnd` 闭环，而不是停在“还能逛”

### 第四部分：one-shot 和原生 resident 口径

2. 已完成任务绝不允许再次伪装成正式推进入口

4. 不再把 `runtime resident` 当成默认世界观
5. 代码只能消费用户手摆的 scene 点位和 resident，不偷改位置

## 这轮要发给别人的协作方向

### 当前卡顿真值先钉死

根据用户新补的 profiler 截图，这轮启动卡顿的主因已经不是泛 UI：

1. 第一大峰值是 `PersistentPlayerSceneBridge.Start()` / `RebindScene(...)`
2. 第二大峰值是 `NavGrid2D.RebuildGrid()` 的整图重建
3. `SpringDay1PromptOverlay / TMP` 有成本，但不是这次最毁 demo 的主因

所以这一轮 prompt 分工必须跟着改，不能再把“启动卡顿”误派给 UI。

### 发给 `存档系统`

唯一主刀：

- 把 `Day1` resident / 剧情长期态收成第一版可落地的持久化 contract，并顺手接住 `PersistentPlayerSceneBridge.Start()` 这条 scene-rebind 启动卡顿责任面

### 发给 `NPC`

唯一主刀：

- 给 `Day1` 提供“原生 resident 被导演完全接管时不会乱跑/乱打招呼/乱回旧逻辑”的 contract，并给存档暴露最小 resident runtime snapshot 面

### 发给 `UI`

唯一主刀：

- 只解决 `Day1` 打包字体异常和字体链 build/runtime 一致性问题，不再误吞启动大卡顿主责

### 本轮不发给 `Town`

原因：

1. `Town` 已经把入口 contract、player-facing contract、`HomeAnchor` readiness 都往前站住了
2. 当前最急的闭环口不在 `Town` scene-side，而在 `day1` 主链、持久态、打包体验
3. 除非后续再次撞到 `Town.unity` 原生 actor 本体缺口，否则这轮不优先继续叫 `Town`
