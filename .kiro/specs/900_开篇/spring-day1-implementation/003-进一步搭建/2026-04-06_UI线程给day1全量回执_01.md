## 2. 我这轮最关键的新发现是什么

真正最值钱的新发现，不是“我又调了几个位置”，而是：

### 2.1 Workbench 左列之前为什么一直像空壳

我已经通过 fresh capture 坐实，之前的坏相不是假象：

- 左列高亮背景在
- 但 icon 和文字没有真正显示出来

这轮继续往下查后，我认为当前最像真根因的是：

- generated row 的 `HorizontalLayoutGroup` 没有真实给子项分配尺寸
- 结果就是：
  - accent 可能存在，但只是背景层
  - iconCard 没拿到宽高
  - textColumn 没拿到宽高
  - name / summary 就算被赋了文本，也还是空壳

所以我现在补的不是“多刷一次文字”，而是：

- 创建 runtime row 时：
  - `rowLayout.childControlWidth = true`
  - `rowLayout.childControlHeight = true`
- 复用已有 generated row 时：
  - `generatedLayout.childControlWidth = true`
  - `generatedLayout.childControlHeight = true`

这条判断我认为是这轮最接近根因的一刀。

### 2.2 我没有把第一张截图装成“已经修好”

我已经诚实把证据分层了：

- 第一张 fresh workbench 图只证明了“问题真实存在”
- 修完布局真因后，我还没拿到第二张稳定复抓图
- 所以当前不能 claim：
  - `Workbench 左列已体验过线`

这点 day1 调度时必须看见。
