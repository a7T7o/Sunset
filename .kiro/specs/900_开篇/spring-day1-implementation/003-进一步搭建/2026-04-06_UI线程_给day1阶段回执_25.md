## 这轮实际做成了什么

### 1. Workbench 左列恢复链继续压深了一刀

我这轮没有再停在“像是 layout/字体问题”这一层，而是把运行时绑定链直接往前推：

1. 在 `SpringDay1WorkbenchCraftingOverlay.TryBindRuntimeShell()` 里新增强制恢复口：
   - 如果当前 runtime 壳已经绑上了 prefab shell
   - 但左列还不是 generated row
   - 且正式 recipe 资源能读取
   - 就直接 `RebuildRecipeRowsFromScratch(forceRuntimePrefabStyle: true)`
2. 这意味着：
   - 旧 prefab 手工 row 不再只是“先凑合复用再赌它自己恢复”
   - runtime 绑定阶段就会主动切到稳定 generated row
3. 这刀针对的是之前已经坐实的坏相：
   - 左列背景在
   - 但 icon / 名称 / 摘要肉眼缺失
   - 可以点但像空壳

### 2. formal / informal / resident 玩家面提示壳又往真实 one-shot 规则靠近了一步

这轮我直接补了 3 个实逻辑：

1. `SpringDay1ProximityInteractionService.ShouldReplaceCandidate()`
   - 现在改成 `ForceFocus -> CanTriggerNow -> Priority -> Distance`
   - 不再是先比距离再比 priority
   - 也就是 formal 不会再因为更远一点，就被更近的 informal / resident 抢走
2. `ShouldUseTaskPriorityOverlayCopy()`
   - 不再要求命中固定“交谈 / 对话 / 按 E 开始对话”那组旧文案
   - 只要同 anchor 上 formal 仍是 `Available`
   - 左下角就会稳定回正成 `进入任务 / 按 E 开始任务相关对话`
3. `PlayerNpcNearbyFeedbackService`
   - 现在 formal priority phase 会直接停止自动 nearby feedback
   - 而且如果气泡已经在屏幕上，会立刻 `HideActiveNearbyBubble()`
   - 不再出现 formal 接管瞬间，旁边还挂着旧环境气泡残影

### 3. formal 已消费后的玩家面 idle 语义，不再全部假装成“闲聊”

我补了一层最小 resident 回落语义：

1. `NPCInformalChatInteractable.ShouldUseResidentPromptTone()`
   - 如果同 NPC 的 formal 已明确消费并让位
   - 会显式报出“当前应走 resident 回落语义”
2. `PlayerNpcChatSessionService`
   - idle caption 不再一律是 `闲聊`
   - 现在会落到：
     - `日常交流`
     - `按 E 聊聊近况`

这还不是最终体验终判，但至少玩家面已经不再把“formal 已让位后的 resident 回落”伪装成普通闲聊同一种脸。

### 4. 测试 / 守门也补厚了

我同步把测试口径往前推了一层：

1. `SpringDay1InteractionPromptRuntimeTests`
   - 把 `formal beats informal` 的 runtime case 改成更硬版本：
   - informal 更近
   - formal 文案不是旧 generic 串
   - 仍应让左下角落到 `进入任务`
2. `SpringDay1DialogueProgressionTests`
   - 新增字符串守门：
     - priority 比较必须在 distance 前
     - formal copy 不再依赖 generic prompt 文案
     - formal consumed 后 resident prompt tone 要暴露
     - formal phase nearby feedback 要压住环境气泡
