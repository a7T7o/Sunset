> ### 🚨 锐评一：打字机协程中的“灾难级 GC 炸弹”（性能红线违规）
> 
> 
> * **你的 Diff 代码**：在 `TypeNodeTextRoutine` 协程的 `for` 循环内部，你写了 `yield return new WaitForSeconds(interval);`。
> * **依据与来源**：在《设计文档》(design.md) 的“性能优化策略 -> 协程优化”章节中，明确规定了：**“打字机效果使用 WaitForSeconds 缓存，避免每帧创建新的 YieldInstruction”**。
> * **架构分析与原因**：你用了 `StringBuilder` 来避免字符串拼接的 GC，这很好。但你居然在处理每秒几十次的逐字循环里，每次都 `new` 一个 `WaitForSeconds` 对象？！这意味着如果一句话有 50 个字，你瞬间就在堆内存里制造了 50 个垃圾对象！这彻底粉毁了我们追求“零GC”的初衷。
> * **修正指令**：必须在 `for` 循环**外部**，提前将 `new WaitForSeconds(interval)` 实例化并缓存到一个局部变量中，然后在循环内部直接 `yield return` 这个缓存变量。
