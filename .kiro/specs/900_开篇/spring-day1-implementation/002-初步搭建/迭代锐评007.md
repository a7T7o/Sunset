### 🚨 致命缺陷一：TMPro 脏标记引发的帧级重绘（性能灾难）

* **溯源定位**：在 `DialogueUI.cs` 的 `Update()` 轮询生命周期中，你写下了：
```csharp
if (dialogueText != null) { dialogueText.text = _dialogueManager.CurrentTypedText; }

```


* **架构剖析与原因**：这是极其典型的 Unity UI 新手陷阱。`Update()` 是逐帧执行的。当对话打字机完成，等待玩家点击“继续”的数秒内，`CurrentTypedText` 字符串没有任何变化。然而，`TextMeshPro` 的 `.text` 赋值操作一旦触发，无论字符串是否相同，都会强行触发内部的脏数据标记（Dirty Flag），导致顶层 Canvas 进行毫无意义的网格重建（Rebuild）和持续的 GC 分配。这直接击穿了我们在《设计文档》中定下的性能红线。
* **强制修正令**：
必须在 `DialogueUI.cs` 中引入本地字符串缓存拦截机制。增加私有变量 `private string _lastUpdatedText = string.Empty;`。
将 `Update()` 中的刷新逻辑严格修改为：
```csharp
if (dialogueText != null && _lastUpdatedText != _dialogueManager.CurrentTypedText)
{
    _lastUpdatedText = _dialogueManager.CurrentTypedText;
    dialogueText.text = _lastUpdatedText;
}

```
