## 5. 必做配置：先把验证场景修到可见

### 5.1 修 `DialogueCanvas`

选中 `DialogueCanvas`，重点检查：

- `RectTransform.localScale` 必须改为 `1,1,1`
- Canvas 保持 Screen Space Overlay 即可
- `Canvas Scaler` 保持当前值即可，当前不是主问题

**这是第一优先级。**
如果这里仍然是 `0,0,0`，后面其他 UI 调整都没有意义。

### 5.2 检查 `DialoguePanel`

建议把 `DialoguePanel` 作为底部大面板使用：

推荐目标：
- Anchor：底部拉伸，或全屏底部区域
- Position：靠近屏幕下方
- Size：足够容纳名字、正文、按钮、头像
- Image 颜色：保留半透明深色底

如果你想最快验收，最简单做法是：
- 把 `DialoguePanel` 拉成屏幕底部横条
- 宽度覆盖大部分屏幕
- 高度足够显示两到三行正文

### 5.3 检查 5 个子元素

当前这些对象还是临时占位，需要你手动摆到合理位置：

#### `SpeakerNameText`
建议：
- 放在面板左上或正文上方
- 字体颜色高亮一些
- 显示说话人名字

#### `DialogueText`
建议：
- 放在面板主体区域
- 区域要足够宽，允许换行
- 这是主文本显示区

#### `ContinueButton`
建议：
- 放在右下角
- Button 文案改为“继续”
- 运行时点击它即可推进节点

#### `PortraitImage`
建议：
- 放在左侧或右侧
- 尺寸适中
- 没头像时脚本会自动隐藏

#### `Background`
建议：
- 如果你保留这个对象，就让它做背景图层
- 它可以铺在面板后面
- 普通对话会恢复原透明度，内心独白会半透明

## 7. `DialogueUI` 的使用说明

`DialogueUI` 当前有两种接线方式：

### 7.1 验证场景模式

在 `DialogueValidation` 场景里：
- 即使 `root/speakerNameText/dialogueText/continueButton/portraitImage/backgroundImage` 没手动拖引用
- `DialogueUI` 也会在 `Awake()` 自动按名字补线

所以在验证场景里，**只要对象名不变**，可以先不手拖引用。

### 7.2 正式场景模式

如果以后把这个 UI 用到正式场景：
- 不要依赖验证场景的自动补线逻辑
- 必须手动把各字段拖好
- 并确保脚本挂在“常驻活跃 Canvas 根节点”上，而不是会被整体隐藏/关闭的面板对象上

这是因为脚本本身已经明确写了防退订约束：挂载位置不对，事件订阅可能永久失效。

## 8. 样例对话现在会验证什么

`DialogueValidationBootstrap` 当前运行时会创建一个临时 `DialogueSequenceSO`，里面有 3 个节点：

1. **乱码节点**
   - 说话人：`陌生旅人`
   - `isGarbled = true`
   - 有 `garbledText`
   - 用于验证“未解码显示乱码 / 已解码显示正式文本”

2. **普通对话节点**
   - 说话人：`艾拉`
   - 正常文本
   - 用于验证标准名字 + 文本 + 头像显示

3. **内心独白节点**
   - 无名字
   - `isInnerMonologue = true`
   - 用于验证斜体、浅灰、背景半透明、隐藏名字和头像

## 11. 输入锁验收

当前输入锁链路是：

- `DialogueManager` 发布 `DialogueStartEvent`
- `GameInputManager` 收到后执行 `SetInputEnabled(false)`
- 对话结束时收到 `DialogueEndEvent`
- `GameInputManager` 执行 `SetInputEnabled(true)`

### 建议验证方式

如果这个验证场景里有玩家对象与输入链路：

- [ ] 对话开始后，角色不能移动/交互
- [ ] 对话结束后，角色恢复输入

如果当前验证场景没有完整玩家链路，也没关系；此时把它视为**代码链路已接通、场景侧未覆盖到玩家验证**即可。

## 12. 正式使用时如何配置真实对话

当前 `DialogueValidationBootstrap` 用的是运行时样例，不是正式剧情资产。

如果你后面要开始接正式内容，建议按下面方式配置。

### 12.1 创建正式对话资产

在 Unity Project 面板中：

1. 新建 `DialogueSequenceSO`
2. 填 `sequenceId`
3. 往 `nodes` 列表里逐条加节点

每个节点至少配置：
- `speakerName`
- `text`
- `speakerPortrait`（可空）
- `isInnerMonologue`
- `typingSpeedOverride`

如果需要乱码链路：
- `isGarbled = true`
- `garbledText = 乱码内容`

### 12.2 触发正式对话

正式游戏里，其他系统应调用：

- `DialogueManager.Instance.PlayDialogue(yourSequence)`

而不是继续依赖 `DialogueValidationBootstrap`。

### 12.3 正式 UI 配置

正式场景里建议：

- 把 `DialogueUI` 挂在常驻 Canvas 根对象
- 把 6 个引用全部手拖完整
- 不依赖验证场景自动补线

需要拖的字段：
- `root`
- `speakerNameText`
- `dialogueText`
- `continueButton`
- `portraitImage`
- `backgroundImage`

## 14. 常见问题排查

### 14.1 Play 后完全看不到 UI

优先检查：
1. `DialogueCanvas.localScale` 是否还是 `0,0,0`
2. `DialoguePanel` 是否在屏幕外
3. 文本/图片对象是否还停留在临时占位点
4. `DialogueValidationBootstrap.playOnStart` 是否为 `true`

### 14.2 看到面板但文本不变

检查：
1. `DialogueManagerRoot` 上是否同时有 `DialogueManager` 和 `DialogueValidationBootstrap`
2. `DialogueUI` 是否挂在 `DialogueCanvas`
3. 对象名字是否仍是：
   - `DialoguePanel`
   - `SpeakerNameText`
   - `DialogueText`
   - `ContinueButton`
   - `PortraitImage`
   - `Background`

### 14.3 Continue 按钮点了没反应

检查：
1. `ContinueButton` 上是否还是 Button 组件
2. 场景里是否有 `EventSystem`
3. `DialogueUI` 是否成功拿到 `continueButton` 引用
4. 是否有其他 UI 遮挡住按钮

### 14.4 第一条没有显示乱码

检查：
1. `DialogueValidationBootstrap.startDecoded` 是否被设成 `true`
2. `DialogueManager.IsLanguageDecoded` 是否被其他逻辑改写
3. 样例节点的 `isGarbled` 是否仍为 `true`

### 14.5 内心独白样式没变化

检查：
1. 第三条节点是否为 `isInnerMonologue = true`
2. `DialogueUI` 是否收到 `DialogueNodeChangedEvent`
3. `backgroundImage` 是否存在可见 Image
