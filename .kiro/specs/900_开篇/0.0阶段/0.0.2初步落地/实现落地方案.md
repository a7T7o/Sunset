## 文件级落地方案

### 1. 新建：`StoryPhase.cs`
- 建议路径：
  - `D:\Unity\Unity_learning\Sunset\Assets\YYY_Scripts\Story\Data\StoryPhase.cs`
- 文件职责：
  - 定义 Day1 的最小阶段枚举
- 本轮只做：
  - `enum StoryPhase`
  - 放入当前 Day1 所需最小阶段
- 本轮不做：
  - 多日剧情扩展
  - 存档映射

### 2. 新建：`StoryManager.cs`
- 建议路径：
  - `D:\Unity\Unity_learning\Sunset\Assets\YYY_Scripts\Story\Managers\StoryManager.cs`
- 文件职责：
  - 成为 Day1 状态的最小正式入口
- 需要包含：
  - `CurrentPhase`
  - `IsLanguageDecoded`
  - `SetLanguageDecoded(...)`
  - `SetPhase(...)`
  - `AdvanceTo(...)` 或同级语义入口
- 本轮建议做成：
  - 最小单例/入口管理器
  - 只服务当前 Day1 checkpoint
- 本轮不要做：
  - 存档持久化
  - 跨场景复杂生命周期

### 3. 扩展：`StoryEvents.cs`
- 文件：
  - `D:\Unity\Unity_learning\Sunset\Assets\YYY_Scripts\Story\Events\StoryEvents.cs`
- 要新增的事件：
  - `DialogueSequenceCompletedEvent`
  - `StoryPhaseChangedEvent`
- 建议字段：
  - `DialogueSequenceCompletedEvent`
    - `SequenceId`
    - `DialogueSequenceSO Sequence`
  - `StoryPhaseChangedEvent`
    - `PreviousPhase`
    - `CurrentPhase`
- 目标：
  - 对话结束后，正式通知剧情推进层

### 4. 扩展：`DialogueSequenceSO.cs`
- 文件：
  - `D:\Unity\Unity_learning\Sunset\Assets\YYY_Scripts\Story\Data\DialogueSequenceSO.cs`
- 要新增的字段建议：
  - `bool decodeLanguageOnComplete`
  - `bool advanceStoryPhaseOnComplete`
  - `StoryPhase nextStoryPhase`
  - `DialogueSequenceSO followupSequence`
- 本轮命名要求：
  - 看到 Inspector 字段就知道“播完后会发生什么”
- 本轮边界：
  - 不做大而全条件系统
  - 不做复杂条件表达式

### 5. 扩展：`DialogueManager.cs`
- 文件：
  - `D:\Unity\Unity_learning\Sunset\Assets\YYY_Scripts\Story\Managers\DialogueManager.cs`
- 本轮改动目标：
  - 在序列自然播放完成时，发布 `DialogueSequenceCompletedEvent`
  - 逐步把 `IsLanguageDecoded` 的最终归属让渡给 `StoryManager`
- 建议改法：
  - 第一阶段先兼容：
    - `DialogueManager` 仍保留临时访问入口
    - 但显示判定优先读 `StoryManager`
  - 第二阶段再考虑完全移除本地状态
- 注意：
  - 不要破坏现有打字机/暂停/UI链
  - 不要把“StopDialogue”与“完成序列”混成同一语义

### 6. 扩展：`NPCDialogueInteractable.cs`
- 文件：
  - `D:\Unity\Unity_learning\Sunset\Assets\YYY_Scripts\Story\Interaction\NPCDialogueInteractable.cs`
- 本轮改动目标：
  - 从单一 `dialogueSequence` 模式扩展成：
    - `initialSequence`
    - `followupSequence`
  - 判定逻辑优先读取：
    - `StoryManager`
    - 若缺失再走本地最小兜底
- 建议最小字段：
  - `DialogueSequenceSO initialSequence`
  - `DialogueSequenceSO fallbackSequence` 或 `followupSequence`
- 注意：
  - 本轮只服务首个 NPC 的“首段 -> follow-up”闭环
  - 不要一步扩成复杂 NPC 对话树

### 7. 可选最小兼容位：`DialogueValidationBootstrap.cs`
- 文件：
  - `D:\Unity\Unity_learning\Sunset\Assets\YYY_Scripts\Story\Managers\DialogueValidationBootstrap.cs`
- 只在必要时处理：
  - 如果这个验证脚本仍直接写 `DialogueManager.IsLanguageDecoded`
  - 那就需要补一次兼容，避免验证场景逻辑断掉

  - 不把验证脚本当成主线逻辑中心

## 推荐实现顺序

### 第 1 步：先补数据与状态骨架
- `StoryPhase.cs`
- `StoryManager.cs`

### 第 2 步：再补事件层
- `StoryEvents.cs`

### 第 3 步：再补资源描述能力
- `DialogueSequenceSO.cs`

### 第 4 步：再补运行时完成回调
- `DialogueManager.cs`

### 第 5 步：最后补 NPC 分流
- `NPCDialogueInteractable.cs`

### 第 6 步：收最小真实数据闭环
- 首段对话资源
- follow-up 对话资源
- 解码状态切换
- 阶段推进

## 每一步的直接验收

### A. `StoryPhase.cs`
- 能编译
- 阶段名可被其他脚本直接引用

### B. `StoryManager.cs`
- 项目里有正式 Day1 状态入口
- `CurrentPhase` / `IsLanguageDecoded` 可被读取

### C. `StoryEvents.cs`
- 对话完成事件与阶段变化事件都可被正式发布

### D. `DialogueSequenceSO.cs`
- Inspector 上可以直接配置“播完以后解码 / 推进阶段 / 切 follow-up”

### E. `DialogueManager.cs`
- 一段对话正常播完后，能够发出“序列完成”信号
- 不破坏现有 UI 行为

### F. `NPCDialogueInteractable.cs`
- 同一 NPC 第二次交互时能切换到 follow-up

### G. 数据闭环
- 首段播完 -> 解码
- 首段播完 -> 推进阶段
- 再交互 -> 播 follow-up
