## 1. 这份文档解决什么问题

上一份总表解决的是“历史需求到底说过什么”，但它还没有回答另外一个更关键的问题：

1. 51 个未完成信号里，哪些其实是同一类问题的不同表述
2. 现在到底应该先做什么、后做什么
3. 哪些是当前必须先拆掉的 live blocker
4. 哪些虽然没做完，但不应该抢在第一优先级

这份文档的目标不是重复讲历史，而是把 Day1 当前全盘未完项重新压缩成真正可执行的清盘顺序，避免继续陷入：

- 框架先长一层、体验仍没闭环
- 看到哪里脏就补哪里，刀口不断漂
- 一个需求同时以“系统缺口 / UI缺口 /剧情缺口 / 现场 blocker”四种身份重复出现，却没人统一归并

## 2. 总体诊断

基于总表，当前结论很明确：

1. Day1 最大的问题不是“缺设计”，而是“已经有很多设计和局部实现，但缺一条条真正过用户体感验收的纵切闭环”。
2. 当前最重的真实债务不在通用框架，而在“前半链 + 工作台/任务体验”。
3. 开场 CG、通用触发器、NPC 引导系统这些并不是不重要，而是不应该抢在最痛的用户可感知问题前面。
4. 现在的未完项里，有相当一部分只是同一根主问题的不同投影，不能按 51 个独立任务来排。

## 7. 后续优先级重排

下面是我认为当前最合理的执行顺序。

这里不是“按需求编号顺排”，而是按：

- 用户痛点强度
- 依赖关系
- mixed hot 风险
- 一刀能闭多少真实体验

综合重排后的结果。

## G0｜先做施工闸门，不算业务完成，但必须先过

### 为什么先做它

因为当前 shared root 上已经存在真实 mixed/hot 风险面。

如果这一步不先做，后面每一刀都可能重新掉进：

- owner 不清
- same-root blocker
- 业务没补到，先把 shared root 搅混

### 这一优先级包含什么

1. 重新钉实 Day1 当前允许动的正式面
2. 重新确认当前 mixed/hot 面谁能碰、谁不能碰
3. 把下一刀限定成一个单纵切，而不是“顺手多补几类东西”
4. 明确本刀白名单根与不可扩认范围

### 完成标准

1. 下一刀的 allowed surfaces 清楚
2. 当前 hot/mixed blocker 不再含糊
3. 白名单预检可过

## P0｜先收最重痛点：Day1 前半链 + 工作台 / 任务体验纵切闭环

### 为什么它排第一

因为当前最强用户痛点、最多返工次数、最多“部分完成”的条目，都集中在这里。

这一刀如果闭环，能同时压掉一大批 `部分完成`：

- 首段对话
- 乱码解码
- HP/EP 节奏
- `Anvil_0` 入口
- 任务卡
- 木材步骤防 bypass
- 制作耗时/进度/工作感
- UI 比例、滚动、自适应、手调、距离提示、一次性 `E`

### 这一优先级包含什么

建议把这一刀定义成单一可验纵切：

1. `NPC001` 首段对话启动
2. 乱码 -> 解码链稳定
3. 血条首次出现
4. 进入工作台真实入口 `Anvil_0`
5. 任务卡与提示链开始工作
6. 木材获取不能 bypass
7. 工作台列表 / 详情 / 制作 / 进度 / 提示 / 退出真正形成闭环
8. 玩家能亲手做出 Day1 第一件工具或完成第一步正式制作目标

### 涉及的核心条目

- `R05`
- `R06`
- `R10`
- `R11`
- `R12`
- `R17`
- `R18`
- `R19`
- `R22`
- `R33-R38`
- `R41-R48`
- `R52`
- `R53`
- `B03`

### 完成标准

1. 玩家可以从首段对话一路走到“工作台成功完成第一轮正式任务目标”
2. 中间没有绕任务顺序的 bypass
3. 工作台 UI 可以手动调，不再是“只能代码上看起来能动”
4. 玩家做东西时有明确的在干活感觉
5. 这条纵切的自验和用户验收都能聚焦，不再散

2. 晚餐回血桥
3. 归途与夜间提醒
4. 自由时段
5. 回住处与睡觉结束
6. “两点前必须睡”的规则威胁与夜间压迫感

### 涉及条目

- `R20`
- `R40`
- `R55`
- `R56`
- `R08`

### 完成标准

1. Day1 不再只停在工作台 / 教学后
2. 玩家能完整走完第一天
3. 结束时能感到“这一天真的结束了”，而不是“流程断在一半”

## 8. 我认为下一刀应该怎么选

如果现在由我继续往下做，我不会直接从 `R03`、`R13` 这类开场氛围项开刀，也不会先去补一个更大的通用触发框架。

我会按下面顺序走：

1. 先过 `G0`
2. 然后直接进入 `P0`

更具体地说，我会把下一刀固定为：

`Day1 前半链 + 工作台 / 任务体验纵切闭环`

也就是：

1. 不再散着补“某个 UI 细节”
2. 不再散着补“某个剧情节点”
3. 而是把 `NPC001 首段对话 -> 乱码解码 -> HP/EP 节奏 -> Anvil_0 -> 任务卡 -> 木材 -> 制作完成` 这一整条玩家真实会体验到的纵切，一次性闭到可验

原因很简单：

1. 这是当前用户最不满意的区域
2. 这是当前 `部分完成` 最密集的一组条目
3. 这条纵切一旦闭环，Day1 立刻会从“很多局部都做过”升级成“至少有一段真正像游戏了”
4. 这也能为后续 `P1` 后半链闭环提供稳定前提

## 9. 一句话总裁定

当前 Day1 不应该继续“按系统分散补漏”，而应该切回“按玩家真实体验纵切收口”。

最合理的顺序是：

`G0 施工闸门 -> P0 前半链 + 工作台体验 -> P1 后半链整日闭环 -> P2 开场与返村氛围 -> P3 必要通用系统 -> P4 最终验收与清扫`

其中，真正的第一把业务刀，应当是 `P0`，而不是别的地方。
