## 3. 四层问题拆解

### 3.1 Git / worktree 真实层

- `NPC` 与 `farm` 的 worktree 都还在；
- 真实功能分支都还在；
- 根仓库报 “branch already used by worktree” 不是 Git 丢失，而是根仓库正在错误地尝试代替功能 worktree；
- 因此当前主问题不在 Git 本体，而在“线程点开后没有回到对应 worktree”。

### 3.2 状态库投影层

- 当前最强冲突仍是 `git_branch` 旧值；
- 两个样本都表现出：
  - `cwd` 已落在独立 worktree
  - `git_branch` 仍停留在 `main`
- 这会让客户端恢复线程时把“worktree 现场”与“根仓库分支语义”混接。

### 3.3 UI / workspace 承接层

- 用户反馈显示：重启后线程会短暂回到正确项目文件夹；
- 但点击线程后又掉回 `Sunset/main`；
- 这说明 UI / workspace 承接层至少存在以下三种可能之一：
  1. 点击线程时重新读取了过时状态投影；
  2. 工作区激活根刷新未和线程元数据一起切换；
  3. 线程自身路由语义与 worktree 归属没有真正绑定。
- 当前最保守结论是：三者并存，尚不能只归咎一层。

### 3.4 用户验收层

用户真正需要的不是“字段对了”，而是：

1. 点开线程回正确 worktree；
2. UI 保持在正确项目组；
3. 不再掉回 `Sunset/main`；
4. 不再误触发根仓库切功能分支；
5. 重启与不重启两条路径都成立。

## 5. 两个核心问题的正式回答

### 5.1 到哪一阶段才能验收到“重启/不重启都正常”

- 只读治理阶段：不能；
- 修复设计阶段：不能；
- 单线程真实修复阶段：只能争取让单线程样本成立；
- 用户真实闭环验收阶段：只有当 `NPC` / `farm` / 治理线程在“重启后”和“不重启”两种路径下都通过，才算真正恢复。

### 5.2 当前是否存在“一跑即恢复 UI”的脚本式能力

- 当前没有已证实可承诺的现成能力；
- 当前最接近的方案是：
  1. 对单线程建立受控修正预案；
  2. 先最小修正状态投影字段；
  3. 再用用户真实点击矩阵做验收；
  4. 若样本稳定，再考虑抽象成更通用脚本。
- 当前 fallback 仍是：
  - 先核定真实 worktree；
  - 必要时重启后复验；
  - 明确这只是 workaround，不是最终恢复。

这些判断现在都只保留为历史阶段快照，不再作为当前执行入口。

### 13.2 新默认路线

当前新的默认路线固定为：

- 默认 Unity 开发现场：`D:\Unity\Unity_learning\Sunset`
- 默认工作模式：主项目优先
- 默认并行方式：允许在主项目体系并行推进普通任务
- worktree 仅保留为：
  - 高风险隔离方案
  - 故障修复方案
  - 特殊长线实验方案

### 13.3 `NPC/farm` 回归评估结论

- `NPC`：可以回归 `main`
  - 业务改动集中在 NPC 相关脚本与资源；
  - 当前真正阻碍“直接整支回归”的，是分支内混入的治理/记忆文件，而不是 NPC 业务改动本身。
- `farm-10.2.2-patch002`：可以回归 `main`
  - 业务改动集中在 `PlacementManager.cs` 与 `PlacementValidator.cs`；
  - 当前真正阻碍“直接整支回归”的，也是不应随业务线一起回归的治理/记忆文件。

### 13.4 当前执行入口

当前不再继续推进热恢复首刀。

当前新的最小执行入口固定为：

1. 为 `NPC` 与 `farm` 建立白名单业务成果回归预案；
2. 只回归业务成果，不整支 merge 独立分支；
3. 业务成果回归后，再单独处理线程锚点回根仓库与 worktree 退役。

## 15. 2026-03-13 设计收口：主项目回归优先于热恢复修复

### 15.1 当前设计主线

自本节开始，线程承接修复设计不再把“继续优化独立 worktree 热恢复”作为最高优先级。

新的设计主线固定为：

- 先把 `NPC` 与 `farm` 的业务成果干净回流到 `D:\Unity\Unity_learning\Sunset`；
- 再把这两条线程的承接语义改回根仓库 `main`；
- 最后把 worktree 降级为例外工具链，而不是默认日常现场。

### 15.2 当前线程锚点设计

对 `NPC` 与 `farm`，目标语义统一为：

- `cwd = D:\Unity\Unity_learning\Sunset`
- `git_branch = main`
- `git_sha = root main 当前白名单提交后的 HEAD`
- rollout 恢复层改回根仓库主项目语义

### 15.3 当前剩余的热恢复定位

热恢复问题从本节开始降级为次级问题。

它仍保留为历史阶段结论：

- 能解释为什么早前点击线程会回落到旧现场；
- 也保留为后续若线程再次漂移时的故障修复工具链。

但它不再阻止 `NPC/farm` 回归主项目体系，也不再阻止我们先把默认开发模式改回 `Sunset` 主项目优先。
