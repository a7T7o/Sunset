## 3. 四层问题拆解

### 3.1 Git / worktree 真实层

- `NPC` 与 `farm` 的 worktree 都还在；
- 真实功能分支都还在；
- 根仓库报 “branch already used by worktree” 不是 Git 丢失，而是根仓库正在错误地尝试代替功能 worktree；
- 因此当前主问题不在 Git 本体，而在“线程点开后没有回到对应 worktree”。

### 3.2 状态库投影层

- 当前最强冲突仍是 `git_branch` 旧值；
- 两个样本都表现出：
  - `cwd` 已落在独立 worktree
  - `git_branch` 仍停留在 `main`
- 这会让客户端恢复线程时把“worktree 现场”与“根仓库分支语义”混接。

### 3.3 UI / workspace 承接层

- 用户反馈显示：重启后线程会短暂回到正确项目文件夹；
- 但点击线程后又掉回 `Sunset/main`；
- 这说明 UI / workspace 承接层至少存在以下三种可能之一：
  1. 点击线程时重新读取了过时状态投影；
  2. 工作区激活根刷新未和线程元数据一起切换；
  3. 线程自身路由语义与 worktree 归属没有真正绑定。
- 当前最保守结论是：三者并存，尚不能只归咎一层。

### 3.4 用户验收层

用户真正需要的不是“字段对了”，而是：

1. 点开线程回正确 worktree；
2. UI 保持在正确项目组；
3. 不再掉回 `Sunset/main`；
4. 不再误触发根仓库切功能分支；
5. 重启与不重启两条路径都成立。

## 5. 两个核心问题的正式回答

### 5.1 到哪一阶段才能验收到“重启/不重启都正常”

- 只读治理阶段：不能；
- 修复设计阶段：不能；
- 单线程真实修复阶段：只能争取让单线程样本成立；
- 用户真实闭环验收阶段：只有当 `NPC` / `farm` / 治理线程在“重启后”和“不重启”两种路径下都通过，才算真正恢复。

### 5.2 当前是否存在“一跑即恢复 UI”的脚本式能力

- 当前没有已证实可承诺的现成能力；
- 当前最接近的方案是：
  1. 对单线程建立受控修正预案；
  2. 先最小修正状态投影字段；
  3. 再用用户真实点击矩阵做验收；
  4. 若样本稳定，再考虑抽象成更通用脚本。
- 当前 fallback 仍是：
  - 先核定真实 worktree；
  - 必要时重启后复验；
  - 明确这只是 workaround，不是最终恢复。

## 6. 单线程最小修复入口

### 6.0 首刀顺序

- 本轮第一刀对象固定为 `NPC`；
- 选择 `NPC` 的原因是：它同时呈现“真实 worktree 正常存在 + 状态库 `git_branch` 仍是 `main` + 用户点击线程掉回 `Sunset/main`”这组最完整冲突链；
- `farm-10.2.2-patch002` 当前定位为并行参照样本：用于验证同类问题是否可复现、以及 `NPC` 预案后续是否值得推广；
- 因此本轮不同时对两个样本进入状态修正。
### 6.1 `NPC`

#### 当前错误现象
- 用户点开线程后掉回 `Sunset/main`
- 根仓库误尝试切 `codex/npc-generator-pipeline`
- 报 worktree 占用错误

#### 为什么这不是 Git 丢失
- 因为真实 worktree 与真实分支都仍然存在；
- 根仓库切不进去，正是因为该分支已被 worktree 正常占用。

#### 如果先不改数据库，最远能推进到哪
- 可以完成：
  - 真实现场核定
  - 受控字段候选清单
  - 备份方案
  - 单线程验收矩阵
- 不能完成：
  - 线程点开即稳定回正确现场

#### 如果进入最小状态修正，优先候选字段
1. `git_branch`
2. `git_sha`
3. `cwd`
4. `rollout_path`

#### 改前备份
- 导出该线程当前整行状态；
- 备份 `state_5.sqlite` 原文件；
- 记录当前真实 worktree 分支与 HEAD。

#### 改后单线程验收
- 打开 `NPC` 线程；
- 记录是否仍掉回 `Sunset/main`；
- 记录是否仍触发根仓库切分支；
- 记录是否仍报 worktree 占用；
- 分别验证“重启后打开”和“不重启直接打开”。

#### 回滚
- 用备份文件恢复 `state_5.sqlite`
- 用备份整行恢复单线程字段
- 再次运行只读基线脚本确认恢复

## 7. 通过标准

对 `NPC` 与 `farm`，只有同时满足以下条件，才允许写成“单线程恢复”：

1. 点开线程回到正确 worktree；
2. 不再掉回 `Sunset/main`；
3. 不再误触发根仓库切功能分支；
4. 不再报 `branch already used by worktree`；
5. UI 项目分组稳定；
6. 重启与不重启两条路径都能成立，或明确写清当前只成立哪一条。
