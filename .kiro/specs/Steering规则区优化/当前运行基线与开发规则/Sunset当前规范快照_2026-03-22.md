# Sunset 当前规范快照（2026-03-22）

## 1. 文件身份
- 本文件是 Sunset 当前唯一规范快照。
- 从 2026-03-22 起，凡是涉及：
  - 默认 Git 开发模型
  - 线程默认如何提交
  - branch / worktree 何时才是例外
  - 当前应遵守哪份活规范
  统一以本文件为准。
- 如果旧文档仍写着 `main-common + branch-task + checkpoint-first + merge-last`，按历史背景理解，不再视为当前默认口径。

## 2. 当前默认开发模型
- 当前默认模型不是 branch-first。
- 当前默认模型是：
  - `main-only + whitelist-sync + exception-escalation`
- 直白说就是：
  - 普通线程默认都在 `D:\Unity\Unity_learning\Sunset @ main` 语义下开发
  - 完成一刀后，默认只提交自己修改的内容
  - 默认只提交自己的白名单路径
  - 默认一线程一次只收一刀自己的 checkpoint
- 治理线程默认不代替业务线程提交。
- 治理线程主要负责：
  - 高危打断
  - 顺序裁定
  - 例外收口

## 3. 只有这些情况才升级治理介入
1. 撞到同一个高危目标
2. 同一个 Scene / Prefab / 热脚本被多人改
3. 跨线程强耦合，无法按白名单拆刀
4. 成果还停在 branch carrier / worktree，尚未迁入 `main`
5. 需要 Unity / MCP live 写，且已有人占用

## 4. 当前高危目标口径
- 同一个 Scene
- 同一个 Prefab
- `Primary.unity`
- `Assets/000_Scenes/SceneBuild_01.unity`
- `Assets/YYY_Scripts/Controller/Input/GameInputManager.cs`
- 任何当前正被他线程 live 写入的 Unity 资源

## 5. 当前 Git 收口口径
- 普通线程完成一刀后，默认直接做白名单 `main` 收口。
- 当前允许：
  - `git-safe-sync.ps1 -Action sync -Mode task -OwnerThread <线程名> -IncludePaths ...`
  - 在 `main` 上执行
  - 只收本线程白名单
  - 保留 unrelated dirty
- 当前不再允许把“shared root 里还有 unrelated dirty”当成默认一刀切阻断理由。
- 真正要阻断的是：
  - 你想把别人的改动也带进去
  - 你在碰高危目标
  - 你在做无边界提交

## 5A. 当前代码正确率闸门
- 自 2026-03-23 起，只要 `task` 模式白名单里包含 `.cs` 文件，`git-safe-sync.ps1` 会在收口前自动触发代码闸门。
- 代码闸门当前固定做 3 件事：
  - 目标文件 UTF-8 检查
  - `git diff --check`
  - 程序集级编译检查
- 代码闸门未通过：
  - 不允许继续收口
  - 不应再把“等 Unity / MCP / 用户贴报错后再修”当作默认流程

## 6. branch / worktree 当前口径
- branch / worktree 没有被彻底废除。
- 但它们现在只属于例外机制：
  - branch carrier 迁入 `main`
  - 高危隔离
  - 特殊实验
  - 独立施工 worktree
- 普通开发不再默认先走 `ensure-branch`。
- `scene-build` 仍是当前例外：
  - `D:\Unity\Unity_learning\Sunset_worktrees\scene-build-5.0.0-001`

## 6A. Unity / MCP live 当前唯一基线
- 自 2026-03-23 起，凡是涉及 Unity / MCP live 验证、Console 读取、Scene / Prefab live 回读，统一先过：
  - `D:\Unity\Unity_learning\Sunset\.kiro\locks\mcp-live-baseline.md`
- 当前唯一有效 MCP server 名：
  - `unityMCP`
- 当前唯一有效 MCP 端点：
  - `http://127.0.0.1:8888/mcp`
- 旧口径以下全部失效：
  - `http://127.0.0.1:8080/mcp`
  - `http://localhost:8080/mcp`
  - `mcp-unity`
- 当前 live 判断不允许只写“Session Active / Configured 成功”；至少要补齐：
  - 当前端口是否监听
  - 当前 pidfile 是否存在
  - 当前会话 resources / templates 暴露的 server 名
  - 当前实例是否真的是本轮目标实例
- 如果 `config.toml + 8888 + pidfile + baseline 脚本` 已通过，但旧线程仍报旧端口、旧桥名或 `resources/templates` 为空，默认先判为“旧会话 MCP 路由缓存未刷新”，不是服务端自动回滚；应改用新线程 / 新会话复核，或先手工对 `8888` 做 initialize + `tools/list`。

## 7. checkpoint 命名口径
- Git SHA 不能自定义。
- 当前统一的是：
  - checkpoint 名称
  - commit message
- 推荐格式：
  - `YYYY.MM.DD_<线程名>_<编号>`
- 例如：
  - `2026.03.22_农田_01`
  - `2026.03.22_spring-day1_01`
  - `2026.03.22_导航检查_01`

## 8. 当前唯一入口顺序
1. 先读本文件
2. 再读 `D:\Unity\Unity_learning\Sunset\AGENTS.md`
3. 再读当前活跃工作区正文
4. 如需 Unity / MCP，再额外核 `.kiro/locks/`

## 9. 一句话口径
- Sunset 现在的默认真相不是“先切 branch 再开发”，而是“默认 main-only 白名单提交，只有少数例外才升级 branch / worktree / 治理介入”。
## 0. 2026-03-23 补强口径
- 当前主模型不变，仍是：`main-only + whitelist-sync + exception-escalation`
- 但从现在起，下面这些补强项一并视为 live 规则：
  - 规则更新必须四入口同轮同步：`AGENTS.md`、本快照、批次 `README.md`、通用前缀 prompt
  - 碰过 `Scene / Prefab / Primary.unity / 热 Unity 资源` 的线程，必须回“热文件卫生字段”
  - 白名单里含 `.unity / .prefab / .asset` 时，收口前必须做资源卫生分类：`有效内容 / 自动副产物 / 调试残留 / 他线脏改`
  - branch / worktree 例外必须走退场四步：先迁最小 checkpoint，再移除旧 blocker，再回到 `main` 语义继续
  - Unity / MCP live 写或验证必须先报四字段：`需要什么实例 / 最多占多久 / 只做什么 / 做完退回什么状态`
  - 线程回执最小字段固定化，并默认直接写回批次固定文件
  - 默认执行纪律改成：`一刀一收`
