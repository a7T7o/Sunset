# Sunset 强制技能闸门与线程回复规范（2026-03-17）
## 1. 文件身份
- 本文件是 `2026-03-17` 新增的现行专项说明。
- 它不宣称“强制 skills 已全部落地”，而是用来明确：
  - 为什么必须补这一层
  - 这层最少要管什么
  - 当前做到哪一步
  - 剩余工作落在哪个阶段代办中

## 2. 当前根本问题
- `Sunset` 现在不是没有规则，也不是没有 skills。
- 当前真实问题是：
  - `rules`
  - `AGENTS.md`
  - `skills`
  还没有组成一个“任何 Sunset 实质性任务都必须先经过”的硬闸门。
- 所以才会反复出现：
  - 明明有现行入口
  - 明明有 skill
  - 但执行时仍然读偏、跳过、漂移

## 3. 启动闸门最少必须回答的事
1. 当前主线目标是什么。
2. 本轮是继续主线、处理中途阻塞，还是明确换线。
3. 当前命中哪个工作区。
4. 当前工作目录 / 分支 / `HEAD` 是什么。
5. 当前共享根目录是否已被其他线程占用。
6. 本轮是否涉及 A 类热文件。
7. 本轮必须先读哪些 `AGENTS.md`、steering、workspace memory。
8. 本轮必须调用哪些同伴 skills。
9. 第一条 `commentary` 应如何说明“理解 + 第一动作”。

## 4. 线程回复的最低结构
- 当前主线目标
- 当前工作目录 / 分支 / `HEAD`
- 本轮实际动作
- 是否触及共享热文件或共享根目录占用
- 已验证事实
- 当前阻塞
- 下一步

## 5. 当前应协同的 skills
- `skills-governor`：做前置核查与 skill 路由
- `sunset-workspace-router`：判工作区与活文档入口
- `sunset-lock-steward`：判 A 类热文件锁
- `sunset-thread-wakeup-coordinator`：产出线程唤醒 / 冻结 / 复工 prompt
- `sunset-doc-encoding-auditor`：判真实乱码与编码治理边界
- `sunset-release-snapshot`：阶段快照与收口摘要
- `sunset-unity-validation-loop`：Unity 验证闭环

## 6. 新增的事故反馈

  - 当前目录是否被别的线程占用
