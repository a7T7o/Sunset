# Sunset 强制技能闸门与线程回复规范（2026-03-17）

## 1. 文件身份
- 本文件定义 Sunset 的项目级启动闸门和最小线程回复结构。
- 它不是在讨论“有没有规则”，而是在规定“实质性任务必须先怎么进入”。

## 2. 强制闸门
- 只要任务发生在 `D:\Unity\Unity_learning\Sunset` 且属于实质性任务，必须先过：
  - `C:\Users\aTo\.codex\skills\sunset-startup-guard\SKILL.md`
- 如果当前会话 skill inventory 未显式列出 `sunset-startup-guard`，视为 discovery gap，不视为闸门失效；必须继续手工执行同级前置核查，并在 `C:\Users\aTo\.codex\memories\skill-trigger-log.md` 写明 `manual-equivalent`。
- 典型实质性任务包括：
  - 代码修改
  - 场景 / Prefab / Inspector 修改
  - rules / skills / AGENTS / memory / 治理文档修改
  - 多文件排查
  - Git 状态与分支治理判断
  - Unity 验证

## 3. 闸门最少要回答什么
1. 当前主线目标是什么
2. 本轮是继续主线还是清阻塞
3. 当前工作目录 / 分支 / HEAD 是什么
4. shared root 当前是否中性
5. 当前命中哪个工作区
6. 需要先读哪些 live 文档
7. 需要哪些 companion skills
8. 如果会进入 Play Mode，离场前如何确保回到 Edit Mode
9. 如果会碰 UI / 字体 / 气泡 / 样式，额外要读哪份表现层规范

## 4. shared root 占用层
- shared root 占用独立记录在：
  - `D:\Unity\Unity_learning\Sunset\.kiro\locks\shared-root-branch-occupancy.md`
- 它不等于 A 类热文件锁。
- 它只用于判断：
  - 当前 shared root 是不是中性现场
  - 是否允许把它当默认进入现场

## 5. MCP 单实例层
- Unity / MCP 单实例占用独立记录在：
  - `D:\Unity\Unity_learning\Sunset\.kiro\locks\mcp-single-instance-occupancy.md`
  - `D:\Unity\Unity_learning\Sunset\.kiro\locks\mcp-hot-zones.md`
  - `D:\Unity\Unity_learning\Sunset\.kiro\locks\mcp-single-instance-log.md`
- 它回答的不是“当前分支对不对”，而是：
  - 当前 Editor / MCP 是否处于可安全读写状态
  - 当前是否命中 Play / Compile / Domain Reload / 共享场景 / 共享 Inspector 热区
- 结论口径：
  - shared root 中性，不代表 Unity / MCP 可安全写
  - 只要命中单实例冲突信号，就先停写、复核现场、必要时只读取证
  - 如果本轮为验证进入了 Play Mode，离场前必须先 Stop 回 Edit Mode

## 6. worktree 红线
- `worktree` 只允许作为：
  - 事故隔离
  - cleanroom
  - 特殊实验
- 不允许再把 `worktree` 常态化为 Sunset 默认开发方式。

## 7. 线程回复最小结构
- 当前主线目标
- 当前工作目录 / 分支 / HEAD
- 本轮实际动作
- 如进过 Play Mode，明确是否已回到 Edit Mode
- 已验证事实
- 当前阻塞
- 下一步
