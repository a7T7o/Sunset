# Sunset 基础规则与执行口径

## 1. 当前唯一主口径
- 默认现场：
  - `D:\Unity\Unity_learning\Sunset @ main`
- 默认开发模型：
  - `main + branch-only`
- shared root 占用状态：
  - `D:\Unity\Unity_learning\Sunset\.kiro\locks\shared-root-branch-occupancy.md`
- worktree 口径：
  - 只作例外，不作日常

## 2. 目录职责
- `.kiro/steering/`
  - 规则正文源
- `Steering规则区优化/当前运行基线与开发规则`
  - 现行入口和当前口径
- `.kiro/specs/Codex规则落地`
  - 正式治理工作区
- `.kiro/specs/000_代办/codex`
  - TD 镜像区，不是工作区

## 3. 线程默认进入方式
1. 先从 shared root 进入
2. 先过 `sunset-startup-guard`
3. 先核：
   - 当前主线
   - 当前工作目录 / 分支 / HEAD
   - shared root 是否中性
4. 治理留在 `main`
5. 业务切到对应 `codex/...`

## 4. Git 口径

### 治理类改动
- 允许留在 `main`
- 收尾使用：
  - `git-safe-sync.ps1 -Action sync -Mode governance`
- 仍需显式给出：
  - `-IncludePaths`
  - 或 `-ScopeRoots`

### 业务类改动
- 不直接在 `main` 上收尾
- 进入实现前先执行：
  - `git-safe-sync.ps1 -Action ensure-branch -BranchName codex/...`
- 收尾使用：
  - `git-safe-sync.ps1 -Action sync -Mode task`

## 5. Unity / MCP 单实例口径
- Git 现场安全，不等于 Unity / MCP 现场安全。
- 只要任务涉及 Unity、MCP、Play Mode、Compile、Domain Reload、Console、Scene / Prefab / Inspector 读写，必须额外核：
  - `D:\Unity\Unity_learning\Sunset\.kiro\locks\mcp-single-instance-occupancy.md`
  - `D:\Unity\Unity_learning\Sunset\.kiro\locks\mcp-hot-zones.md`
  - `D:\Unity\Unity_learning\Sunset\.kiro\locks\mcp-single-instance-log.md`
- 当前默认口径：
  - Unity 视为单实例共享资源
  - 写操作默认不假设天然排队安全
  - 读操作也不假设一定能读到稳定最终态
- 出现以下任一信号时，先停止继续写入并退回只读核查：
  - Play Mode 切换
  - 编译 / Domain Reload
  - 对象失效、端口占用、MCP 返回中间态
  - 场景 / Prefab / Inspector 状态突然跳变

## 6. 当前治理工作区与代办区规则
- 正式治理阶段一律开在：
  - `D:\Unity\Unity_learning\Sunset\.kiro\specs\Codex规则落地`
- `000_代办/codex` 禁止再出现：
  - 根层 `memory.md`
  - 阶段 `tasks.md`
  - 阶段正文
  - 执行证据正文
- `000_代办/codex` 只允许：
  - TD 文件
  - 使用说明
  - 指向正式工作区

## 7. 四件套口径
- 根层默认只保留：
  - `memory.md`
  - `memory_0.md`
  - `design.md` 按需
- `tasks.md` 只属于阶段目录，不再在治理工作区根层常驻
- `requirements.md` / `design.md` 只在确有必要时创建

## 8. 线程汇报最小结构
- 当前主线目标
- 当前工作目录 / 分支 / HEAD
- 本轮实际动作
- 已验证事实
- 当前阻塞
- 下一步

## 9. 一句话口径
- 现在的 Sunset 不是“代办区兼任工作区”，而是“规则正文、live 入口、正式治理工作区、TD 镜像”四层分离；Git 中性现场和 Unity / MCP 单实例安全也必须分层判断。
