## 3. Git 执行口径

### 治理类改动
- 范围：
  - 规则
  - 索引
  - `memory`
  - Hook 文档
  - 脚本说明
- 允许直接留在 `main`
- 收尾方式固定为：
  - `git-safe-sync.ps1 -Action sync -Mode governance`
  - 白名单提交

### 真实任务改动
- 范围：
  - `Assets/`
  - `Packages/`
  - `ProjectSettings/`
  - 业务脚本
  - 场景
  - Prefab
  - ScriptableObject
  - 资源
- 不直接在 `main` 上收尾
- 收尾方式固定为：
  1. 从默认入口做 `ensure-branch`
  2. 切到 `codex/xxx`
  3. 用 `git-safe-sync.ps1 -Mode task` 白名单同步

### 白名单原则
- 白名单只解决“本轮带哪些文件”。
- 白名单不等于放宽分支规则。
- 禁止无边界 `git add -A`。

## 4. 多线程协作口径
- 当前真正的第一优先级，不是回复漂亮，而是共享状态清楚。
- 所有线程推进前，都要先看：
  - `Sunset活跃线程总表_2026-03-15.md`
  - `Sunset文件认领与锁定表_2026-03-15.md`
  - `Sunset当前工作树dirty与WIP归属表_2026-03-15.md`
- 如果本轮改动会改变认领范围、WIP 边界或线程状态，必须同步共享表，不允许只写在本线程自己的 `memory` 里。
- 用户不应继续充当线程之间的手工中继站。

## 5. 线程汇报口径
- 每条线程汇报都必须先写“当前主线目标”。
- 必须区分：
  - 已验证事实
  - 当前判断 / 推测
- 必须写清：
  - 当前目录
  - 当前分支
  - 当前 `HEAD`
  - 当前阻塞
  - 下一步
  - 验收点
- 固定模板统一见：`Sunset线程统一汇总模板_2026-03-15.md`

## 6. 记忆更新口径
- 只要本轮有实质性结论、改动、验证或稳定判断，就必须更新记忆。
- 更新顺序固定为：
  1. 子工作区 `memory.md`
  2. 父工作区 `memory.md`
  3. 线程 `memory_0.md`
- 记忆必须写清：
  - 当前主线目标
  - 本轮子任务 / 阻塞
  - 已完成事项
  - 涉及文件
