## 4. MCP 当前真实定性
- 当前项目安装的是 `com.coplaydev.unity-mcp`，并且 Unity 侧日志明确出现 `Successfully registered with Claude Code using HTTP transport.`。
- 这条提示说明：Unity 包内置配置器当前成功登记的是 `Claude Code` 的 HTTP MCP 客户端，不是对 `Codex` 会话成功回执的证明。
- 当前 `C:\Users\aTo\.codex\config.toml` 的活配置已收束为只维护 `[mcp_servers.unityMCP]` 这一条 HTTP 入口；旧的 `[mcp_servers.mcp-unity]` `stdio` 配置已从活配置中移除。
- 当前需要区分两件事：
  - `unityMCP`：现在是 Sunset 默认且唯一应使用的 Codex Unity MCP 入口；
  - 历史 `mcp-unity`：只允许作为过往迁移记录或备份语境存在，不再允许继续作为当前工具链口径。
- 当前治理结论固定为：
  - Sunset 默认客户端应统一到 `Codex`
  - 默认只维护 `unityMCP` 这条 HTTP 入口
  - `Claude Code` 注册只保留为明确切换客户端时的例外配置，不再作为 Sunset 默认口径
