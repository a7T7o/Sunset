## 一、核心差异与适配原则

### 1.1 Kiro vs Cursor 核心差异

| Kiro（自动） | Cursor（手动） | 适配方案 |
|-------------|---------------|----------|
| 自动继承对话（>80%压缩后生成快照） | 无此机制 | 不需要融入 |
| Hook 自动触发规则加载 | Rules 配置 + globs | `.cursor/rules/` + frontmatter |
| agentStop 自动更新 Memory | 手动触发 | 任务结束提醒用户 |
| 自动生成继承会话快照 | 输入指令触发 | "执行对话快照操作" |

### 1.2 不需要融入的特性

- 上下文压缩检测（Kiro 自动检测）
- 自动继承对话生成
- Hook 的自动化触发机制

### 1.3 需要融入的特性

- Steering 规则体系（`.kiro/steering/` → `.cursor/rules/`）
- 手动 Memory 更新流程
- 手动快照操作流程（"执行对话快照操作"）
- 规则智能加载思路（通过 Cursor 的 globs 实现）

## 五、使用指南

### 5.1 日常使用

1. **规则自动加载**：每次对话自动加载 `rules.md`
2. **手动加载规则**：输入 `#规则名`（如 `#ui`、`#trees`）
3. **Memory 更新**：任务结束后手动执行或由 AI 提醒

### 5.2 手动快照

当需要保存长对话上下文时，输入：
```
执行对话快照操作
```

### 5.3 指令对照表

| Kiro（自动） | Cursor（手动） |
|-------------|---------------|
| Hook 自动加载 | `#规则名` 或 Rules globs |
| 自动更新 Memory | 任务结束手动更新 |
| 自动生成快照 | 输入"执行对话快照操作" |
