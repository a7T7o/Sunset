# Sunset 统一汇总验收模型（2026-03-15）

## 1. 目的
- 本模型用于回答“什么时候才能真正写‘进入正常并行开发’”。
- 后续不再允许用“某条线程说自己完成了”代替整个项目的收口判断。

## 2. L1-L5 验收层

## L1. 规则与入口层
- 必须满足：
  - `Sunset当前唯一状态说明_2026-03-13.md`
  - `Sunset Git系统现行规则与场景示例_2026-03-15.md`
  - `线程分支对照表.md / .json`
  - 活跃线程总表
  - 文件认领与锁定表
  这些入口之间没有互相打架的口径。
- 不通过时的输出：
  - 明确是哪两份文档冲突
  - 说明谁是现行入口，谁需要降级或修正

## L2. Git 与工作树归属层
- 必须满足：
  - 当前 tracked dirty / untracked 都已归属
  - 默认基线与线程私有 WIP 边界明确
  - 不存在“来源不明、谁都说不清”的改动
- 不通过时的输出：
  - 逐项列出未归属路径
  - 直接指定 owner 线程或标记“待统一快照确认”

## L3. 编译与 Console 层
- 必须满足：
  - 没有新的项目级红编译
  - Console 没有新的阻断性错误
  - warning 已按责任线程分流，不再被泛化成“项目没恢复”
- 注意：
  - 运行态提示、工具链断开提示、已知共享 warning，不自动等于项目级失败
  - 但若影响当前主线验收，必须在对应线程里单独上提
- 不通过时的输出：
  - 指明是项目级阻断、线程级阻断，还是工具链级阻断

## L4. Live 场景与关键链路层
- 必须满足：
  - `unityMCP` 能稳定读取活动场景与 Console
  - 当前主场景关键链路存在并能被现场核到
  - 关键组件引用若为空，必须明确是自动补线设计还是实际缺口
- 当前已知示例：
  - `Primary` 当前可读取
  - `NPCs/001` 当前可读取且挂有 `NPCDialogueInteractable`
  - `DialogueCanvas` 当前可读取，但 `DialogueUI.fontLibrary=null` 仍属真实缺口
- 不通过时的输出：
  - 直接列出缺失对象、空引用对象或行为级缺口

## L5. 正常并行开发准入层
- 只有在 L1-L4 都至少达到“无项目级阻断”的前提下，才能写“进入正常并行开发”。
- 额外必须满足：
  - 所有活跃线程都按统一模板完成过至少一轮新快照
  - 用户不再需要手动在不同线程之间转述现场状态
- 不通过时的输出：
  - 明确写“当前仍处于恢复后过渡态”，并列出唯一剩余准入缺口

## 3. 当前最小达标定义
- 当前不是要求所有线程都零 warning、零 WIP。
- 当前要求的是：
  - 规则唯一
  - 归属清楚
  - 项目不被新的红编译卡死
  - live 场景能验证关键链路
  - 用户不再充当中继站

## 4. 当前使用方式
- 每次准备宣布“某一阶段收口完成”前，先按 L1-L5 过一遍。
- 若任何一层失败，结论必须停在那一层，不允许越级宣布“总恢复通过”。
