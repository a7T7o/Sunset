***### 一、核心差异本质：kiro vs Claude 规则执行逻辑***

***你感知到的核心差异可以总结为 \*\*“规则加载粒度”和“调度机制”\*\* 的原生能力不同：***

***| 维度                | kiro 机制                                  | Claude 原生机制                          |***

***|---------------------|--------------------------------------------|------------------------------------------|***

***| 规则加载时机        | 每个对话/任务重新加载规则（会话级）| 全局规则（Claude.md）持久生效，无原生会话级加载 |***

***| 规则调度            | 内置智能匹配、自动/手动加载逻辑（钩子触发） | 无原生调度能力，仅遵循 system prompt + 对话上下文 |***

***| 规则作用域          | 可按场景（specs迭代/审核补丁等）动态切换   | 全局规则固定，会话上下文需手动注入场景规则 |***



***要让Claude复刻kiro的工作模式，核心是\*\*用“Claude的Prompt分层 + 命令行脚本封装”模拟kiro的会话级加载/智能匹配\*\*，同时把.kiro的规则体系适配到Claude的Prompt体系中。***



***### 二、Claude规则体系快速复盘（结合你的参考文档）***

***你参考的CSDN文档核心是：***

***- Claude的`Claude.md`本质是\*\*全局System Prompt（系统提示词）\*\*，所有对话都会优先加载这份规则；***

***- Claude无“自动钩子/动态加载”能力，但可以通过「命令行参数注入Prompt」「脚本封装规则加载逻辑」来模拟；***

***- 你在VSCode命令行使用的“全局Claude”，本质是Claude CLI工具（或VSCode插件调用Claude API），核心是通过CLI参数/脚本控制Prompt输入。***



***### 三、对齐kiro工作模式的完整解决方案（VSCode命令行版）***

***#### 前提：目录结构约定（基于你的.kiro）***

***```***

***.kiro/***

***├── hooks/          # 自动钩子（后续封装为脚本触发逻辑）***

***├── specs/          # 项目核心（迭代文件/三件套/锐评回应/审核补丁）***

***└── steering/       # 核心规则***

    ***├── rules.md    # 总规则（全局基础）***

    ***├── communication.md  # 对话级规则（会话级动态加载）***

    ***└── 000-context-recovery.md  # 上下文恢复规则（会话级前置）***

***```***



***#### 步骤1：规则文件重构（适配Claude的Prompt分层）***

***将.kiro的规则拆分为「全局规则」和「会话级规则」，对应Claude的System Prompt和会话前置Prompt：***

***| 规则类型       | 对应文件                          | 作用                          |***

***|----------------|-----------------------------------|-------------------------------|***

***| 全局规则       | .kiro/steering/rules.md           | 持久生效（如核心开发规范）    |***

***| 会话级前置规则 | .kiro/steering/000-context-recovery.md | 每次对话先加载（上下文恢复）  |***

***| 会话级动态规则 | .kiro/steering/communication.md   | 每次对话加载（对话交互规范）  |***

***| 场景化规则     | .kiro/specs/\[场景].md（如审核补丁.md） | 按需求智能匹配（触发式加载）  |***

***| 钩子规则       | .kiro/hooks/\[hook].sh             | 封装为脚本，触发规则加载      |***



***#### 步骤2：配置Claude全局规则（对应kiro的基础规则）***

***Claude的`Claude.md`（全局System Prompt）等价于kiro的“总规则”，需将`.kiro/steering/rules.md`内容注入：***



***##### 操作（Claude CLI版）：***

***1. 找到Claude CLI的配置文件（通常在`~/.claude/config.toml`或通过`claude config`查看）；***

***2. 修改`system\_prompt`字段，将`.kiro/steering/rules.md`的内容完整粘贴进去：***

   ***```toml***

   ***# ~/.claude/config.toml***

   ***\[default]***

   ***api\_key = "你的Claude API Key"***

   ***model = "claude-3-5-sonnet-20240620"***

   ***system\_prompt = """***

   ***# 全局核心规则（来自.kiro/steering/rules.md）***

   ***【粘贴rules.md的所有内容】***

   ***"""***

   ***```***

***3. 验证：执行`claude chat`，Claude会默认遵守这些全局规则（和kiro的基础规则对齐）。***



***#### 步骤3：封装会话级规则加载脚本（模拟kiro的对话级加载）***

***kiro“每个对话重新加载规则”的核心，可通过\*\*脚本自动拼接会话级规则+用户输入\*\* 实现。***



***##### 步骤3.1：编写加载脚本（命名为`kiro-claude.sh`，放在.kiro目录下）***

***```bash***

***#!/bin/bash***

***# 功能：模拟kiro的会话级规则加载，自动拼接上下文恢复+对话规则+用户输入***



***# 1. 加载会话级前置规则（上下文恢复）***

***CONTEXT\_RECOVERY=$(cat .kiro/steering/000-context-recovery.md)***

***# 2. 加载对话级核心规则***

***COMMUNICATION=$(cat .kiro/steering/communication.md)***

***# 3. 读取用户输入（如果是交互式，或从命令行参数读取）***

***USER\_INPUT="$\*"***



***# 4. 拼接完整Prompt（规则前置 + 用户输入）***

***FULL\_PROMPT="***

***# 会话级规则（上下文恢复）***

***$CONTEXT\_RECOVERY***



***# 会话级交互规则***

***$COMMUNICATION***



***# 用户当前需求***

***$USER\_INPUT***

***"***



***# 5. 调用Claude CLI，传入拼接后的Prompt***

***claude chat -m "claude-3-5-sonnet-20240620" -p "$FULL\_PROMPT"***

***```***



***##### 步骤3.2：赋予脚本执行权限***

***在VSCode命令行执行：***

***```bash***

***chmod +x .kiro/kiro-claude.sh***

***```***



***#### 步骤4：实现规则的智能匹配（模拟kiro的智能调用）***

***kiro能“智能匹配specs场景（迭代/审核/补丁）”，可通过\*\*脚本参数+场景规则模板\*\* 实现：***



***##### 步骤4.1：场景规则模板化（在.kiro/specs下创建场景映射）***

***```***

***.kiro/specs/***

***├── template-迭代文件.md    # 迭代文件场景规则***

***├── template-审核补丁.md    # 审核补丁场景规则***

***├── template-锐评回应.md    # 锐评回应场景规则***

***└── scene-mapping.sh        # 场景匹配逻辑***

***```***



***##### 步骤4.2：修改`kiro-claude.sh`，增加场景匹配逻辑***

***```bash***

***#!/bin/bash***

***# 新增：场景参数（如 --scene=审核补丁）***

***SCENE=""***

***USER\_INPUT=""***



***# 解析命令行参数（区分场景和用户输入）***

***while \[\[ $# -gt 0 ]]; do***

  ***case "$1" in***

    ***--scene=\*)***

      ***SCENE="${1#\*=}"***

      ***shift***

      ***;;***

    ***\*)***

      ***USER\_INPUT+="$1 "***

      ***shift***

      ***;;***

  ***esac***

***done***



***# 加载基础会话规则（上下文恢复+交互规则）***

***CONTEXT\_RECOVERY=$(cat .kiro/steering/000-context-recovery.md)***

***COMMUNICATION=$(cat .kiro/steering/communication.md)***



***# 智能加载场景化规则（模拟kiro的场景匹配）***

***SCENE\_PROMPT=""***

***if \[ -n "$SCENE" ]; then***

  ***SCENE\_FILE=".kiro/specs/template-$SCENE.md"***

  ***if \[ -f "$SCENE\_FILE" ]; then***

    ***SCENE\_PROMPT=$(cat "$SCENE\_FILE")***

  ***else***

    ***SCENE\_PROMPT="# 未匹配到场景规则：$SCENE"***

  ***fi***

***fi***



***# 拼接最终Prompt***

***FULL\_PROMPT="***

***# 会话级规则（上下文恢复）***

***$CONTEXT\_RECOVERY***



***# 会话级交互规则***

***$COMMUNICATION***



***# 场景化规则（$SCENE）***

***$SCENE\_PROMPT***



***# 用户当前需求***

***$USER\_INPUT***

***"***



***# 调用Claude***

***claude chat -m "claude-3-5-sonnet-20240620" -p "$FULL\_PROMPT"***

***```***

***#### 步骤6：模拟kiro的“自动钩子（hook）”***

***kiro的自动钩子可通过\*\*脚本触发+规则注入\*\* 实现，比如“当修改specs文件时自动加载对应规则”：***



***1. 在.kiro/hooks下创建`pre-specs-hook.sh`：***

   ***```bash***

   ***#!/bin/bash***

   ***# 钩子逻辑：检测specs文件修改，自动加载对应规则并调用Claude***

   ***MODIFIED\_FILE=$1***

   ***SCENE=$(echo $MODIFIED\_FILE | grep -oE "迭代|审核|锐评") # 提取场景关键词***

   ***if \[ -n "$SCENE" ]; then***

     ***.kiro/kiro-claude.sh --scene=$SCENE "分析修改后的文件：$MODIFIED\_FILE，检查是否符合规则"***

   ***fi***

   ***```***

***2. 在VSCode中配置文件保存钩子（通过VSCode插件如`File Watcher`），当修改.kiro/specs下的文件时，自动执行`pre-specs-hook.sh`。***



***### 四、验证与调优（适配specs开发场景）***

***1. \*\*基础验证\*\*：执行`kiro "列出当前生效的所有规则"`，检查Claude是否返回.kiro的全局规则+会话级规则；***

***2. \*\*场景验证\*\*：执行`kiro --scene=审核补丁 "分析specs/补丁方案.md"`，检查Claude是否加载对应场景规则并分析文档；***

***3. \*\*调优\*\*：***

   ***- 如果规则过长导致Claude上下文溢出，拆分规则为“核心规则+按需引用”（在Prompt中用“参考规则：\[规则ID]”，脚本按需加载）；***

   ***- 如果智能匹配不准确，优化`scene-mapping.sh`的关键词匹配逻辑（如用正则/模糊匹配）。***



***### 五、关键注意事项***

***1. \*\*Claude上下文限制\*\*：Claude-3-5-Sonnet的上下文上限是200k tokens，需控制规则文件大小（避免拼接后Prompt超限）；***

***2. \*\*API Key安全\*\*：Claude CLI的API Key不要硬编码在脚本中，通过环境变量`export CLAUDE\_API\_KEY="你的Key"`配置；***

***3. \*\*规则同步\*\*：修改.kiro的规则后，需重启Claude CLI（或重新加载配置）确保全局规则生效；***

***4. \*\*VSCode终端兼容\*\*：如果使用PowerShell（而非bash/zsh），需将脚本改为`.ps1`格式（核心逻辑不变，语法适配PowerShell）。***
