## 本轮唯一主刀

把 `CloudShadowManager` 当前已有的云运行态数据，接入 Sunset 的正式存档系统。

这轮只做“一刀可验”的结果：

- 同一次游戏里保存
- 退出到主菜单或重载场景后读档
- `Primary` / `Town` 各自恢复到保存时的云状态

不要把这轮扩成“所有天气/光影/特效系统统一存档重构”。

## 建议实施方向

默认优先沿用 `CloudShadowManager` 现有运行态结构，不另造第二套含义重复的数据模型。

你需要确认并落地这件事：

1. `CloudShadowManager` 对外提供最小可存/可恢复数据：
   - 每个场景一个独立 state
   - 至少包含：
     - scene key
     - seed
     - cloudIdCounter
     - lastSpawnTime
     - lastSpawnFailureLogTime
     - consecutiveSpawnFailures
     - clouds[]
   - 每朵云至少包含：
     - position
     - spriteIndex
     - scale
     - id
     - lifetime
     - stuckTime
2. 这些数据进入 `SaveDataDTOs`
3. `SaveManager` 写档时收集云状态
4. `SaveManager` 读档后把云状态回灌给对应场景的 `CloudShadowManager`
5. 如果场景尚未加载：
   - 允许先存在 save runtime cache
   - 等对应 scene 的 `CloudShadowManager` 初始化时再吃进去
