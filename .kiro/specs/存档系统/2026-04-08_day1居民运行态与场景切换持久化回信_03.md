## 这轮实际做成了什么

1. resident 最小运行态 contract 已经落地
   - 在 `SpringDay1NpcCrowdDirector` 新增了 `ResidentRuntimeSnapshotEntry`。
   - 当前正式持久化的最小字段是：
     - `npcId`
     - `sceneName`
     - `residentGroupKey`
     - `resolvedAnchorName`
     - `positionX / positionY`
     - `facingX / facingY`
     - `isActive`
     - `isReturningHome`
     - `underDirectorCue`
   - 这条 contract 明确只处理“回到稳定 resident 节点”，没有扩成“恢复任意导演帧”。

2. resident 运行态已经同时接进“跨场景运行时缓存 + 正式槽位存档”
   - `SpringDay1NpcCrowdDirector` 现在会在场景切换前缓存当前 scene resident 最小运行态。
   - 同一个 session 里 `Town -> Primary -> Town` 时，resident 会优先吃这份 runtime cache，而不是直接掉回 scene 编辑态起点。
   - `StoryProgressPersistenceService` 现在会把这份 resident snapshot 跟 `Day1` 长期剧情态一起序列化进 `StoryProgressState`。
   - 读档后，这份 resident snapshot 会回灌回 `SpringDay1NpcCrowdDirector`，让当前场景先恢复，未激活场景留待下次 scene bind 时接管。

3. 普通读档现在已经按 `sceneName` 切场
   - `SaveManager` 不再把不同场景的玩家坐标硬塞进当前 scene。
   - 如果槽位里的 `player.sceneName` 和当前 scene 不一致，现在会先 `LoadSceneAsync(targetScene)`，等 scene ready 后再真正 apply save。
   - 这条链同时修了一个老坑：
     - 玩家 scene 名现在按 `SceneManager.GetActiveScene().name` 保存，不再错误吃到 `DontDestroyOnLoad`。

4. 玩家复位链已经收掉那条 debug 味很重的暴力瞬移
   - `SaveManager.RestorePlayerData()` 现在优先走 `PersistentPlayerSceneBridge.TryApplyLoadedPlayerState(...)`。
   - 也就是先取消导航，再通过持久玩家桥稳定写回位置，而不是继续走“禁 Animator / 改 Rigidbody type / 递归清 Tool / 下一帧抓内鬼”的旧链。

5. scene-rebind 启动责任面已先真收一刀
   - `PersistentPlayerSceneBridge` 现在避免了初始场景的重复 `RebindScene()`。
   - 也就是把 `BeforeSceneLoad -> sceneLoaded -> Start` 这条链里，初始 scene 的双重 rebind 先去掉了。
   - 这不是完整性能总治理，但确实把一条明确重复重活拿掉了。

## 现在还没做成什么

1. 这轮没有把 scene-rebind 的所有重操作都拆掉
   - 还没有继续下切 `RebindPersistentCoreUi()` 的延后重绑。
   - 也没有继续把 traversal/nav 那一层再做更细的按需刷新。
   - 当前只先去掉“初始场景重复 rebind”这一刀。

2. 这轮没有去接“任意导演帧恢复”
   - 不存 typing
   - 不存 bubble
   - 不存当前 formal 对白播到哪一句
   - 不存导演 path 中途帧

3. 这轮没有替 `spring-day1` 接最后一跳导演语义
   - 当前 resident snapshot 只保证“别回编辑态起点 + 保留合法 resident 节点语义”。
   - 如果未来要做更细的导演桥接，仍然应由 `spring-day1` own 最后一跳语义恢复。

## 当前 save contract 已经能覆盖哪些 Day1 case

1. 已完成 formal/dialogue 不因恢复而重播
2. `StoryManager.CurrentPhase`、语言解码、关系值、工作台首次提示、HP/精力、Day1 教学长期态会恢复
3. resident 不再只能依赖 scene 原始编辑位
4. `Town -> Primary -> Town` 同 session 切场后，resident 有最小 runtime cache 可回灌
5. 槽位 save/load 会带 resident 最小运行态
6. `Primary` 读 `Town` 档这类跨场景读档，不再默认把坐标硬写进错误 scene

## 还不能覆盖哪些 case

2. formal 对白播到半句中途恢复
3. typing / bubble / nearby / selfTalk / walkAway 过程态恢复
4. `spring-day1` 最后一跳导演语义级桥接
5. scene-rebind 全量性能问题的彻底治理
