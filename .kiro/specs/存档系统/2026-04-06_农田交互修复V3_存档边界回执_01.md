- 列表：
  - `SaveManager.LoadGame()` 当前恢复顺序：`gameTime -> player -> 旧 inventory 兼容壳 -> PersistentObjectRegistry.RestoreAllFromSaveData(worldObjects)`；尤其是“时间先于世界对象恢复”这条不能乱。
  - `PersistentObjectRegistry` 的主语义：
    - 原地读档走 `PruneStaleRecords()`，不能退回 `Clear()`
    - `TryRegister()` 的 GUID 冲突自愈语义
    - 反向修剪（存档里没有、现场有的对象要被禁用/销毁）
  - `WorldObjectSaveData` 的三元契约：`objectType + prefabId + genericData`
  - `DynamicObjectFactory` 的恢复链：`实例化(通常先 inactive) -> 设置 GUID -> 注册 -> Load -> 再激活/进入 Start`
  - 现有 runtime item 持久化闭环：`InventoryService / EquipmentService / ChestController / WorldItemPickup` 的耐久、动态属性、数量、品质
  - 已经接入 `IPersistentObject` 的旧基线对象：
    - `InventoryService`
    - `EquipmentService`
    - `FarmTileManager`
    - `CropController`
    - `ChestController`
    - `WorldItemPickup`
    - `TreeController`
    - `StoneController`
