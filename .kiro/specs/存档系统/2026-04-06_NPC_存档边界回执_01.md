## 一句话结论

- 你对 NPC 存档边界的总判断：第一版存档只该收 `NPC 长期进度`，也就是 `关系阶段 + formal 是否已消费 + 会影响后续正式内容是否出现的耐久进度`；`聊天过程态 / 气泡态 / 近身轻反馈态 / 临时站位态` 都不该直接存。

- 列表：

- formal 对话是否已消费

- resume intro cooldown / informal chat cooldown
- 当前气泡文本
- 当前正在打字 / 等 NPC 回复 / 自动续聊中的 session state
- `LastInterrupted* / LastCompleted* / LastResume*` 这类本轮会话痕迹
- `PlayerNpcNearbyFeedbackService` 最近一次触发痕迹
- 当前 NPC 临时互动焦点
- NPC 当前临时站位 / 临时面向 / 当前正在显示哪颗气泡

## formal / informal 的关键边界

- 结论：`formal consumed` 属于长期进度，应该持久化；`informal` 的聊天过程不属于长期进度，不该直接续存。正确口径是“存正式内容是否已一次性消费”，不是“存玩家上次闲聊到第几句、还剩几秒回复、当前泡泡停在哪个字”。

- 一句话理由：
