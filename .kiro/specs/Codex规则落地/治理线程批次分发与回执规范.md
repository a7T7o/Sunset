# 治理线程批次分发与回执规范

## 文件身份
- 本文件只约束 `Codex规则落地` 这条治理线程。
- 它不约束业务线程如何写功能。
- 它只回答一件事：
  - 当治理线程需要向多个线程分发 prompt，并要求回写收件单时，应该怎么做。

## 适用场景
- 同一轮里要向多个 Sunset 线程分发专属 prompt。
- 需要线程自己去领取对应文件，而不是让我手工逐条转抄。
- 需要把线程回复压缩到最小格式，减少聊天上下文污染。
- 需要从固定回收卡里收件，而不是靠聊天里长回复做判断。

## 硬规则
1. 这类“批次分发文件”必须按轮次新建。
2. 批次分发文件必须放在：
   - `D:\Unity\Unity_learning\Sunset\.kiro\specs\Codex规则落地\`
3. 不要把批次分发文件固化在某个阶段目录里。
4. 阶段目录只保留：
   - 线程专属 prompt
   - 固定回收卡
   - 阶段文档与治理裁定
5. 批次分发文件不是长期维护资产。
6. 一轮分发结束后：
   - 可以保留为本轮证据
   - 也可以在后续收口时删除或归档
   - 但不能把它表述成“以后一直复用的固定 prompt”

## 批次分发文件必须包含什么
- 本轮批次名称或用途
- 线程要去哪个目录领取自己的专属文件
- 线程必须把结果回写到哪个固定回收卡
- 线程在聊天里只能回复哪些最小字段
- 本轮禁止事项

## 推荐最小回复格式
- 已回写文件路径
- 是否完成本轮要求
- 一句话摘要

不要让线程在聊天里长篇复述回收卡内容。

## 治理线程自己的动作顺序
1. 先确认当前命中的正式工作区与本轮阶段。
2. 如果要群发，先在 `Codex规则落地` 根层生成一份本轮批次分发文件。
3. 再让线程去各自阶段目录领取同名专属 prompt。
4. 收件时优先读取固定回收卡，不依赖聊天长回复。
5. 做完裁定后，再决定是否进入下一轮串行准入。

## 配套技能口径
- 这条规范应由治理线程专用 skill 触发。
- 如果当前会话里该 skill 还不可见，治理线程也必须手工执行等价流程。

## 一句话口径
- 批次分发文件是治理线程的“本轮临时入口”，不是阶段资产，更不是永久模板。
