1. 先发 `导航检查` 的阶段二。
2. 等 `导航检查` 完成 `grant -> ensure-branch -> docs checkpoint -> return-main` 闭环后，再开启下一个真实写入槽位。

## 为什么这样排
- `导航检查` 的首个 checkpoint 是文档固化，不需要先碰 Unity / Play Mode / 热文件，最适合做阶段二低风险验闸。
- `农田交互修复V2`、`spring-day1`、`NPC` 都更贴近真实业务推进，但它们都更容易触发 Unity / MCP / Play Mode 或热文件风险，所以不建议一起放行。
- `遮挡检查` 的最小整改边界虽然可控，但一旦扩围，很容易触到 Prefab、代码链路甚至热文件，所以不适合作为第一槽位。
