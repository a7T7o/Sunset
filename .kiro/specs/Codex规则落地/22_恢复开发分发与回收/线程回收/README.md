# 线程回收目录说明

## 用途
- 本目录是阶段 22 的统一收件箱。
- 后续 5 个线程的阶段一 / 阶段二结果，不再只停留在聊天里，必须回写到这里的固定文件。

## 回写规则
- 每个线程只写自己的文件。
- 只填写“由线程填写”的区块，不要改固定头部说明。
- 如果现场阻断导致无法继续，也必须把阻断写进对应区块。
- 如果当前状态已经进入“等用户终验”，不能只写一句“等用户复测”：
  - 必须先写清线程已完成哪些自验
  - 必须附验收步骤、预期结果、失败判读
  - 必须附可直接填写的验收回执单
- 如果线程做过 Unity / MCP live 取证，也必须写清：
  - 取证目标是什么
  - 是否在拿到证据后立刻 `Pause / Stop`
  - 当前是否已退回 `Edit Mode`
- 回写完成后，线程在聊天里只需要做一句简短确认：
  - 已回写到哪个文件
  - 当前是阶段一还是阶段二
  - 一句话摘要

## 治理线程后续会做什么
- 我会优先读取这里的文件，而不是再次从聊天里人工摘证据。
- 阶段一回收后，我会在同一文件的“治理裁定区”写明：
  - 是否允许进入阶段二
  - 若允许，允许的最小 checkpoint
  - 若不允许，阻断点是什么
