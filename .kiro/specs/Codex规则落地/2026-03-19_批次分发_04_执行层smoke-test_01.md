# 批次分发｜2026-03-19｜执行层 smoke test 01

## 本轮用途
- 本轮不是继续复用旧的 queue-aware 修复后重测 prompt。
- 本轮是执行层新口径上线后的第一轮 smoke test，专门验证三件事：
  - `request-branch` 在并发抢槽时能否稳定返回 `GRANTED / LOCKED_PLEASE_YIELD`
  - waiting 线程能否把恢复点安全放进 Draft 沙盒，而不是再次写脏 `main`
  - `return-main` 是否会按队列状态输出正确的 `POST_RETURN_EVIDENCE_MODE`

## 本轮硬规则
1. shared root live 调度命令一律使用稳定 launcher：
   - `powershell -ExecutionPolicy Bypass -File C:\Users\aTo\.codex\tools\sunset-git-safe-sync.ps1`
2. 如返回 `GRANTED` 或 `ALREADY_GRANTED`，才允许继续 `ensure-branch`。
3. 如返回 `LOCKED_PLEASE_YIELD`：
   - 不得继续重试 `ensure-branch`
   - 不得在 Sunset 仓库里写任何 tracked 文件
   - 必须把最小恢复点写入脚本输出提供的 `DRAFT_FILE_HINT`，或写入 `.codex/drafts/<OwnerThread>/`
   - 只做最小聊天回执，然后停下等待
4. 如成功进入分支：
   - 本轮只做只读最小核对
   - 不做业务代码 / 场景 / Prefab / 资源修改
   - 完成后立即 `return-main`
   - 读取 `POST_RETURN_EVIDENCE_MODE`
   - 即使 `return-main` 成功，本轮也不要写 tracked 回执；如果需要补充复盘，继续放 Draft
5. 本轮全部线程默认禁止：
   - `Unity / MCP / Play Mode`
   - `Primary.unity`
   - `GameInputManager.cs`
   - 长篇聊天复述

## 一句话口径
- 这轮 smoke test 测的是“并发申请 + Draft 挂起 + return-main 分流”，不是业务功能落地。
