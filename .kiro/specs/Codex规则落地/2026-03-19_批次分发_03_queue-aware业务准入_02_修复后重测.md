## 固定收件箱
- 本轮固定收件箱仍保留在：
  - `D:\Unity\Unity_learning\Sunset\.kiro\specs\Codex规则落地\23_前序阶段补漏审计与并发交通调度重建\2026.03.19_queue-aware业务准入_02_修复后重测\线程回收\`
- 但本轮有一个新增硬规则：
  - 业务线程本轮不要直接改这些 tracked 收件卡文件。
  - 原因：等待态线程一旦在 `main` 上写 tracked 文件，会把 shared root 再次写脏，直接污染 queue 实盘。
  - 因此本轮收件卡由治理线程根据聊天最小回执统一回填。

## 聊天最小回执
- 线程在聊天里只回复：
  - 已读取的专属 prompt 路径
  - `request-branch` 结果
  - `ensure-branch` 是否成功
  - `return-main` 是否成功
  - 一句话摘要
- 如果拿到 `LOCKED_PLEASE_YIELD`，允许额外补：
  - `ticket`
  - `queue_position`

## 本轮硬规则
1. shared root live 调度命令一律使用稳定 launcher：
   - `powershell -ExecutionPolicy Bypass -File C:\Users\aTo\.codex\tools\sunset-git-safe-sync.ps1`
2. 如返回 `GRANTED` 或 `ALREADY_GRANTED`，才允许继续 `ensure-branch`。
3. 如返回 `LOCKED_PLEASE_YIELD`：
   - 不得继续重试 `ensure-branch`
   - 不得在 Sunset 仓库里写任何 tracked 文件
   - 不得直接改收件卡
   - 只做最小聊天回执，然后停下等待治理线程后续唤醒
4. 如成功进入分支：
   - 本轮只做“只读最小核对”
   - 不做业务代码/场景/Prefab/资源修改
   - 不写 tracked 回收卡
   - 核对完立即 `return-main`
5. 本轮全部线程默认禁止：
   - `Unity / MCP / Play Mode`
   - `Primary.unity`
   - `GameInputManager.cs`
