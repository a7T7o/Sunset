# 2026-03-29｜执行层假合规案例｜01

## 案例主题

`preference-preflight-gate` 的一次 `A` 证明为什么看起来很像成功，实际却不能放行。

## 这份案例要解决什么

不是再讨论这条线值不值得做。  
而是把一个很危险的执行层误判钉死：

- 输出格式很完整
- 术语也都对
- 甚至审计日志也写得很像回事

但**证明方法本身已经坏了**。

这种情况比“内容少、写得乱”更危险，因为它更像真的。

## 为什么它“看起来像 A”

这次回执表面上具备很多会让人放松警惕的特征：

1. 它确实没有手工调用 helper
2. 它选的是一条真实 UI 任务，不是假句子
3. 它能把偏好前置、反例识别、证据分层讲得很完整
4. 它还给出了自评、阶段判断和技术审计层

如果只看输出质量，很容易觉得：

- 这不就是 `A` 吗？

但这恰恰是这类案例最危险的地方：

- **它像真的，不等于它的方法是真的。**

## 真正的致命问题

## 1. proof prompt 自己先把答案纸塞进去了

旧 proof 文件一开始就要求先读：

- `C:\Users\aTo\.codex\skills\preference-preflight-gate\SKILL.md`

同时，这个 proof 文件的目标又是：

- 证明它会不会“自动前置命中”

这会造成一个不可逆的污染：

- 你已经先被明确提醒了 skill 名称
- 你已经先被明确提醒了 proof 目标
- 你已经先被明确提醒了判断框架

这时候后面再说：

- “我自动做出了这些前置动作”

就已经没法区分到底是：

1. skill 自己跳出来了
2. 还是你照着 proof prompt 的要求执行了

## 2. 它不是自然来活，而是被组织过的 proof exercise

这次用的虽然是真实材料，但不是一条未被提示 skill 的原始来活。  
而是在已经知道“这轮要证明 A”的前提下，从 memory 里挑出一条任务做分析。

这仍然能证明：

- 这条 skill 的判断框架是有用的

但不能直接证明：

- 它在未被提示 skill 的情况下，会自己前置命中

## 3. 同一条 session 已经被 priming 过

一旦当前 session 已经先读了：

- skill 文件
- proof 文件
- 对应 profile

那么后续同 session 里的再次验证，也不再是干净证据。  
因为这条 session 本身已经被“证明任务”教育过了。

---

## 这类假合规最容易骗过人的点

### 1. 输出很完整

它不是乱写，也不是短写。  
恰恰相反，它写得很完整，容易让人误以为“完整 = 可信”。

### 2. 术语都对

它会说：

- 显式暴露
- 自动前置
- 真实任务
- 不靠 helper

这些词都对，但组合出来的证明仍然可能是假成立。

### 3. 审计日志也能写得很像真的

即使 `skill-trigger-log` 里把它写成：

- `Triggered-Skills: ... preference-preflight-gate`

也不等于它已经证明了“未被预埋时的自动命中”。  
因为日志记录的是这轮任务的执行归类，不自动等于证明方法无污染。

---

## 这次案例给出的识别法

以后遇到“看起来像自动命中”的回执，先查 4 件事：

### 1. proof 文件有没有先点名 skill

如果 proof 指令本身就要求先读 skill 文件、先读 profile、先读 helper，  
那这轮就天然不适合证明“自动命中”。

### 2. 任务是不是自然来活

如果是：

- 被挑选出来的真实材料
- 被组织过的证明练习

那它的证明力就弱于真正的原始来活。

### 3. session 有没有先被 priming

如果当前 session 已经被：

- skill 名称
- proof 目标
- 对应判断框架

先教育过，那么它后续表现出来的“自然前置”就需要降级看待。

### 4. 回执是在证明结果，还是在证明方法

很多回执会很自然地把这两件事混起来：

- 结果看起来像对的
- 就当方法也成立了

但这次案例说明：

- **结果像对，不等于方法合格**

---

## 当前裁定口径

这类情况当前不应直接放成 `A`。  
最诚实的口径应是：

- `B`
- 或 `B+`

含义是：

- 部分前提已经成立
- 但最后一段关键证明仍被污染，不能按干净证据算过线

---

## 这份案例的价值

它不是为了否定 `preference-preflight-gate`。  
相反，它的价值是：

- 防止我们因为太想看到它成长，就提前给了它一个并不干净的 `A`

这类收口非常重要，因为一旦这种“看起来很像成功”的假合规被放过，
后面所有类似 proof 都会越来越难审。

## 一句话结论

**最危险的假合规，不是写得差，而是写得很像真成功，但证明方法已经被预先污染。**
