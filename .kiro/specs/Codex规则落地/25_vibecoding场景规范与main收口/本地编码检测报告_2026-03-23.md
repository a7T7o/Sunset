# 本地编码检测报告 2026-03-23

## 1. 来源与学习结论
- 参考页面：
  - [〖解决 Codex 修改文件后中文乱码疑问：根源在终端编码，不在 VS Code〗](https://www.cnblogs.com/gccbuaa/p/19227315)
- 该文核心结论：
  - 问题根因不是编辑器单点，而是“终端写入编码”和“编辑器读取编码”不一致。
  - 在 Windows 下，最危险的组合是：
    - 编辑器默认按 UTF-8 读写
    - PowerShell 仍按旧默认编码或混乱编码执行写入
  - 要真正解决，必须同时统一：
    - 编辑器编码
    - 终端控制台编码
    - `$OutputEncoding`
    - PowerShell 文件写入默认编码
  - 推荐做法：
    - 显式设置 VS Code `"files.encoding": "utf8"`
    - 在 PowerShell Profile 中统一 `chcp 65001`、`[Console]::InputEncoding`、`[Console]::OutputEncoding`、`$OutputEncoding`、`$PSDefaultParameterValues`
    - Profile 文件本身使用 UTF-8 无 BOM 写入
