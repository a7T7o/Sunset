# 恢复开发总控与线程放行规范 - 2026-03-21

## 文件身份
- 这份文件是当前这一轮恢复开发的总控口径。
- 它回答的是三件事：
  - 哪些线程现在可以开发。
  - 哪些线程必须等待。
  - 你作为项目经理应该按什么顺序发 prompt、收回执、再决定下一条。
- 后续如果任何线程回执改变了风险、顺序或边界，治理线程必须先更新这份文件，再告诉你新的口径。

## 当前总原则
1. shared root 业务线程按单槽位串行推进。
2. 每条 shared root 线程做完一个最小 checkpoint 后，必须先回执给治理线程，再决定下一条。
3. Unity / MCP 仍然是全局单写者模型。
4. `scene-build` 是当前唯一被放行持续使用 Unity / MCP 的特殊线程，但它不等于“其他线程可以一起进 Unity / MCP”。
5. 治理线程负责更新规范、更新 prompt、裁定下一条，不让你靠旧聊天内容自行猜测。

## 给项目经理的最简执行法
1. 默认一次只发一条 shared root prompt。
2. 当前如果 `scene-build` 正在施工，其他线程不得自行进入 Unity / MCP。
3. 某条线程回执回来后，把回执贴给治理线程。
4. 治理线程会做三件事：
   - 判断这条线程是否真正收口。
   - 判断下一条能不能发。
   - 如果口径变了，直接更新本文件和对应 prompt 文件，再告诉你。
5. 你不要复用旧 prompt；以后都以本轮目录下的“当前开发放行”文件为准。

   - 必要的治理记忆

   - 哪个文件已更新
   - 现在该发哪条
