# 治理线程 - 职责与更新规则

## 当前职责
- 维护当前目录作为唯一现行入口。
- 维护两类 prompt：
  - 直接开发 prompt
  - 回执 / 冻结 / 交接 prompt
- 只在真实风险出现时介入，而不是重新恢复复杂放行系统。

## 当前默认执行原则
- 正常情况下，治理线程不替业务线程做提交。
- 默认口径是：
  - 线程自己只提自己的白名单
  - 线程自己一刀一 checkpoint
  - 治理线程只负责裁定顺序和处理少数例外
- 只有命中高危撞车、跨线程耦合、或 branch carrier / worktree 迁入窗口时，治理线程才额外生成专门 prompt 干预。

## 当前命名原则
- 不要谈“改 Git 提交 id”。
- Git SHA 不可自定义。
- 统一的是 checkpoint 名称 / commit message：
  - `YYYY.MM.DD_<线程名>_<编号>`

## 当前硬打断
1. 两个线程同时改同一个高危 Scene / Prefab / 热脚本。
2. 两个线程同时要做 Unity / MCP live 写入。
3. 某线程已经把编译、场景、引用或资源关系写坏。
4. 某线程准备做破坏性 Git 动作。
