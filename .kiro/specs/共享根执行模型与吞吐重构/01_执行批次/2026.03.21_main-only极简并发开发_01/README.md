## 当前默认提交规则
- 默认就是：
  - 每个线程只提交自己修改的内容
  - 每个线程只提交自己的白名单路径
  - 每个线程一次只收一刀自己的 checkpoint
- 如果白名单里包含 `.cs` 文件：
  - `git-safe-sync.ps1` 收口前会自动跑代码闸门
  - 代码闸门没过，就不允许继续收口
- 只有命中这些少数例外时，治理线程才升级介入：
  - 撞到同一个高危目标
  - 同一个 Scene / Prefab / 热脚本被多人改
  - 跨线程强耦合
  - 成果还在 branch carrier / worktree，尚未迁入 `main`

## 当前命名规则
