## 2026-04-04 只读补记：Deepseek 系 Day1 与 NPC Town 化证据

### 用户目标
- 用 Deepseek/Gemini 原案反查 Day1 早期剧情与角色承载，验证“NPC 后续都应该在 Town，只有村长跟着走几段剧情”是否符合原案。

### 本轮实际完成
- 对比了 `Deepseek聊天记录001.md`、`Deepseek-2-P1.md`、`省流版VIP8.md` 与 `春1日_坠落_融合版.md`。
- 收束出的稳定事实：
  - Deepseek 的原始口述要求与重写后的 Day1 剧本都把主线空间压成 `森林/矿洞口 → 村庄 → 小屋 → 小屋外农田 → 饭馆 → 回小屋`。
  - `马库斯` 是唯一持续 escort 的 NPC；`艾拉` 负责小屋治疗；`卡尔` 负责饭馆冲突。
  - `老杰克 / 老乔治 / 老汤姆 / 小米` 在 Day1 只出现在可选的 Town 探索里，属于“认脸铺垫”，不是强制主链。
  - `VIP8` 的 6 核心 NPC + 2 隐藏 roster 没把 `卡尔` 列为核心常驻；同时明确批注“NPC 日常行为路径建议改为固定位置”。

### 关键结论
- 原案更支持“Town 内固定落位 + 马库斯少量 escort”，而不支持“多个 NPC 跟着主角四处跑”的 Day1 结构。
- 需要保留的边界：
  - `艾拉` 必须在 Day1 小屋段出现；
  - `卡尔` 应保留为饭馆的一次冲突钉子；
  - 其余核心 NPC 更适合留在各自 Town 点位上等待可选接触或后续天数展开。
