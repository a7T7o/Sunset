# 龙虾工具定位与阶段目标

**创建日期**: 2026-03-10

---

## 1. 文档目的

这份文档只回答一件事：

> 在当前 `Sunset` 项目里，我们要把“龙虾”做成一个什么样的工具。

它不是功能清单，也不是技术报告，而是后续所有设计、执行、验收的总锚点。

---

## 2. 龙虾的定位

龙虾不是下面这些东西：

- 不是只会聊天的模型壳子
- 不是为了展示多 agent 链路而存在的演示玩具
- 不是脱离 `Sunset` 主项目的抽象自动化实验
- 不是一开始就追求 `24x7` 无人值守的幻想系统

龙虾当前应该被定义为：

> **一个服务于 `Sunset` 项目的、可持续接任务、可验证执行、可记录、可升级的人机协作工具。**

更直白一点说：

- 它要能接 `Sunset` 里的真实任务
- 它要能把任务做出结果
- 它要能告诉人“做到哪了、为什么这样做、还差什么”
- 它要能在做不动时清楚升级，而不是乱跑

---

## 3. 共同目标

我们当前的共同目标，不是先把龙虾做成“无限运行的 AI 生命体”，而是先把它做成：

> **一个专业、稳定、可验收的 `Sunset` 项目执行工具。**

这个目标包含 4 个关键词：

### 3.1 专业

- 能理解 `Sunset` 的工作区、规则、记忆体系
- 不乱切主线
- 不把支撑子任务当主任务
- 不因为一个新话题就偏离长期目标

### 3.2 稳定

- 遇到报错先收敛定位
- 遇到中断可以恢复
- 不靠一时的上下文记忆硬撑
- 能把关键结论写回工作区

### 3.3 可验收

- 每次推进都要有用户看得见的产物
- 要有明确的验收口径
- 要能区分“跑通一段链路”和“交付了一项能力”

### 3.4 可接任务

- 用户不需要一次性把全部任务列表交出来
- 龙虾应先具备承接单个真实任务的能力
- 再逐步扩展为承接一组任务、一个阶段、一个工作流

---

## 4. 当前阶段不应再混淆的重点

### 4.1 持续运行很重要，但不是当前唯一重点

“持续运行”是目标的一部分，但它的前提是：

- 先会接任务
- 先会交付
- 先会验收
- 先会升级

否则持续运行只会持续地产生噪音。

### 4.2 当前更优先的是“有任务可做”

在 `Sunset` 场景下，更关键的问题不是：

- 能不能把多 agent 再串长一点

而是：

- 龙虾能不能在当前项目中稳定承接一个真实任务
- 做完后有没有明确结果
- 结果能不能被用户直接验收

### 4.3 因此，当前阶段的核心不是扩功能，而是立工具骨架

当前应该优先完成的不是更多技术链路证明，而是龙虾作为工具的基础制度：

- 工具定位
- 操作手册
- 验收指南
- 最小可运行闭环

---

## 5. 龙虾应具备的基础能力

在正式引入 `tasks.md / progress.md / blockers.md` 之前，龙虾至少要先具备以下能力：

### 5.1 接任务能力

- 能判断当前任务属于哪个 `Sunset` 工作区
- 能先读规则再动手
- 能明确本轮主线目标与支撑子任务关系

### 5.2 做任务能力

- 能把一个任务收束成最小可执行动作
- 能优先产出用户可验收结果
- 能避免无关扩散

### 5.3 记任务能力

- 能把结论和恢复点写回正确位置
- 能在下一轮继续主线，而不是重新迷路

### 5.4 验任务能力

- 能区分“我做了”与“已通过验收”
- 能产出明确的验证结果
- 能说明未完成部分

### 5.5 升级能力

- 能在阻塞时准确升级给人
- 说明卡点、已做尝试、需要什么帮助、帮助后从哪继续

---

## 6. 阶段目标

### 阶段 A：专业工具定型期

目标：

- 明确龙虾是什么
- 明确它该怎么工作
- 明确怎么验收它是否够专业

本阶段产物：

- `龙虾工具定位与阶段目标.md`
- `龙虾操作手册.md`
- `验收指南.md`

### 阶段 B：最小闭环建立期

目标：

- 让龙虾能稳定承接一个 `Sunset` 真实任务
- 做到“接收 -> 执行 -> 验证 -> 记录 -> 回复”

本阶段产物：

- 最小可运行闭环说明
- 示例任务演示

### 阶段 C：任务账本化期

目标：

- 正式引入 `tasks.md / progress.md / blockers.md`
- 把工作从“会话推动”变成“账本推动”

### 阶段 D：持续工作期

目标：

- 在低风险范围内实现多轮连续推进
- 中断可恢复
- 长时运行不丢状态

---

## 7. 当前阶段的非目标

当前暂时不把下面这些当第一优先级：

- 为了展示能力而继续拉长 OpenClaw 多 agent 链路
- 一上来追求 `24x7` 无人值守
- 先堆满完整任务池
- 先做复杂自动调度器
- 先把所有 `Sunset` 任务都喂给龙虾

这些都应建立在“龙虾已经是一个专业可用工具”之后。

---

## 8. 当前阶段的成功标准

如果达到以下条件，就说明龙虾开始具备“专业工具”雏形：

1. 能清楚说明自己在 `Sunset` 项目中的定位与边界。
2. 能按照固定工作方式承接单个真实任务。
3. 能输出用户可直接验收的结果，而不是只输出过程痕迹。
4. 能把主线、子任务、阻塞、恢复点说清楚。
5. 能在多轮对话中维持同一条主线，不再被支撑问题带偏。

---

## 9. 一句话定义

> **龙虾要先成为 `Sunset` 项目里的专业任务执行工具，然后才有资格谈持续运行系统。**
