## 2. NPC 当前稳定承诺

### 2.1 状态语义

当前 `NPCAutoRoamController` 的核心状态仍然是：

- `Inactive`
- `ShortPause`
- `Moving`
- `LongPause`

当前稳定口径：

- `ShortPause`：短暂停留阶段，可触发短气泡、自言自语、短节奏停顿
- `Moving`：NPC 正在执行路径移动
- `LongPause`：长停留阶段，可承载更明显的停顿、聊天、自言自语

这套状态语义当前视为稳定基线。

### 2.2 IsMoving 语义

当前 `IsMoving` 分两层：

- `NPCAutoRoamController.IsMoving`：状态层意义上的“当前是否处于 Moving”
- `NPCMotionController.IsMoving`：动画桥接层基于速度阈值得出的“当前是否在动”

当前承诺：

- 我不会在导航线程排查期间随意改掉这两个语义
- 如果未来必须改，我会先明确同步“改的是状态语义、动画语义，还是两者都改”

### 2.3 最终位移语义

当前 NPC 最终位移仍以 `NPCAutoRoamController` 为主导：

- 路径点推进在 `NPCAutoRoamController`
- 动画和速度桥接在 `NPCMotionController`
- 最终位移主路径仍是 `rb.MovePosition(...)`

当前承诺：

- 这条 `MovePosition` 语义在导航排查阶段视为稳定
- 我不会在你们还没收住导航 P0/P1 时，又把 NPC 侧最终运动方式偷偷改成另一套

## 6. 导航线程当前真正要解决的问题

下面这些，才是现在真正属于导航线程的核心问题。

### P0

- 玩家右键自动导航时，路线遇到移动中的 NPC，当前仍会出现“玩家推着 NPC 走”
- NPC 与 NPC 在相遇时，当前仍可能撞成一团、顶住、挤压，而不是自然绕开

### P1

- 玩家路径执行面对突然经过的 NPC，没有形成稳定的动态避让
- NPC 自己的漫游执行也没有形成稳定的局部避障 / 分离 / 让行
- 缺少动态障碍占位与实时重规划闭环
- 缺少窄路会车、路口礼让、优先级处理
- 玩家 agent 与 NPC agent 还缺少更明确的优先级 / 半径 / 运动语义协商

一句话说：

不是“NPC 不会走”，而是“会走了，但不会像人一样在动态环境里互相让开”。
