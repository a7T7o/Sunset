## 1. NPC 当前已稳定的核心设计

### 1.1 状态语义

当前 `NPC` 漫游状态机仍以 [NPCAutoRoamController.cs](/D:/Unity/Unity_learning/Sunset/Assets/YYY_Scripts/Controller/NPC/NPCAutoRoamController.cs) 为唯一基线，核心状态为：

- `Inactive`
- `ShortPause`
- `Moving`
- `LongPause`

当前我对导航线程的正式口径是：

- `ShortPause`：
  - 表示短暂停留阶段
  - 允许短句气泡、自言自语、阶段性停顿
  - 是长停留触发计数链的一部分
- `Moving`：
  - 表示 NPC 正在执行路径位移
  - 当前所有动态避让接线，也都是在这个阶段参与
- `LongPause`：
  - 表示长停留阶段
  - 可能触发环境聊天、自言自语、长停顿
  - 也是当前“睡眠态 / 非移动态”的主要来源之一

### 1.2 `IsMoving` 语义

当前 `IsMoving` 语义分两层：

- [NPCAutoRoamController.cs](/D:/Unity/Unity_learning/Sunset/Assets/YYY_Scripts/Controller/NPC/NPCAutoRoamController.cs)
  - `IsMoving => state == RoamState.Moving`
- [NPCMotionController.cs](/D:/Unity/Unity_learning/Sunset/Assets/YYY_Scripts/Controller/NPC/NPCMotionController.cs)
  - `IsMoving` 是动画桥接层依据当前速度和阈值计算出的“当前是否正在动”

对导航线程的稳定承诺是：

- 我当前**不会主动更改** `NPCAutoRoamController` 的 `Moving / ShortPause / LongPause` 状态定义
- 我当前**不会主动更改** `NPCMotionController` 中 `IsMoving` 的判定语义
- 如果后续联调必须触碰这两个定义，我会先把变更范围单独通知导航线程，不会静默改

## 2. NPC 当前的最终位移语义

### 2.1 当前事实

当前 `NPC` 的最终位移执行，仍然是：

- [NPCAutoRoamController.cs](/D:/Unity/Unity_learning/Sunset/Assets/YYY_Scripts/Controller/NPC/NPCAutoRoamController.cs)
  - 在 `TickMoving(...)` 中先计算 `nextPosition`
  - 然后：
    - `motionController.SetExternalVelocity(velocity)`
    - 如果有刚体：`rb.MovePosition(nextPosition)`
    - 否则回退到 `transform.position`

而 [NPCMotionController.cs](/D:/Unity/Unity_learning/Sunset/Assets/YYY_Scripts/Controller/NPC/NPCMotionController.cs) 当前的职责仍是：

- 桥接动画
- 读取/缓存速度
- 通过 `SetExternalVelocity(...)` 接收外部速度
- 在停止时把 `rb.linearVelocity = Vector2.zero`

### 2.2 对导航线程的明确承诺

当前阶段，NPC 线程**坚持保留**下面这条位移语义：

- `NPCAutoRoamController` 负责最终路径执行
- `NPCMotionController` 负责速度桥接与动画驱动
- `NPC` 最终仍使用 `rb.MovePosition(...)` 作为主位移方式

也就是说，我当前**不会主动把 NPC 改成和玩家完全同一种最终位移写法**。  
这件事不是因为我认为它永远不该改，而是因为：

- 现在导航线程正在排查“共享 solver 是否被最终执行层覆盖”
- 如果我这边同时又把最终位移方式改了
- 导航线程就会很难判断当前 P0 失败到底是共享 solver 问题、执行层冲突问题，还是我这边又改了地基

所以这一阶段的正式承诺是：

- `MovePosition` 暂时视为 NPC 侧稳定基线
- 除非导航线程明确要求进入“统一最终运动语义”联调阶段
- 否则 NPC 线程不会自己抢跑把它改掉
