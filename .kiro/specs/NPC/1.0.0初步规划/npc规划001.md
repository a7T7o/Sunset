# NPC 规划 001

## 用户原始需求锚点

- 目标：做一个让 NPC 自动移动的系统，并与当前导航能力全面适配。
- 核心策略：
  - 随机间隔移动。
  - 每次运动完成后会有一个休息区间。
  - 短暂停留时间区间为 0.5 秒到 3 秒。
  - 累计短暂停留 3 次到 5 次后，进入一次长停留。
  - 长停留时弹出气泡对话框，内容可以是自言自语，也可以与身边 NPC 进行对话聊天。
- 规划用途：后续所有 NPC 脑洞、扩展和实现，都以这份文档为主线继续追加。

## 当前实现对齐结果

- 已完成的行为闭环：
  - NPC 会在可行走区域内随机取点移动。
  - 移动结束后会进入短停。
  - 短停累计到阈值后进入长停。
  - 长停时优先寻找附近可聊天 NPC。
  - 找到合适对象时，双方都会进入附近聊天流程。
  - 找不到合适对象时，会展示自言自语气泡。
- 已完成的支撑内容：
  - `NPCRoamProfile` 用于统一漫游参数。
  - `NPCBubblePresenter` 负责气泡文本与显示时长。
  - `NPCAutoRoamControllerEditor` 提供调试入口。
  - 默认资源已落在 `Assets/111_Data/NPC/NPC_DefaultRoamProfile.asset`。

## 当前真实验证

- `Primary` 场景里已经验证 `001` / `002` / `003` 可以自动漫游。
- 本轮 live 证据中再次出现了附近聊天正样本：
  - `001.LastAmbientDecision = "chatting with 002"`
  - `002.LastAmbientDecision = "joined chat with 001"`
- 当前 Console 中未再复现 `Animator is not playing an AnimatorController`。

## 下一轮可继续发散的方向

- 细化“长停留对话”的语义来源：
  - 地点相关

  - 身边对象相关
- 细化 NPC 个体差异：
  - 不同 NPC 使用不同 RoamProfile
  - 不同 NPC 使用不同自言自语词库
  - 不同 NPC 拥有不同聊天倾向
- 细化导航策略：
  - 家/岗位锚点
  - 时间段行为切换
  - 特定触发时的优先移动目标
