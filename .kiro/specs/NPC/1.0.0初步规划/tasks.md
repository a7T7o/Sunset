# 实现计划 - NPC 自动移动初步规划

## 任务列表

- [ ] 1. 固化 V1 范围与验收口径
  - 明确第一阶段只覆盖“单 NPC 随机移动 + 短停 / 长停 + 自言自语气泡”
  - 明确“附近 NPC 配对聊天”属于第二阶段，不混入第一阶段实现
  - 明确 V1 不包含完整日程系统、关系系统、多人围聊和跨楼层复杂导航

- [ ] 2. 明确 NPC 自动移动的状态机
  - 定义 `短停留`、`随机移动中`、`长停留`、`气泡表达中` 的状态边界
  - 明确状态切换条件、计时器来源和打断规则
  - 明确短停 `0.5s ~ 3s`、短停累计 `3 ~ 5` 次、长停触发的节奏规则

- [ ] 3. 设计随机活动范围与目标点选择策略
  - 定义 NPC 的 home 点 / 活动锚点概念
  - 定义活动半径、随机点采样和不可达点重试规则
  - 明确随机点必须走现有导航可达性校验，不能绕开障碍

- [ ] 4. 设计 NPC 导航适配层
  - 评估现有 `NavGrid2D`、`PlayerAutoNavigator`、`INavigationUnit` 的复用方式
  - 明确 NPC 侧是复刻玩家导航代理、抽公共导航代理，还是包一层轻量 NPC agent
  - 明确移动输入、停止距离、卡住处理和取消逻辑如何对接现有 NPC 运行时

- [ ] 5. 设计自动移动与动画联动方式
  - 明确自动移动期间如何驱动 `NPCMotionController`
  - 明确运动中始终播放 `Move`、停驻和聊天中播放 `Idle`
  - 明确左右方向继续采用 `Side + flipX`，不引入额外动作状态

- [ ] 6. 设计长停留期间的气泡表现层
  - 明确气泡显示是新建 `NPCBubblePresenter`，还是复用现有对话运行时的一部分
  - 核实现有 `DialogueNode.isBubble` 能否作为语料或显示接点
  - 明确气泡的显示时长、打断方式、重复冷却和文本来源

- [ ] 7. 设计第二阶段的轻量聊天规则
  - 明确附近 NPC 搜索半径与筛选条件
  - 明确只在长停留阶段尝试聊天，移动中禁止聊天
  - 明确找不到可聊天对象时回退为自言自语

- [ ] 8. 定义配置化数据入口
  - 明确哪些参数先放在组件 Inspector 中
  - 明确哪些参数后续适合抽为 `NPCRoamProfile`
  - 整理第一阶段必须暴露的参数：活动半径、短停区间、长停触发阈值、气泡频率等

- [ ] 9. 输出第一版手工验收清单
  - 验证 NPC 不会穿墙、不离开活动区域
  - 验证短停 / 长停节奏符合规划
  - 验证长停时能稳定显示气泡
  - 验证第二阶段接入聊天前，第一阶段自言自语链条已经完整可测
