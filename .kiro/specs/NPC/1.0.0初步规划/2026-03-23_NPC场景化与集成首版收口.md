# NPC 场景化与集成首版收口

## 1. 原有配置

- `D:\Unity\Unity_learning\Sunset\Assets\000_Scenes\Primary.unity` 当前已经实例化了 `001 / 002 / 003` 三个 NPC prefab。
- 当前 Scene 里对这三个 NPC 的可见覆盖主要还是位置、缩放和名称；没有读到 `homeAnchor` 的场景级覆盖证据。
- 三个 prefab 原先都默认指向同一份 `NPC_DefaultRoamProfile.asset`。
- `003.prefab` 已经挂有 `NPCBubbleStressTalker`，但“测试 NPC / 正式 NPC”的分层方式此前更多停留在口头约定和单个 prefab 上。

## 2. 问题原因

- NPC 第二阶段已经具备“会动、会说话、能验气泡”的能力，但缺少一套明确的场景化落地入口。
- 之前更像是“先把 003 做成能测的样本”，还没有把“正式模式 / 验证模式 / 场景批量接入方式”整理成可复用流程。
- 这也是为什么我前面一直反复说“NPC 自己还有场景化与集成工作没做完”：能力已经有了，但真正可持续配置和批量落地的入口还不够清晰。

## 3. 建议修改

- 新增 `D:\Unity\Unity_learning\Sunset\Assets\111_Data\NPC\NPC_BubbleReviewProfile.asset`
  - 专门服务 `003` 这类气泡验证样本。
  - 当前参数倾向于更小活动范围、更短路径、更少停顿轮数，并关闭环境聊天，避免验证目标被漫游语义稀释。
- 新增 `D:\Unity\Unity_learning\Sunset\Assets\Editor\NPCSceneIntegrationTool.cs`
  - 统一处理 Scene 里选中的 NPC：
  - 批量补 Home Anchor
  - 批量吸附 Anchor 到 NPC 当前点位
  - 批量切换 Production / BubbleReview Profile
  - 批量移除或补挂 `NPCBubbleStressTalker`
- 将 `D:\Unity\Unity_learning\Sunset\Assets\222_Prefabs\NPC\003.prefab` 的默认 `roamProfile` 切到 `NPC_BubbleReviewProfile.asset`
  - 让 `003` 从“临时压测 NPC”正式转成“长期保留的验证样本 prefab”。

## 4. 修改后效果

- `001 / 002` 继续作为正式 NPC 基线，默认走 `NPC_DefaultRoamProfile.asset`。
- `003` 明确作为气泡与验证样本，默认走 BubbleReview Profile，并保留 `NPCBubbleStressTalker`。
- 以后需要把 Scene 中的 NPC 批量整理成正式模式或验证模式时，不需要再靠手工逐个点 Inspector。
- Home Anchor、Profile 和测试组件的配置入口，现在已经从“零散功能”变成“成体系的场景集成工具”。

## 5. 对原有功能的影响

- 不改导航核心，不改 `NavGrid2D`，不改玩家导航链路。
- 不改变 `NPCAutoRoamController` 的 `Moving / ShortPause / LongPause` 语义。
- 不改变当前 `001 / 002 / 003` 的碰撞基线。
- 真正的 Scene 级 Home Anchor 落点和活动区域摆放，下一步仍需要在 `Primary.unity` 中按具体场景构图做人工布置；这一层属于已有高风险 Scene 写入，不在这次首版入口收口里硬写死。
