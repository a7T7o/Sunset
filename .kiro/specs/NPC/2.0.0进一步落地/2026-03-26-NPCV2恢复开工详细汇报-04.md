# 2026-03-26 NPCV2 恢复开工详细汇报-04

## scene audit 五段式结论

- 原有配置：
  - `Primary.unity` 中 `001 / 002 / 003` 都挂在根层级 `NPCs` 下，三者原本只有 `navGrid -> NavigationRoot` 的 scene override，`NPCAutoRoamController.homeAnchor` 仍为 `null`。
  - scene 内原本不存在 `001_HomeAnchor / 002_HomeAnchor / 003_HomeAnchor` 节点。
  - 当前写窗口前置复核结果是：`D:\Unity\Unity_learning\Sunset @ main @ 18cf7427d97e749b0557f6d835124e44787c3e17`，`Primary.unity` 物理锁可获取，shared root 仍是 `main + neutral`。
- 问题原因：
  - 三个 NPC prefab 默认 `homeAnchor = null`，而 scene 侧此前也没有补最小落点，导致这轮无法把 `HomeAnchor` 作为明确的 scene 起点交付。
  - `unityMCP` 当前无法握手，`scripts/check-unity-mcp-baseline.ps1` 返回 `baseline_status: fail`、`issues: listener_missing`，所以这轮不能把 Unity live 回读或 Play 验证伪装成已完成。
- 建议修改：
  - 只在 `Primary.unity` 内新增三个最小空物体：`NPCs/001_HomeAnchor`、`NPCs/002_HomeAnchor`、`NPCs/003_HomeAnchor`。
  - 三个 anchor 都挂在 `NPCs` 下，基础位置先与当前 scene 中 `001 / 002 / 003` 的既有站位对齐。
  - 只给 `NPCAutoRoamController.homeAnchor` 回填 scene 引用，不改 `navGrid`、`roamProfile`、路线、停留点、节奏或导航核心。
- 修改后效果：
  - `001 / 002 / 003` 现在都已有明确 `HomeAnchor` scene 节点与引用。
  - 三个 anchor 的 hierarchy 路径、局部坐标、世界坐标都已可直接回读。
  - `git diff --check -- 'Assets/000_Scenes/Primary.unity'` 已通过，说明本轮 scene YAML 没留下空白符级脏点。
- 对原有功能的影响：
  - 本轮没有改 `GameInputManager.cs`，没有改 `navGrid` 指向，没有改 roam profile，也没有扩到气泡、关系成长、路线或停留点。
  - 当前能确认的是“离线序列化层已补齐最小 `HomeAnchor`”；Unity live 稳定性仍待用户打开 Editor 后做终验。
