# NPC P1-01 三名 NPC 场景点位方案卡

- 对应任务：
  - `T-P1-01`
- 日期：
  - `2026-03-27`
- 状态：
  - 已完成设计卡

## 1. 目的

这份文档只做一件事：

- 把 `001 / 002 / 003` 从“只有 HomeAnchor”推进到“下一轮可以往 scene 里落的最小点位方案”。

这里先给出：

- 常驻区
- 1 到 2 个停留点
- 1 个公共相遇点
- 1 个边缘自言自语点

并且每个点都写明角色理由、建议命名和空间语义。

## 2. 统一命名规则

后续进 scene 时，点位对象建议统一命名为：

- `NPC_{编号}_Home_Base`
- `NPC_{编号}_Stay_{语义}`
- `NPC_{编号}_Stay_{语义}_B`
- `NPC_SHARED_Meet_{语义}`
- `NPC_{编号}_Solo_{语义}`

例如：

- `NPC_001_Stay_SquareWatch`
- `NPC_002_Stay_YardSun`
- `NPC_SHARED_Meet_MainCrossing`
- `NPC_003_Solo_ObservationEdge`

## 3. 001｜村长

### 3.1 常驻区

- 点位名：
  - `NPC_001_Home_Base`
- 空间语义：
  - 村庄主通路和公共活动面之间的“看得见人流、也看得见生产区”的位置
- 角色理由：
  - `001` 的核心不是躲在角落，而是站在能观察村子运行状态的位置
  - 他需要一个“站住就像在看整个村务节奏”的起点

### 3.2 停留点 A

- 点位名：
  - `NPC_001_Stay_SquareWatch`
- 空间语义：
  - 靠近广场或主要通行面，但不正挡路
- 角色理由：
  - 这是他最像“巡视”和“看大家忙得怎么样”的点

### 3.3 停留点 B

- 点位名：
  - `NPC_001_Stay_WorkRouteEdge`
- 空间语义：
  - 靠近生产路线边缘、运输路径旁或工作区入口
- 角色理由：
  - `001` 的第二停留点不该只是复制“广场”，而应该像是在看事情有没有稳住

### 3.4 公共相遇点

- 点位名：
  - `NPC_SHARED_Meet_MainCrossing`
- 空间语义：
  - 主路径交汇、大家自然会打照面的路口
- 角色理由：
  - `001` 和其他两人相遇时，最自然的不是贴着家门口聊，而是在公共动线上短停交流

### 3.5 边缘自言自语点

- 点位名：
  - `NPC_001_Solo_PathShoulder`
- 空间语义：
  - 主路边缘、稍微退半步的位置
- 角色理由：
  - 让他像是在“站住看一会儿，再心里过一遍今天的安排”

## 4. 002｜村长女儿

### 4.1 常驻区

- 点位名：
  - `NPC_002_Home_Base`
- 空间语义：
  - 生活气更强、光线舒服、带一点院落或住家边缘感的位置
- 角色理由：
  - `002` 的基线不是巡逻，而是“更贴近日常生活流动”

### 4.2 停留点 A

- 点位名：
  - `NPC_002_Stay_YardSun`
- 空间语义：
  - 有日照、适合站一会儿发呆或看看周围的院边位置
- 角色理由：
  - 这是最适合承接她生活化轻气泡的点

### 4.3 停留点 B

- 点位名：
  - `NPC_002_Stay_HouseSidePause`
- 空间语义：
  - 住区外缘或小路径边、可短暂停一下的位置
- 角色理由：
  - 让她有一个比常驻区稍微更游离、但仍然带“生活感”的第二点

### 4.4 公共相遇点

- 点位名：
  - `NPC_SHARED_Meet_MainCrossing`
- 空间语义：
  - 与 `001 / 003` 共用的自然打照面点
- 角色理由：
  - `002` 的相遇不该只发生在自己院里，她需要能自然进入公共生活面

### 4.5 边缘自言自语点

- 点位名：
  - `NPC_002_Solo_GardenEdge`
- 空间语义：
  - 边缘一点、安静一点、像是会自己轻轻说几句的角落
- 角色理由：
  - 这里更适合她承接“心情 / 天气 / 周围人忙碌状态”的自言自语

## 5. 003｜研究型二儿子

### 5.1 常驻区

- 点位名：
  - `NPC_003_Home_Base`
- 空间语义：
  - 靠近研究台、材料位、观察位或视角变化比较明显的地方
- 角色理由：
  - `003` 的 base 点应该像“我站在这儿是为了看现象”，不是单纯发呆

### 5.2 停留点 A

- 点位名：
  - `NPC_003_Stay_StudyBench`
- 空间语义：
  - 更贴近研究台或材料观察面的点
- 角色理由：
  - 这是最像他会“停下来再确认一遍”的位置

### 5.3 停留点 B

- 点位名：
  - `NPC_003_Stay_RouteObserve`
- 空间语义：
  - 能看到路径变化、人流和距离关系的位置
- 角色理由：
  - 他适合拥有一个“看别人怎么走”的第二观察点

### 5.4 公共相遇点

- 点位名：
  - `NPC_SHARED_Meet_MainCrossing`
- 空间语义：
  - 让 `003` 也能被公共动线拉出来，不一直缩在研究位
- 角色理由：
  - 这样他和 `001 / 002` 的相遇才会更自然，不是纯脚本拉近

### 5.5 边缘自言自语点

- 点位名：
  - `NPC_003_Solo_ObservationEdge`
- 空间语义：
  - 更偏边角、能观察到地形变化或路口节奏的位置
- 角色理由：
  - 这是最适合他说“我得再看一遍”“这个现象值得记一下”的点

## 6. 三人差异总结

- `001`
  - 核心是“看村子怎么运转”
  - 点位应更偏主路径、广场、工作线边缘

- `002`
  - 核心是“生活感、情绪感、光照和停顿”
  - 点位应更偏院边、住区外缘、柔和停留位

- `003`
  - 核心是“观察、记录、研究”
  - 点位应更偏研究台、观察位、路径变化点

## 7. 下一轮落 scene 时的最小切片

如果后续进入 `T-P1-02 ~ T-P1-05`，建议第一轮只落：

- `NPC_001_Stay_SquareWatch`
- `NPC_002_Stay_YardSun`
- `NPC_003_Stay_StudyBench`
- `NPC_SHARED_Meet_MainCrossing`
- 三个 `Solo_*` 边缘点

这样能用最小对象数先把三人的空间倾向区分出来，不需要一上来把路线和时间调度全做满。
