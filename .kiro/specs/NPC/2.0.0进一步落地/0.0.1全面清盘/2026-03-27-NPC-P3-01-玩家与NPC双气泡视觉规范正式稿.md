## 1. 目的

这份规范只做一件事：

- 把“玩家气泡”和“NPC 气泡”收成一套可直接落代码的正式视觉语言

后续 `T-P3-02 ~ T-P3-05` 都必须以这份文档为准，不再靠口头记忆或临时感觉调整。

## 2. 当前已锁定的 NPC 气泡基线

现有 NPC 气泡不是抽象概念，而是已经有明确实现基线：

- 世界空间头顶气泡
- `10` 字一行的确定性分行
- 气泡整体高于角色头顶，不压脸
- 倒三角尾巴朝下指向角色
- 尾巴单独轻微跳动
- 整体只允许很轻的浮动
- 明显的内边距与文字留白

当前已锁定的核心布局参数来自：

- [NPCBubblePresenter.cs](/D:/Unity/Unity_learning/Sunset/Assets/YYY_Scripts/Controller/NPC/NPCBubblePresenter.cs)

当前 NPC 基线参数口径：

- `bubblePadding`
  - `82 x 42`
- `textSafePadding`
  - `24 x 22`
- `fontSize`
  - `32`
- `tailSize`
  - `34 x 24`
- `tailYOffset`
  - `-28`
- `preferredCharactersPerLine`
  - `10`
- `show / hide`
  - `0.14 / 0.10`
- `tail bob`
  - `26 @ 0.85`

这意味着后续玩家气泡不能另起炉灶，必须先继承这套“尺寸、边距、节奏、尾巴指向”的骨架。

## 3. 双气泡总原则

### 3.1 必须保持同一产品体系感

- 不能做成两套完全不同的 UI
- 不能把玩家气泡做成旁白框、聊天窗、漫画框之类的异体系产物
- 玩家 / NPC 必须一眼看出是同一个产品里的两种角色语言

### 3.2 必须一眼能区分玩家与 NPC

- 不能再靠“文本内容不同”来区分
- 必须在第一眼观感上就能看出：
  - 这是 NPC 在说
  - 还是玩家侧在冒想法 / 轻反馈

### 3.3 区分手段只允许落在这 4 层

1. 配色
2. 字体气质
3. 细微动效权重
4. 语义层级

不建议靠：

- 完全不同的形状
- 完全不同的排版规则
- 完全不同的尾巴逻辑

## 4. 双气泡共用骨架

后续玩家 / NPC 必须共用以下结构：

- 世界空间 Canvas
- 头顶锚定逻辑
- `10` 字一行分行口径
- 同级的字体尺寸档位
- 同级的气泡内边距
- 同级的边框厚度
- 同级的 show / hide 节奏
- 同类的尾巴单独轻微跳动
- 同类的阴影和层级处理

换句话说：

- 玩家气泡是“同骨架换语言”
- 不是“另一套新 UI”

## 5. NPC 气泡正式语言

NPC 气泡继续维持当前方向，不再回退：

- 角色身份：
  - 他人在说话 / 冒环境表达
- 主体观感：
  - 稳
  - 夜色感
  - 带一点暖边
- 颜色语言：
  - 深灰蓝主体
  - 暖金边框
  - 深色阴影
  - 偏暖浅色文本
- 情绪印象：
  - 稳住空间
  - 让人知道“这是场景中的别人”

NPC 气泡后续仍承担：

- 单人环境表达
- 玩家靠近轻反馈
- NPC / NPC 相遇对话
- 状态反馈

## 6. 玩家气泡正式语言

### 6.1 玩家气泡的定位

玩家气泡不是正式 `DialogueUI`。

它的定位是：

- 玩家此刻的短想法
- 靠近 NPC 时的轻反馈
- 面对环境 / 互动点时的即时反应

所以玩家气泡必须看起来：

- 更像“自己的念头”
- 但仍属于 Sunset 当前头顶气泡体系

### 6.2 玩家气泡的推荐视觉方向

玩家气泡采用：

- 晨雾青绿色系
- 更清爽一点的边框
- 更干净一点的文字墨色

正式推荐色板：

- `Player Fill`
  - `#DDE9E1`
  - 近似 `RGBA(0.87, 0.91, 0.88, 0.96)`
- `Player Border`
  - `#58756A`
  - 近似 `RGBA(0.35, 0.46, 0.42, 1.00)`
- `Player Shadow`
  - `#10211C`
  - 近似 `RGBA(0.06, 0.13, 0.11, 0.28)`
- `Player Text`
  - `#1E2A24`
  - 近似 `RGBA(0.12, 0.16, 0.14, 1.00)`
- `Player Text Outline`
  - `#F3EEE3`
  - 近似 `RGBA(0.95, 0.93, 0.89, 0.92)`

### 6.3 玩家气泡的第一眼印象

玩家气泡应给人的第一反应是：

- 这是“我自己”的反馈
- 更轻
- 更近
- 更清醒

但不能给人的感觉是：

- 完全跳戏
- 像系统提示
- 像另一个游戏里的聊天框

## 7. 字体规范

### 7.1 共通原则

- 保持当前中文可读性第一
- 保持当前像素 / 半像素的世界观一致性
- 不能为了区分玩家气泡，直接去改 `DialogueChinese*` 共享字体资产

### 7.2 NPC 字体

- 继续沿用当前 NPC 气泡可用字体链
- 保持偏暖、偏像素、偏场景内角色表达的感觉

### 7.3 玩家字体

首版实现口径：

- 允许玩家气泡有单独的 `TMP_FontAsset` 槽位
- 但如果当前没有安全的新字体入口，就先沿用现有字体资源
- 通过颜色、字重观感和气泡语言先拉开差异

硬规则：

- 不准在 `T-P3-02` 里顺手改 `DialogueChinese*`
- 不准把“换共享字体资产”偷换成“实现玩家气泡”

## 8. 布局与动效规范

### 8.1 共用布局

玩家 / NPC 默认共享：

- `bubblePadding`
- `textSafePadding`
- `fontSize`
- `tailSize`
- `preferredCharactersPerLine`
- `show / hide` 时长

这样做的原因是：

- 保证同体系感
- 保证后续维护时不会一边越调越大、一边越调越小

### 8.2 允许微差异的项

玩家气泡允许只做非常克制的差异：

- `visibleFloatAmplitude`
  - 可比 NPC 再轻一点
- `tailBobAmplitude`
  - 可比 NPC 小 `10% ~ 20%`
- `bubbleLocalOffset`
  - 可按玩家头顶素材高度做微调

但不允许：

- 换成不同排版逻辑
- 改成另一套尾巴结构
- 另开一套完全不同尺寸档

## 9. 一眼区分规则

如果玩家与 NPC 同时出现，用户必须能在半秒内判断：

- 深底暖边的是 NPC
- 浅雾青绿的是玩家

如果需要读完文字才知道谁是谁，这份规范就算失败。

## 10. 实现护栏

`T-P3-02` 开工时必须遵守：

1. 不碰 `Primary.unity`
2. 不碰 `DialogueChinese*`
3. 不把 `PlayerThoughtBubblePresenter` 改成与 NPC 完全脱钩的另一套系统
4. 不重写现有 `10` 字一行和内边距骨架
5. 优先做“样式层差异”，不要先扩到关系成长、正式对话轨或导航联调

## 11. 正式验收口径

`T-P3-01` 完成的标准不是“写了一篇设计想法”，而是后续实现时可以直接据此判断通过或失败：

- 玩家 / NPC 一眼可区分
- 又仍像同一产品
- 玩家气泡不抢正式对话框职责
- NPC 气泡继续保留当前已经被锁定的尾巴方向、留白、尺寸与节奏
- 后续实现不需要再回头争论“到底要不要另起一套样式”

## 12. 一句话结论

双气泡的正确方向不是“玩家另做一套新 UI”，而是在当前 NPC 气泡已经锁定的骨架上，给玩家建立一套更轻、更近、更清醒的青绿色视觉语言，并且严格守住共享尺寸、边距、尾巴逻辑和动效节奏。
