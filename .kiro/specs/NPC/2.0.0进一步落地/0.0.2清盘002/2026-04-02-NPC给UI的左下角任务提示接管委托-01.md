# 2026-04-02 NPC 给 UI 的左下角任务提示接管委托

## 当前已接受基线

- `NPC` 这条线当前主刀只守：
  - NPC 会话状态机
  - 玩家 / NPC 双气泡
  - 自动续聊
  - 跑开中断
- `shared prompt shell / 玩家面 UI 壳` 不再由 `NPC` 主刀。
- 用户最新明确裁定：
  - 如果当前对象同时存在 `Spring` 任务语义和 `NPC` 闲聊语义，
  - 左下角的 `E` 提示不要再显示成“闲聊”，
  - 而应优先显示“进入任务 / 任务相关对话”的内容。

## 本轮唯一主刀

- 只做：
  - `左下角 InteractionHintOverlay` 的提示内容仲裁
  - 让它在 `Spring` 任务优先时显示任务语义，而不是 `NPC` 闲聊语义

## 本轮允许 scope

优先只在这组 shared/UI 文件内处理：

  - 气泡偏移

  - NPC 漫游 / 出生点 / HomeAnchor / scene 路线

- 不要顺手把头顶 world hint 全案一起吞掉；这轮只守左下角内容仲裁。

## 用户最新反馈，必须带着做

用户当前的真实不满点不是“有没有提示”，而是：

1. 当前 NPC 头上和聊天气泡已经够挤了。
2. 左下角如果此时真正更该触发的是 `Spring` 任务链，
   那么左下角再写“闲聊”就是误导。
3. 玩家看到 `E` 提示时，应该先看到当前最该做的事。

所以这轮不是视觉微调，而是“提示语义不能说错话”。

## 完成定义

满足以下 4 条，才算这刀完成：

1. 当同一个 NPC 当前存在 `Spring` 任务语义时：
   - 左下角 `InteractionHintOverlay`
   - `caption/detail`
   必须优先显示任务相关内容
   不能继续显示“闲聊 / 按 E 开口”。
2. 当当前对象没有 `Spring` 任务优先语义时：
   - 左下角仍可正常显示 `NPC` 闲聊提示。
3. 这次修改不破坏：
   - 工作台
   - 床
   - 其他 `SpringDay1` 交互对象
   的左下角提示链。
4. 不得通过回改 `NPC` 会话逻辑来“假装解决 UI 仲裁”。

## 你应该怎么理解这刀

这刀本质上是：

- `shared/UI` 决定“左下角该说哪一句话”

而不是：

- `NPC` 决定全局所有目标的 `E` 提示语义

## 建议实现方向

你可以自由实现，但结果上至少要达到：

- proximity service 在同帧候选里，能识别“当前任务优先对象”
- overlay 展示文本不再只拿最近对象的默认 caption/detail
- 当 `NPCDialogueInteractable` / `Spring` 正式任务链当前应优先时，`NPCInformalChatInteractable` 的“闲聊”文案必须被压下去
