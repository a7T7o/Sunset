## 1. 这轮我实际又压实了什么

这轮不是继续泛讲“以后要像村民”，而是把 `101~301` 真正压成了可读、可查、可验证的 resident 语义层：

1. `SpringDay1NpcCrowdManifest` 现在不再只有：
   - `sceneDuties`
   - `semanticAnchorIds`
   - `growthIntent`

   - `residentBeatSemantics`

   - `beatKey`
   - `presenceLevel`
   - `flags`
   - `note`

   - `GetResidentPresenceLevel(...)`
   - `GetResidentBeatFlags(...)`
   - `HasResidentBeatFlags(...)`

## 6. 哪些事还必须等你 deployment 层先动

我这轮没有吞：

1. `CrowdDirector`
2. `Town runtime contract`
3. resident 实际部署与常驻摆放

所以接下来你如果要把“本来就住在村里”真正变成玩家肉眼看到的现实，还是得你先改：

1. 谁什么时候已经在场
2. 谁是早就在村里，只是此刻第一次被玩家看见
3. 谁只是 deployment 退远，而不是被 runtime 现生出来

## 7. 正式剧情现在已经被我压成“一次性消耗”

这轮我顺手把一个新的硬约束也落进了 NPC own：

1. `NPCDialogueInteractable` 里的 formal 段现在一旦被消费过
2. 后面就不会再靠重复交互把同一段 formal 或 followup 重播出来
3. formal 消费后，会自然把交互入口让回：
   - informal chat
   - resident 日常句池
   - phase 后的非正式补句

也就是说，后面你在 deployment / director 那边接的时候，可以按下面这套理解：

1. `one-shot`
   - 正式剧情链
   - 播完即完
2. `post-consume`
   - resident 日常闲聊
   - phase 后背景补句
   - 非正式聊天

不要再把 formal 和后续闲聊当成一锅可重复内容。
