## 二、你现在继续 own 的，不是部署代码，而是 4 类 resident 语义

### 1. 哪些人本来就该在村里

请你按 `101~301` 这 8 人，继续明确：

1. 谁是“白天默认就该在村中活动的人”
2. 谁更像“屋内 / 摆摊 / 边缘劳作中，只是玩家未近看”
3. 谁更像“夜里才更容易被注意到”
4. 谁不该被写成“平时像不存在，只有轮到剧情才忽然出厂”

### 2. 各 phase 的“在场方式”矩阵

请你继续把后半段内容按“常驻居民化”去理解，而不只是按 cue 说台词：

1. `EnterVillage_PostEntry`
2. `DinnerConflict_Table`
3. `ReturnAndReminder_WalkBack`
4. `FreeTime_NightWitness`
5. `DayEnd_Settle`
6. `DailyStand_Preview`

每段不只要讲“谁说什么”，还要讲：

1. 谁本来就在
2. 谁只是被玩家第一次真正看见
3. 谁在看，但不出声
4. 谁让位
5. 谁装忙
6. 谁是背景压力，不该被写成突然生成的功能人

### 3. 常驻居民语境下的气味和强度

请你继续守住你已经做好的群像空气，但再明确一层：

1. 这些人对玩家不是“临时围观团”
2. 而是“本来就在这个村里活着的人”
3. 所以他们的对白、目光、停顿、回避、冷处理，都应当像“长期生活关系里的反应”
4. 不要再让后半段内容像任务临时刷出来的功能块

### 4. 哪些人后续不该常驻化过头

如果你判断某几个角色：

1. 更适合低频出现
2. 或只在特定夜段/背景段更成立
3. 或不该白天就高频站脸

也请你明确说出来，不要把“常驻化”误写成“所有人全程站在玩家眼前”。
