# NPC 0.0.2清盘002 现状排查与验收回退矩阵

- 版本：v1.0
- 日期：2026-03-27
- 状态：当前有效
- 作用：
  - 把 `0.0.1全面清盘` 到当前用户复测反馈之间的真实状态说清楚
  - 把 `spring-day1` 与 `NPC` 的交叉边界重新划干净
  - 把哪些内容已经被用户实测、哪些没有、哪些要回退重做写实
  - 给后续 `0.0.2清盘002` 的施工任务表提供唯一问题底稿

## 1. 这份文档为什么存在

`0.0.1` 的推进并不是完全无效。

但用户刚刚给出的现场反馈说明：

- 当前“能运行”不等于“能验收”
- 当前“有一条靠近时的短反馈”不等于“非正式聊天系统已经成立”
- 当前“001 能按 E 进 spring-day1 的正式剧情”也不等于“NPC 自己的聊天体系已经成立”

所以 `0.0.2` 不是把 `0.0.1` 推翻重来，而是：

- 保留能复用的底座
- 回退错误的完成宣称
- 重新按用户的真实体验目标拆任务

## 2. 用户刚刚明确的新口径

用户这轮给出的口径，核心不是“小修小补”，而是把 NPC 交互体验重新钉回正确目标。

### 2.1 spring-day1 与 NPC 的边界现在重新划定

- `001` 当前已有一条 `spring-day1` 负责的正式剧情对话链
- 这条链包括：
  - 靠近后出现悬浮提示
  - 按 `E` 进入正式剧情对话模式
- 但从现在起：
  - “和 NPC 的谈话聊天”整体应由 `NPC` 线负责和处理
  - `spring-day1` 已经做出来的 `001` 正式剧情链不能被随意破坏
  - 工作台的悬浮提示虽然也丑，但这条不是 NPC 线负责

### 2.2 用户要的“非正式聊天”不是现在这种单向短反馈

用户明确指出，当前实现的“玩家走近后 NPC 单方面冒一句短反馈”不构成闭环。

用户要的非正式聊天是：

1. 玩家走到 NPC 附近
2. 玩家按 `E`
3. 先出现玩家自己的气泡
4. 玩家气泡里出现玩家对 NPC 说的话
5. NPC 延迟 `0.5 ~ 1.0s`
6. NPC 转向玩家
7. NPC 冒出自己的回复气泡
8. 玩家再次按 `E`
9. 刷新玩家气泡内容
10. 玩家气泡文本变化有动效
11. NPC 再延迟后回复
12. 整个过程里：
   - 按 `E` 是跳过当前文本动效
   - 不是跳过整句对话

### 2.3 聊天中断和跑路反应是系统需求，不是可有可无

用户明确要求：

- 玩家聊到一半跑开
- NPC 要按好感度和角色性给出不同反应
- 这些反应可以是：
  - 气泡
  - 表情
  - 漂浮提示
  - 好感度上升 / 下降
  - 气愤
  - 迷惑
  - 无语

同时玩家自己也要有中断聊天时的玩家气泡表达，例如：

- 我先走一步
- 我还有事
- 不想聊了
- 哦，你看天上有飞碟
- 你鞋带松了

这说明后续不是只要“NPC 有回复”就算完成，而是要做成：

- 有起手
- 有来回
- 有中断
- 有情绪
- 有关系影响

### 2.4 好感度不能只是静态存着

当前关系阶段系统只是底座，还没有成为真正可体验的聊天分歧系统。

用户已经把目标说得很明确：

- 后续肯定要有不同的对话包库
- 要有好感度与聊天、跑路、中断、冒犯之间的联动
- 这次必须把相关逻辑和设计处理好

## 3. 当前代码 / 资源现场排查结论

以下结论全部来自当前仓库实际代码与资源，不是主观猜测。

## 3.1 `001` 的正式剧情对话链当前挂在 `spring-day1` 资产上

当前实锤事实：

- `Assets/222_Prefabs/NPC/001.prefab`
  上存在 `NPCDialogueInteractable`
- 该组件当前绑定：
  - `dialogueSequence = SpringDay1_FirstDialogue.asset`
- 对应正式剧情资源：
  - `Assets/111_Data/Story/Dialogue/SpringDay1_FirstDialogue.asset`
  - `Assets/111_Data/Story/Dialogue/SpringDay1_FirstDialogue_Followup.asset`

也就是说：

- `001` 现在能按 `E` 进入正式剧情，不是 NPC 自己的日常聊天系统已经做完
- 而是因为 `spring-day1` 已经把 `001` 的正式剧情入口接好了

## 3.2 `002 / 003` 当前没有 `E` 键聊天入口

当前实锤事实：

- `Assets/222_Prefabs/NPC/002.prefab`
  没有 `NPCDialogueInteractable`
- `Assets/222_Prefabs/NPC/003.prefab`
  也没有 `NPCDialogueInteractable`
- 它们目前只有 `NPCBubblePresenter` 等基础组件

所以用户现在无法对 `002 / 003` 按 `E` 聊天，不是“用户没触发对”，而是当前资源层就根本没接这条入口。

## 3.3 当前悬浮提示框确实是 `spring-day1` 那套 UI

当前 `001` 与工作台都在复用：

- `Assets/YYY_Scripts/Story/UI/SpringDay1WorldHintBubble.cs`

这意味着：

- 当前 NPC 的 `E` 提示 UI 样式确实是 `spring-day1` 的共用提示框
- 用户说它丑，是对当前现场的准确反馈
- 但工作台提示虽然同样丑，不属于 NPC 本轮施工目标

因此后续正确做法不是直接乱改整套 `spring-day1` 工作台提示逻辑，而是：

- 重新划出 NPC 自己的提示风格责任
- 必要时做“共享骨架、不同皮肤”或“NPC 单独皮肤层”

## 3.4 我前一轮做的不是“非正式聊天”，只是“靠近时 NPC 单向短反馈”

当前代码：

- `Assets/YYY_Scripts/Service/Player/PlayerNpcNearbyFeedbackService.cs`

当前真实行为：

- 玩家靠近 NPC
- 系统自动找最近 NPC
- 直接从 `roamProfile / dialogueContentProfile` 里拿一句 `playerNearbyLines`
- 让 NPC 自己冒一句短反馈

当前缺失：

- 没有玩家按 `E` 发起
- 没有玩家自己的回合
- 没有一来一回
- 没有文本推进状态
- 没有“跳过动效但不跳句”的控制
- 没有聊天中断判定
- 没有跑路反应
- 没有情绪反应挂点

所以它只能算：

- “日常近身短反馈底座”

不能再把它叫成：

- “非正式聊天已完成”

## 3.5 当前关系阶段底座还没真正挂进聊天闭环

当前已有内容：

- `Assets/YYY_Scripts/Data/NPCRelationshipStage.cs`
- `Assets/YYY_Scripts/Service/Player/PlayerNpcRelationshipService.cs`
- `NPCDialogueContentProfile` 中按阶段分流的 `playerNearbyLines`

当前真实作用范围：

- 只影响“靠近时 NPC 单方面说一句”的短反馈取句

当前还没有：

- 玩家发起聊天后的多轮分支
- 好感变化原因
- 跑路 / 打断 / 被冷落后的关系变化
- 好感升降后的表情 / 状态反馈
- 正式对话内容解锁联动

所以它现在是：

- 可复用底座

不是：

- 关系成长系统已可验收

## 5. `0.0.2` 现在真正要解决的问题

从当前现场看，`0.0.2` 不该再围着“短反馈再润一下”继续修。

现在真正的问题有 6 个：

### 问题 1：NPC 聊天 ownership 没真正统一

- `001` 正式剧情链在 `spring-day1`
- NPC 自己的日常聊天体系并未真正成立
- `002 / 003` 完全没有 `E` 聊天入口

### 问题 2：NPC 非正式聊天没有闭环

- 现在只有 NPC 单方面冒一句
- 没有玩家回合
- 没有多轮来回
- 没有跳过动效而不跳句

### 问题 3：提示 UI 很丑，而且 NPC 与工作台共用口径混在一起

- 当前 UI 是 `spring-day1` 的共用提示
- NPC 应该拿回自己可控的提示样式责任
- 但又不能顺手把工作台这条线一起吞掉

### 问题 4：关系成长还没挂到真正聊天系统

- 现在只影响单向短反馈
- 没有成为真实聊天分支

### 问题 5：中断 / 跑路 / 情绪 / 好感变化完全没接

- 这是用户明确点名要的
- 不能继续当“以后再说的小尾巴”

### 问题 6：对话包库和内容结构还不够支持真正聊天

- 当前内容资产更适合：
  - 单人环境气泡
  - 近身短反馈
  - NPC / NPC 相遇句
- 还不够支撑：
  - 玩家发起句
  - NPC 回复句
  - 多轮聊天
  - 跑路中断句
  - 情绪反馈句

## 6. `0.0.2` 的边界结论

从这一刻开始，`0.0.2` 的正确目标不是：

- 把现在的近身短反馈再修漂亮一点

而是：

1. 重新划清 `spring-day1` 与 NPC 的聊天边界
2. 建立真正的 NPC 非正式聊天闭环
3. 把 `002 / 003` 的 `E` 键聊天入口接起来
4. 让关系成长进入真实聊天系统
5. 为中断 / 跑路 / 情绪 / 好感变化建立第一版可玩的结构

## 7. 当前一句话结论

`0.0.1` 留下的可用资产主要是：

- 角色内容底座
- 玩家 / NPC 双气泡底座
- 关系阶段底座

但用户当前真正否掉的是：

- “把单向近身短反馈当成非正式聊天已完成”这个判断。

因此 `0.0.2` 必须从“真正的按 E 发起的 NPC 非正式聊天闭环”重新开刀。
