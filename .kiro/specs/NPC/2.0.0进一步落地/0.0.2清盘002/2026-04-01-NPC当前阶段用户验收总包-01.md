# 2026-04-01 NPC 当前阶段用户验收总包

## 1. 这份包是干什么的

这份文档只做一件事：

- 把 `NPC` 这条线现在到底做到哪一步、哪些能测、哪些还不能装成过线、哪些已经不是 `NPC` 当前 own 的东西，一次性说清楚。

这份包故意把结论拆成 3 层：

1. `结构 / checkpoint`
2. `targeted probe / 线程自测`
3. `真实入口体验 / 用户终验`

当前必须诚实承认：

- `NPC` 非正式聊天这条线，前两层已经站住不少。
- 但第三层，也就是你真正肉眼玩起来的最终体验，当前还没有被你正式放行。

## 2. 当前主线与当前阶段

### 当前主线

- `002 / 003` 的 NPC 非正式聊天体验闭环。
- 重点不是再证明“代码会跑”，而是确认：
  - 玩家发起是否自然
  - 自动续聊是否自然
  - 跑开中断是否收得干净
  - 双气泡是否还能看

### 当前阶段

- 当前阶段应判为：
  - `线程自测已过`
  - `等待人工体验终验`

### 当前 exact-own 边界

按最新 live 边界，`NPC` 当前只保留 3 个 exact-own 底座文件：

- `Assets/YYY_Scripts/Controller/NPC/NPCBubblePresenter.cs`
- `Assets/YYY_Scripts/Service/Player/PlayerNpcChatSessionService.cs`
- `Assets/YYY_Scripts/Service/Player/PlayerThoughtBubblePresenter.cs`

这意味着：

- 我这轮可以对 `NPC` 会话、玩家/NPC 双气泡、自动续聊、跑开收束负责。
- 我这轮不再把 shared prompt shell / 左下角统一提示壳 / Day1 玩家面 UI 整合当成自己主刀。

## 3. 我已经自己验过什么

以下是当前已经站住的自验证据：

### 结构 / checkpoint 已成立

- `002 / 003` 非正式聊天主链已成立：
  - 玩家先说
  - NPC 延迟回复
  - 第二轮会自动续聊
- `E` 的职责已收窄：
  - 发起聊天
  - 跳过当前打字机 / 等待 / 收尾
  - 不再负责“每一轮都按一次才推进对白”
- 跑开中断链已从旧 typed interrupt 链切开：
  - 先清旧会话气泡
  - 再显示玩家离场句 / NPC 反应
- 玩家 / NPC 气泡打字机开始时不再透明渐显。
- 会话期双方气泡已有临时错位底座，不再完全正中重叠。

### targeted probe / 线程自测 已通过

这 4 条运行态自测当前已全绿：

- `002 closure`
- `002 interrupt`
- `003 closure`
- `003 interrupt`

当前线程自测拿到的硬结论是：

- `002 / 003` 都能完成至少两轮来回。
- `002 / 003` 跑开中断都能收尾，不再是“完全没链路”。
- 自动续聊已经成立，不再要求玩家每轮重按 `E`。

## 4. 现在还不能装成“已过线”的东西

以下内容当前都不能直接说“过了”：

### 4.1 NPC own 仍待你终验

- 双气泡在真实镜头里是否还会重叠得难看。
- 跑开时玩家气泡是否还会鬼畜、卡首字、半透明残留。
- NPC 在跑开后是否还会残留等待态、残留气泡、收尾割裂。
- 自动续聊的观感是否自然，还是虽然逻辑对了但体感很怪。
- `E` 的体感是否已经变成“只跳过动效”，而不是“还在偷偷抢推进逻辑”。
- 玩家气泡和 NPC 气泡的样式差异是否已经足够清晰。

### 4.2 可观察，但不记入本轮 NPC own 过线

以下项目你可以顺手看，但当前不应再记成 `NPC` 本轮 own 验收项：

- 左下角统一提示 UI 的最终观感
- 头顶小箭头与左下角提示是否完全一致同步
- `001` 的正式剧情提示卡是否够好看
- 唯一提示 / 唯一 `E` / 最近目标切换的一致性

这些属于：

- `shared prompt shell`
- `Story / Day1 / UI` 玩家面整合

如果你看到这些有问题，请照样写进回执，但我会把它们标成：

- `shared/UI 观察项`

## 6. 你现在真正要测的主矩阵

下面这组是本轮最值得你亲自测的核心矩阵。

## 6.1 必测 1：`002` 开聊闭环

### 编号

- `ACC-01`

### 入口条件

- 进入 `Primary`。
- 靠近 `002`。
- 当前没有正式对白、背包或其他阻塞 UI 抢占。

### 操作步骤

1. 靠近 `002`。
2. 按一次 `E` 发起闲聊。
3. 看玩家是否先出自己的气泡。
4. 等 NPC 回复。
5. 不再按第二次 `E`，先观察它是否会自动进入下一轮。
6. 如果打字太慢，可按 `E` 跳过当前打字机。
7. 让整段对话正常聊完。

### 预期结果

- 玩家先说，不是 NPC 单方面先冒话。
- NPC 会延迟一下再回复。
- 第二轮会自动开始，不需要你再按一次 `E` 才推进。
- `E` 的作用更像“跳过当前打字/停顿”，不是“强行推进下一句”。
- 聊完后双方气泡都会正常收走。

### 失败现象

- 按 `E` 后根本开不起来。
- 玩家不先说，NPC 直接先说。
- 第一轮说完后停死，必须再按一次 `E` 才能继续第二轮。
- 聊完后气泡不收，或只收一边。

### 失败判读

- 这条失败优先记为：`NPC own`

## 6.2 必测 2：`003` 开聊闭环

### 编号

- `ACC-02`

### 入口条件

- 与 `ACC-01` 相同，只是对象换成 `003`。

### 操作步骤

1. 靠近 `003`。
2. 按一次 `E` 发起闲聊。
3. 观察第一轮。
4. 观察第二轮是否自动续上。
5. 让整段正常聊完。

### 预期结果

- 与 `002` 相同，但对象是 `003`。
- `003` 的回复口气应明显还是它自己的角色口吻，不该像 `002`。

### 失败现象

- `003` 完全不能发起。
- `003` 第一轮能开，第二轮不自动续。
- `003` 聊完不收尾。

### 失败判读

- 这条失败优先记为：`NPC own`

## 6.3 必测 3：玩家自己说话时直接跑开

### 编号

- `ACC-03`

### 入口条件

- 对 `002` 或 `003` 都可以。

### 操作步骤

1. 靠近 NPC 并按 `E` 发起。
2. 玩家气泡刚开始打字时，立刻往外跑出交互范围。
3. 不要立刻再靠回去，先看收束。

### 预期结果

- 不应再出现“玩家气泡卡在第一个字一直鬼畜”。
- 不应再出现“玩家气泡保持半透明残留”。
- 会进入一条明确的离场收束：
  - 玩家离场句
  - NPC 反应
  - 最后双方收尾

### 失败现象

- 玩家气泡卡在一个字来回抖。
- 玩家气泡一直残留。
- 只有靠回 NPC 附近，旧文本才继续往下打。
- NPC 进入等待态后卡住不收。

### 失败判读

- 这条失败优先记为：`NPC own`

## 6.4 必测 4：NPC 等回复时跑开

### 编号

- `ACC-04`

### 操作步骤

1. 发起聊天。
2. 等玩家第一句完成。
3. 在 NPC 还没开始回话、处于“等 NPC 回复”的短停顿时跑开。

### 预期结果

- 不会残留旧句。
- 会触发离场收束。
- NPC 不会永远停在“还等着继续聊”的状态。

### 失败现象

- NPC 一直像在等下一句。
- 会话逻辑结束了，但旧气泡还挂着。

### 失败判读

- 这条失败优先记为：`NPC own`

## 6.5 必测 5：NPC 正在说话时跑开

### 编号

- `ACC-05`

### 操作步骤

1. 发起聊天。
2. 等 NPC 开始回话。
3. 在 NPC 句子正在打字时直接跑开。

### 预期结果

- NPC 旧句会被中断收束，不应卡死。
- 之后会出现离场句 / NPC 反应链，或者至少干净结束。

### 失败现象

- NPC 句子被卡在半句。
- NPC 气泡或玩家气泡一直挂着。
- 跑开后过很久都不收尾。

### 失败判读

- 这条失败优先记为：`NPC own`

## 6.6 必测 6：自动续聊和 `E` 职责

### 编号

- `ACC-06`

### 操作步骤

1. 发起聊天。
2. 第一轮说完后，先不要按 `E`。
3. 观察它是否自己续到第二轮。
4. 再在某一轮打字过程中按 `E`，看是不是只跳过当前动效。

### 预期结果

- 不按 `E` 也会自己续下一轮。
- 按 `E` 时：
  - 当前字会快速结束
  - 但不会把整句、整轮、整场对话直接跳没

### 失败现象

- 不按 `E` 就停死。
- 一按 `E` 整句直接没了，或者直接跳到下一轮。

### 失败判读

- 这条失败优先记为：`NPC own`

## 6.7 必测 7：正常聊完后的收尾

### 编号

- `ACC-07`

### 操作步骤

1. 发起聊天。
2. 不跑路，让整段自然聊完。
3. 观察最后收尾。

### 预期结果

- 不应只收一边。
- 不应留下半透明气泡。
- 不应出现“逻辑结束了但气泡还在”的尾巴。

### 失败现象

- 玩家气泡收了，NPC 气泡没收。
- 或者反过来。
- 收尾时闪烁、鬼畜、硬停半句。

### 失败判读

- 这条失败优先记为：`NPC own`

## 6.8 必测 8：双气泡样式差异与重叠

### 编号

- `ACC-08`

### 操作步骤

1. 发起一次完整聊天。
2. 特别观察玩家气泡和 NPC 气泡：
   - 外观
   - 方向
   - 位置
   - 是否压在一起

### 预期结果

- 玩家和 NPC 的气泡不应长得像同一套。
- 会话期两边应有明显区分。
- 至少在单对单闲聊里，不应重叠到看不清谁是谁。

### 失败现象

- 两边气泡几乎一样。
- 玩家和 NPC 的气泡压在一起，很难读。
- 同屏时明显不知道先看谁。

### 失败判读

- 单对单情况下优先记为：`NPC own`
- 多 NPC 同屏严重混乱可先记为：`NPC 观察项（后续可能再拆 scene/UI）`
