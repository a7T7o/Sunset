## 1. `0.0.2` 的总目标

把 NPC 从“只有 001 能走 spring-day1 正式剧情 + 其余 NPC 只会单向短反馈”的状态，推进到：

1. `001 / 002 / 003` 都有清晰的 NPC 聊天入口
2. 正式剧情轨与 NPC 非正式聊天轨边界明确
3. NPC 非正式聊天是玩家按 `E` 发起的双向闭环
4. 关系阶段真正作用到聊天体验
5. 跑路中断 / 情绪 / 好感变化进入第一版可玩状态

## 2. 状态字段定义

- `pending`
  - 还没开始
- `in-progress`
  - 已进入施工
- `done`
  - 已完成且有对应证据
- `blocked-hotfile`
  - 被 `Primary.unity`、共享热区或单实例 live 写窗口阻塞
- `blocked-cross-owner`
  - 与 `spring-day1` 或其他线程存在交叉 ownership，必须先划边界
- `redefined`
  - `0.0.1` 里旧任务口径失效，已在 `0.0.2` 中重定义

## 3. 总执行纪律

`0.0.2` 必须继续遵守：

1. 不把 `Primary.unity` mixed cleanup 偷换成业务推进
2. 不碰 `DialogueChinese*`
3. 不碰 `GameInputManager.cs`，除非确实被证明成为当前阻塞点
4. 不抢工作台提示 UI 的业务责任
5. 不再把“靠近时 NPC 单向冒一句”包装成“非正式聊天闭环已完成”
6. 任何“已验收 / done”都必须以用户明确测到为准，没说就是未放行
