# 2026-04-02 NPC 本轮自省与下一轮施工清单

## 1. 先把用户这轮新裁定钉死

以下内容是本轮新的硬要求，后续不允许再凭感觉发挥：

### 1.1 气泡避让与箭头

- 玩家 / NPC 气泡现在避让过头了。
- 正确要求不是“大幅分家”，而是：
  - 适当避让
  - 不要严重重叠
  - 但也不要飞得很远
- 气泡本体可以微调偏移；
- 但箭头不能乱跑。
- 箭头要跟着说话方头顶走，保持“谁在说话，箭头就指谁”。

### 1.2 自动跳过时长

新的硬公式已经钉死：

- 基础时间：`1.0s`
- 每个字：`0.08s`
- 也就是：
  - `10` 个字约 `1.8s`
- 长句不再像现在这样跳得太快。

### 1.3 NPC 气泡样式

- 当前 NPC 气泡样式漂了。
- 下一轮要回到用户最初认可的那套 NPC 壳，不再沿用现在这版错误样式。
- 这件事不能再让我自己“觉得也还行”。

### 1.4 提示壳与遮挡

- 当前 NPC 头上的提示壳仍然会和聊天气泡打架。
- 这件事现状很差，不能再说成“只是小尾巴”。
- 左下角如果当前是 `Spring` 任务语义：
  - `E` 不要显示成闲聊
  - 而要显示进入任务 / 任务相关内容
- 这一条已外包给 `UI` 主刀。

### 1.5 NPC 漫游扎堆

- 当前 NPC 扎堆严重。
- 目标不是取消漫游，而是：
  - 继续漫游
  - 但不要一直互相挤在一起
- 如果 NPC 互撞，就不要只留物理尴尬：
  - 后续要补一组 NPC 之间的吵架 / 拌嘴 / 顶嘴对话包。

## 2. 我这轮没做好的地方

### 2.1 我把避让做成了“逻辑正确但观感难看”

- 我先解决了“不要重叠”这个结构问题；
- 但把偏移做得过大，结果画面被拉散了。
- 这属于典型的：
  - `结构站住了`
  - 但 `真实体验没站住`

### 2.2 我没有守住“箭头必须服务说话归属”

- 当前实现里更在意“气泡别撞”，
- 没有把“箭头必须跟说话方头顶”作为最高约束。
- 这是我对视觉归属优先级判断错了。

### 2.3 我把 NPC 气泡样式做漂了

- 用户已经明确指出：
  - 当前 NPC 气泡样式不对
  - 要回到最初认可的那版
- 这说明我在视觉回退点上没有守住正式基线。

### 2.4 我没有把节奏公式收成硬参数

- 现在自动跳过太快，说明我之前做的是“差不多能用”，不是“按明确公式执行”。
- 这类节奏项以后必须先写死公式，再实现。

### 2.5 我还没有把 NPC 扎堆问题真正进入 scene 级处理

- 之前更多精力放在会话逻辑和中断链。
- 但用户现在已经明确把：
  - 出生点
  - 行动路线
  - 扎堆
  - 互撞后内容反应
  提升成下一阶段正式问题。

### 2.6 我对 shared/UI 与 NPC own 的运行时边界还不够“玩家可感知”

- 虽然我在 owner 上已经把 shared prompt shell 释放出去了，
- 但玩家看到的仍是：
  - 头顶提示还在挡
  - 左下角文案还会说错话
- 也就是说：
  - 工程边界理清了
  - 体验边界还没真正理顺
