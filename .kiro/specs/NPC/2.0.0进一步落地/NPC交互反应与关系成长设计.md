# NPC交互反应与关系成长设计

- 版本：v1.0
- 日期：2026-03-25
- 状态：当前有效
- 关联入口：
  - `需求拆分.md`
  - `NPC系统实施主表.md`

## 1. 文档目的

这份文档负责定义：

- NPC 与玩家之间的轻交互层
- 好感度 / 关系成长方向
- 玩家 / NPC 双气泡体系
- 受击 / 工具命中 / 反应兼容框架

它的目标不是一次写完所有实现，而是先把结构定清楚，避免后续又回到零散特判。

## 2. 双轨交互结构

后续 NPC 交互必须分成两条轨道：

### 2.1 正式对话轨

特点：

- 长内容
- 带推进
- 可受剧情、任务、阶段约束

### 2.2 日常气泡轨

特点：

- 短内容
- 即时表达
- 可重复出现
- 承担环境感、关系感和轻互动反馈

要求：

- 两条轨不能互相替代
- 但要能彼此预热和承接

## 3. 好感度 / 关系成长方向

当前先定义结构，不强制本轮实现完整数值系统。

至少要预留：

- 关系阶段
- 好感度变化原因
- 轻交互语气差异
- 正式对话内容解锁层次

建议最小阶段：

1. 陌生
2. 认识
3. 熟悉
4. 亲近

后续要求：

- 同一句轻交互反馈，不能在所有阶段都一样
- 同一个命中 / 工具行为，也不能在所有关系阶段都得到一样的反应

## 4. 玩家 / NPC 双气泡体系

### 4.1 总原则

- 保持同一套尺寸、边距、动效逻辑
- 保持同一产品体系感
- 但要让玩家气泡与 NPC 气泡一眼可区分

### 4.2 玩家气泡方向

后续玩家气泡应具备：

- 独立配色
- 更适合玩家身份的字体选择
- 与 NPC 气泡统一的节奏感

### 4.3 NPC 气泡方向

后续 NPC 气泡继续承担：

- 单人环境表达
- 对玩家的短反馈
- NPC / NPC 相遇对话
- 状态反馈

## 5. 受击 / 工具命中 / 反应兼容框架

### 5.1 触发来源分类

后续至少要区分：

- 普通接触
- 玩家攻击
- 工具挥击
- 误伤 / 范围影响
- 特殊交互触发

### 5.2 反应层级

建议最小层级：

1. 无视
2. 注意到
3. 轻微不满
4. 警惕
5. 抗议 / 明显反应
6. 关系下降
7. 特殊剧情反应

### 5.3 兼容原则

- 不能直接打穿当前导航 / 漫游状态机
- 不能要求 NPC 运动基线被整体推翻
- 要能按角色差异给出不同反馈
- 要能和好感度联动

## 6. 当前直接可落地的第一刀

本轮直接可落地、且风险低的不是完整反应系统，而是：

1. 建立更清晰的人设化日常话术基线
2. 把 `003` 的验证样本文案从纯测试句，推进到“带研究者角色感的验证句”
3. 为后续关系成长与命中反应留下分类空间，而不在本轮硬写一套大而全系统

## 7. 后续实施优先级

推荐顺序：

1. 先把角色化日常表达稳定
2. 再把玩家 / NPC 双气泡样式规范定住
3. 再接轻交互与关系阶段
4. 最后接受击 / 工具命中 / 反应兼容

## 8. 当前一句话结论

这条线当前不该急着一次性做完整反应系统，而是先把“角色化日常表达 + 关系成长结构 + 双气泡视觉方向”定稳，再接更重的交互反应逻辑。
