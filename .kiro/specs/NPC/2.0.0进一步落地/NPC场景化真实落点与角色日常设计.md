## 1. 文档目的

这份文档只负责 NPC 的场景化真实落点与角色日常表达。

它重点回答：

- 001 / 002 / 003 在村子里各自是谁
- 他们应该出现在哪里
- 他们为什么会停在某些地方
- 他们平时会说什么
- 他们彼此相遇时，为什么会聊、聊什么

它不负责导航底座实现，也不负责受击 / 工具命中反应细节。

## 2. 当前角色基线

### 2.1 `001` 村长

角色定位：

- 村务核心人物
- 稳重
- 习惯先观察再发言
- 关注秩序、生产、节奏和人的状态

说话风格：

- 简洁
- 带判断
- 不轻浮
- 有“先把事情稳住”的气质

### 2.2 `002` 村长女儿

角色定位：

- 更贴近日常生活层
- 情绪和观察都更细
- 对人与人的关系、环境感受更敏感

说话风格：

- 更柔和
- 更生活化
- 更容易流露当下情绪
- 更适合承载轻松、温度感强的气泡内容

### 2.3 `003` 村长二儿子 / 研究型

角色定位：

- 偏研究和观察
- 容易注意到现象、节奏、路径、材料和变化
- 会把很多日常事情当成可观察对象

说话风格：

- 有记录感
- 偏推测和思考
- 对细节敏感
- 会自然出现“我得再看一遍”“这个现象值得记一下”之类表达

## 3. 场景化落点原则

### 3.1 不是只摆一个 `Home Anchor`

场景化真实落点至少包含：

- 家 / 常驻区
- 常见停留点
- 经过点
- 更可能遇见其他人的公共点
- 更适合单人自言自语的边缘点

### 3.2 每个 NPC 应有不同的空间倾向

`001`：

- 更适合靠近村庄主通路、广场、村务感更强的位置
- 视野相对开阔，像是在看村子的运行状态

`002`：

- 更适合庭院边缘、采光舒服、生活气更强的位置
- 更容易在让人觉得“会发呆、会顺手说几句”的地方出现

`003`：

- 更适合研究台、边角观察点、路口观察位、地形变化明显的位置
- 更像是因为“想看看这里的变化”而停留

### 3.3 相遇不该纯靠距离触发

后续 NPC / NPC 对聊的自然原则：

- 先有合理空间关系
- 再有合理时间关系
- 最后才有气泡触发

不允许长期停留在“两个 NPC 靠近了就突然对话”的纯机械观感。

## 4. 日常气泡分类

### 4.1 单人环境气泡

用途：

- 角色在长停留、短停留或观察时对环境的自然表达

要求：

- 必须符合角色身份
- 不再使用纯测试口吻

### 4.2 玩家靠近时的轻响应气泡

用途：

- 玩家靠近但还没进入正式对话时的轻量反馈

要求：

- 要有礼貌层级
- 后续要能和好感度联动

### 4.3 NPC / NPC 相遇对话

用途：

- 营造“他们彼此认识、关系不同、说话方式不同”的生活感

要求：

- 不能所有组合都说同样的话
- 至少要区分：
  - `001 <-> 002`
  - `001 <-> 003`
  - `002 <-> 003`

### 4.4 状态型气泡

用途：

- 忙碌、思考、观察、疲惫、轻松、被打扰时的短表达

要求：

- 未来可和关系、场景、命中反应系统联动

## 5. 当前直接落地的第一刀

本轮直接落地的内容只做“现在就能安全推进、又不撞导航主战场”的部分：

1. 为 `001 / 002 / 003` 建立独立的角色化日常 roam profile
2. 让 prefab 不再共用一套通用环境聊天文案
3. 让 `003` 的验证样本文案也开始带角色感，而不是继续停留在纯测试味句子
4. 让生成器以后重新生成这三名 NPC 时，也默认走各自的人设 profile

下面这些仍是下一刀：

场景化真实落点的第一步，不是继续给 NPC 加测试功能，而是先让 `001 / 002 / 003` 开始说出“像他们自己会说的话”，并为后续真实空间落点保留明确的角色方向。
